# code_test

알고리즘 코드 테스트 문제를 직접 생성하고, 풀고, 검토하는 저장소입니다. 런타임과 테스트는 모두 **Bun**을 사용합니다.

## 디렉터리 구조

문제 하나당 디렉터리 하나를 사용하며, 다음 파일들이 나란히 위치합니다.

```
src/<유형>/<문제명>/
├── <문제명>.ts             # 문제 주석 + 풀이 함수
├── <문제명>-problem.md     # 완성된 문제 서술
├── <문제명>.test.ts        # 테스트 코드
├── <문제명>-guide.mdx      # 문제 해설서 (React 시뮬레이션 포함 MDX)
└── <문제명>-analysis.md    # 풀이 분석 보고서
```

## 문제 생성 방법

`gen-problem` 스킬을 사용합니다.

```
/gen-problem <파일 경로(확장자 제외)>
```

동작 순서:

1. `src/<파일명>.ts`의 주석을 참조하여 완전한 문제를 만듭니다.
2. 문제는 `src/<파일명>-problem.md`로 작성됩니다. 문제에는 **함수 인터페이스, 제약 조건, 문제 상세, 예시**가 포함됩니다.
3. 문제에 대응하는 테스트 코드를 작성하고, 커버리지가 충분한지 검토합니다. 테스트 구성 기준은 `.claude/skills/gen-problem/checklist.md`를 따릅니다.
   - 바운더리 테스트
   - 성능 테스트 (CPU가 1초에 10^8 연산 가능하다는 가정)
   - 모든 엣지 케이스 테스트

조건: 함수 파라미터명에 언더바 prefix를 쓰지 않으며, 문제에 풀이 힌트를 포함하지 않습니다.

## 문제 검토 방법

풀이를 작성한 뒤 `analyze-solution` 스킬로 분석 보고서를 받습니다.

```
/analyze-solution <파일 경로(확장자 제외)>
```

분석 항목:

1. 문제 해결 접근 방향이 올바른지
2. `bun test` 결과에 기반한 정확성
3. 최적화 가능성 — 개선 여지가 있으면 핵심 아이디어를 상세히 설명하고, 알고리즘 원형이 있다면 원형에서 해결책을 유도하는 전개 과정을 기술

보고서는 `src/<파일명>-analysis.md`에 작성됩니다. 버그 수정 시에는 개선 내용과 남은 버그가 추가되고, 풀이 컨셉이 바뀐 경우 기존 내용을 보존한 채 새 분석이 추가됩니다. 보고서의 '개인 정리' 항목은 수정되지 않습니다.

## 문제 가이드 생성 방법

문제 풀이 해설서가 필요하면 `guide-for-problem` 스킬을 사용합니다.

```
/guide-for-problem <파일 경로(확장자 제외)>
```

`src/<파일명>.ts`의 주석을 참고하여 `src/<파일명>-guide.mdx`를 생성합니다. 해설서는 `.claude/skills/guide-for-problem/solving-problem-canvas.md` 구조를 따르며, 핵심 아이디어는 다음 순서로 구성됩니다.

1. 원천이 되는 아이디어 및 알고리즘 설명
2. 그 원형에서 문제의 솔루션으로 유도하는 과정 (생략 없이, 주장에는 이유를 동반)

해설서는 MDX 문서로, 알고리즘 전개 과정을 단계별로 조작·재생할 수 있는 React 시뮬레이션(`## 시뮬레이션` 섹션)을 포함합니다. 시뮬레이션은 공용 모듈 `src/_guide-sim`을 `import` 해 사용하며, 각 해설서는 선언형 `steps` 데이터와 프리셋 `view`만 정의합니다.

## 테스트 실행

```bash
bun test                          # 전체 테스트
bun test src/<경로>/<파일명>.test.ts  # 단일 파일
bun test --watch                  # 워치 모드
```
