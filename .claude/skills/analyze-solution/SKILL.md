## 솔루션 src/$ARGUMENTS.ts에 대해 분석한다.

1. 문제 해결 접근 방법에 대한 방향성이 올바른지 분석한다.
2. !`bun test src/$ARGUMENTS.test.ts 2>&1; true` 테스트 결과에 기반하여, 정확성 측면에서 분석한다.
3. 최적화 측면에서 분석한다.
   3.1. 개선 가능하다면, 핵심 아이디어에 대해 자세히 설명한다.
   3.1.1 알고리즘 원형이 존재한다면, 원형에서 문제 해결책을 유도하는 전개 과정을 꼼꼼히 기술한다.

4. 분석 보고서는 src/$ARGUMENTS.md에 작성해라.
   4-1. 버그 수정의 경우, 보고서에 개선된 내용과 남은 버그에 대한 내용을 추가해 달라.
   4-2. 컨셉이 바뀐 경우, 보고서 내용을 변경하지 말고, 추가해 달라.

## 제약 조건

- 파일에 변경이 없는데 재실행 하면, 무시해라.
- '개인 정리' 항목은 수정하지 말아라.
- 파일의 최신 상태를 보고 있는지 확인 해라
- 핵심 아이디어 설명 시, 초등학생도 이해할 수 있을 정도로 꼼꼼하게 설명한다. 단, 비유나 묘사 없이, 정식 표기법과 용어만 사용한다.
