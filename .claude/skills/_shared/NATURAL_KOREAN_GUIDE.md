# 자연스러운 한국어 가이드 (영어투 교정 + 수사 팔레트)

이 문서는 **집필(tech-deepdive)** 과 **글쓰기 검토(review-writing)** 가 공통으로 참조하는, *문장이 한국어답게
읽히는가*에 대한 기준이다. [`STYLE_GUIDE.md`](STYLE_GUIDE.md)가 **저자 고유의 목소리**(톤·호흡·도입/마무리 패턴)를
다룬다면, 이 문서는 그 목소리를 **번역기에서 갓 뽑은 듯한 영어투**로 망치지 않도록 막는다. 두 문서는 함께 본다.

## 0. 왜 이 가이드인가 — "영어투"의 정체

영어를 한국어로 옮길 때 단어는 우리말인데 **문장의 골격이 영어**로 남는 경우가 있다. 독자는 문법 오류가 없는데도
"번역체 같다", "AI가 쓴 것 같다"고 느낀다. 원인은 대개 다음 셋이다.

1. **영어 수사의 직역** — "the myth of X"를 `X라는 신화`로, "a costly misconception"을 `비싼 오해`로 옮긴다.
   영어에서는 관용이지만 한국어 입말에는 없는 비유다.
2. **영어 문장 구조의 잔존** — 무생물 주어가 사람처럼 행동하고(이 기능은 ~을 *가능하게 한다*), 가주어("그것은 ~이다")가
   남고, 명사화가 과하고(구현은 ~을 *요구한다*), 피동이 겹친다(*고려되어져야* 한다).
3. **레지스터 부정합** — 친근한 본문에 사설 어미("~해야 합니다")가 섞이는 식의 톤 충돌(§4).

원칙은 하나다. **번역하지 말고, 그 뜻을 한국어 화자라면 어떻게 말할지로 다시 쓴다.** 의미가 같아도 표현이 다르면
다시 고른다.

## Part A — 영어투 안티패턴 카탈로그

각 항목: **증상(영어 원형) → 왜 어색한가 → 자연스러운 대안.** 검토 시 이 표를 그대로 검색용으로 쓴다.

### A1. 영어 비유·수사의 직역 (가장 잘 드러나는 신호)

| 영어투 | 원형 | 자연스러운 대안 |
| --- | --- | --- |
| `X라는 신화` | the myth of X | `흔히 X라고들 하지만`, `X라는 흔한 오해`, `X라고 믿기 쉽지만` |
| `비싼 오해`, `값비싼 실수` | a costly misconception | `이 오해는 대가가 크다`, `이걸 놓치면 크게 데인다`, `뼈아픈 착각` |
| `~의 끝에서` | at the end of the day | `결국`, `따지고 보면` |
| `하루의 끝에` | at the end of the day | `결국`, `무엇보다` |
| `그림의 큰 부분` | the big picture | `전체 그림`, `큰 틀`, `판 전체` |
| `방 안의 코끼리` | the elephant in the room | (직역 금지) `모두가 알면서 안 꺼내는 문제` |
| `X를 위한 은총알은 없다` | no silver bullet | `X를 한 방에 푸는 묘책은 없다` |

> 신호어: **신화 / 환상 / 비싼·값비싼 + 추상명사 / 은총알 / 빙산의 일각 / 양날의 검** 이 추상 개념과 붙으면
> 직역을 의심한다. 한국어 입말에 같은 뜻의 토착 표현이 있으면 그쪽을 쓴다.

### A2. 무생물 주어 + 능동 (사물이 사람처럼 행동)

영어는 "This feature **enables** you to…", "The protocol **allows**…"처럼 사물을 주어로 세운다. 한국어는
**사람을 주어로** 두거나 사물을 도구로 내린다.

- ✗ `이 기능은 빠른 연결을 가능하게 합니다.` → ✓ `이 기능을 쓰면 빠르게 연결할 수 있습니다.`
- ✗ `STUN은 공인 IP를 알 수 있게 해줍니다.` → ✓ `STUN으로 공인 IP를 알아냅니다.`
- ✗ `이 글은 ~을 설명하는 것을 목표로 합니다.` → ✓ `이 글에서는 ~을 설명합니다.`

### A3. 가주어·형식주어 ("그것은 ~이다")

영어의 "It is X that…", "There is…" 구문을 직역하지 않는다. 한국어는 주어를 비우거나 핵심을 앞세운다.

- ✗ `그것이 바로 ICE가 하는 일입니다.` → ✓ `ICE가 하는 일이 바로 이것입니다.` / `이게 ICE의 일입니다.`
- ✗ `~라는 사실을 기억하는 것이 중요합니다.` → ✓ `~라는 점을 꼭 기억하세요.`

### A4. 과한 명사화 (동사로 풀기)

영어는 동작을 명사로 굳혀 쓴다("implementation **requires** configuration"). 한국어는 **동사로 풀 때** 읽힌다.

- ✗ `이 기능의 구현은 별도 서버의 운영을 요구합니다.` → ✓ `이 기능을 구현하려면 서버를 따로 운영해야 합니다.`
- ✗ `연결의 수립을 위한 후보의 수집` → ✓ `연결을 맺으려고 후보를 모으는 과정`

### A5. 피동 남용·이중 피동

- ✗ `~로 고려되어집니다`, `~되어지는` (이중 피동) → ✓ `~로 봅니다`, `~되는`
- ✗ `이 값은 브라우저에 의해 설정됩니다.` → ✓ `이 값은 브라우저가 설정합니다.`
- 능동으로 바꿀 수 있으면 능동으로. 행위자가 분명하면 `~에 의해`를 버린다.

### A6. 전치사·접속의 직역

| 영어투 | 원형 | 대안 |
| --- | --- | --- |
| `~에 대하여`, `~에 대한` 남발 | about / regarding | `~을`, `~의`, `~를 두고` (대개 생략 가능) |
| `~을 통해` 남발 | through / via | `~로`, `~으로써`, `~을 거쳐` |
| `~의 관점에서` 남발 | from the perspective of | `~로 보면`, `~ 입장에서` |
| `~와 함께` (도구) | along with | `~와`, `~과 같이` (사람 아닌데 `함께`면 의심) |
| `~에 기반하여` | based on | `~을 바탕으로`, `~에 따라` |

### A7. 군더더기 부사·연결어 (영어 filler 직역)

- `기본적으로`(basically) / `사실상`(essentially) / `결론적으로`(in conclusion) 을 **논증 없이** 문두에 깔지 않는다.
  진짜 결론일 때만 쓴다.
- `~라고 말할 수 있습니다`(it can be said that) → 직접 단정한다. `~입니다.`
- `~할 필요가 있습니다`(need to) → `~해야 합니다`.

### A8. 복수 `들`·관사 `하나의` 남발

- 한국어는 복수를 굳이 표시하지 않아도 된다. ✗ `개발자들은 문제들을 만납니다.` → ✓ `개발자는 여러 문제를 만납니다.`
- `a/an`을 옮긴 `하나의`는 대개 군더더기. ✗ `하나의 예시로` → ✓ `예를 들어`.

### A9. 호칭 `당신`의 기계적 사용

독자를 끌어들이는 직접 호명(STYLE_GUIDE F6)은 좋지만, 영어 "you"를 문장마다 `당신`으로 옮기면 번역체가 된다.
한국어는 주어를 비우거나 `여러분`, 청유형(`~해 보세요`)으로 끌어들인다.

- ✗ `당신은 당신의 코드에서 당신이 원하는 것을…` → ✓ `여러분 코드에서 원하는 것을…`

## Part B — 자연스러운 수사 팔레트 (레지스터)

어색함은 단어뿐 아니라 **레지스터(말의 결)**가 어긋날 때도 생긴다. wiki(`korean-writing-styles`) 3종을 의식하고 **고른다**.

- **서정체** — 개인 경험·고유명사·시점으로 현장감. *오프닝 후킹*에 좋다. 함정: 추상화되면("많은 사람들이",
  "우리는 종종") 가짜 서정체가 된다. 진짜는 구체(쌍문동·2002년 봄호처럼)로 산다.
- **일상체** — 1인칭 고백 + 구체 수치 + 독자 호칭. *기술 블로그 본문의 기본*. 솔직한 한계 노출이 신뢰를 만든다.
  함정: 이모지·볼드 남발은 산만.
- **정론체** — 수치로 현황 → 원인 → 구체적 요구. *결론·설득*에 좋다. 함정: 모든 문장이 `~해야 합니다`로 수렴하면
  AI 티. **결론 어미를 다양화**한다(`~을 권합니다`, `~이면 충분합니다`, `여기서 갈립니다`, `시험대에 올랐습니다`).

**기술 글의 의식적 혼합**: 서정체 오프닝 → 일상체 본문 → 정론체 결론. **가장 흔한 실수**는 *정론체 어휘 + 일상체 구조*의
혼종 — "결론적으로 이 기능은 반드시 활용해야 합니다" 같은 어색한 문장. 본문 톤과 어미를 맞춘다.

> 이 팔레트는 STYLE_GUIDE Part 1(저자 고유 톤: 격식체 기본 + 단문 강타 + 당위 마무리)을 **대체하지 않는다.**
> 저자 색을 유지하되, 그 안에서 레지스터가 어긋나거나 영어투가 끼지 않게 다듬는 보조 기준이다.

## Part C — 자가 점검 (집필·검토 공용)

집필자는 초고 후, 검토자는 채점 시 아래를 본문 검색으로 훑는다. 하나라도 걸리면 §A 대안으로 고친다.

1. **직역 비유**: `신화 / 환상 / 비싼·값비싼 [추상명사] / 은총알 / 빙산 / 양날` 검색 → 토착 표현으로.
2. **무생물 주어**: `~은/는 ~을 가능하게/제공/허용` 패턴 → 사람 주어 또는 도구격으로.
3. **가주어**: `그것은/그것이 ~이다`, `~하는 것이 중요/필요하다` → 핵심을 앞세워 직접.
4. **명사화 사슬**: `~의 ~의 ~` 명사 3연속, `구현은/수립은/수집은 ~을 요구/필요` → 동사로 풀기.
5. **이중 피동**: `되어진/되어집/여겨지어` → 능동 또는 단일 피동.
6. **전치사 직역**: `~에 대하여 / ~을 통해 / ~의 관점에서`가 한 문단에 2회 이상 → 절반 이상 교체.
7. **filler**: 문두 `기본적으로 / 사실상 / 결론적으로`가 논증 없이 → 삭제.
8. **`들`·`하나의` 남발**: 무생물 복수 `들`, 관사 `하나의` → 제거.
9. **결론 어미 단조**: 마지막 단락 어미가 전부 `~해야 합니다`면 → 다양화.
10. **레지스터 충돌**: 친근한 본문에 사설 어미가 튀거나, 그 반대인 곳 → 톤 통일(§B).

> 적용 경계: **저자 고유의 격식체·단문 강타·당위 마무리는 영어투가 아니다**(STYLE_GUIDE Part 1 보존). 이 가이드는
> *번역체·레지스터 어긋남*만 잡는다. 자연스러움을 핑계로 저자 색을 깎지 않는다.
