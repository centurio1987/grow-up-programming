# knuthOptimization 해설

## 성능 목표 예측

| 항목 | 값 |
|------|-----|
| 입력 크기 $n$ | $1 \leq n \leq 5000$ |
| 값 범위 | $\text{freq}[i] \geq 0$ |

**naive 구간 DP의 한계.** 구간 DP 점화식

$$dp[i][j] = \min_{i \leq k < j} \bigl( dp[i][k] + dp[k+1][j] \bigr) + S(i,j)$$

을 순진하게 구현하면 구간 수 $O(n^2)$, 각 구간에서 분할점 $k$를 $O(n)$ 탐색하므로 전체 $O(n^3)$이다. $n = 5000$이면 $1.25 \times 10^{11}$ 연산으로 수 분이 걸려 사실상 불가능하다.

**목표 복잡도와 근거.** 비용 함수 $S(i,j)$가 사각 부등식과 단조성을 동시에 만족하면 최적 분할점 $opt[i][j]$가 단조성 조건

$$opt[i][j-1] \leq opt[i][j] \leq opt[i+1][j]$$

을 만족한다. 이를 이용해 탐색 범위를 줄이면 전체 $O(n^2)$. $n = 5000$에서 $2.5 \times 10^7$으로 충분히 빠르다.

**공간 복잡도.** $dp[n][n]$과 $opt[n][n]$ 두 배열이 각각 $O(n^2)$. $n = 5000$이면 $25 \times 10^6$ 개의 숫자로 약 200 MB — 언어 및 환경에 따라 한계에 달할 수 있으므로 정수 크기(32비트 vs 64비트)에 주의한다.

---

## 목표 함수

```ts
function knuthOptimization(freq: number[]): number
```

### 파라미터 표

| 파라미터 | 의미 | 제약 |
|---------|------|------|
| `freq` | 인접 파일 크기 배열. `freq[i]`는 $i$번째 파일의 크기 | 길이 $n$, $1 \leq n \leq 5000$, $\text{freq}[i] \geq 0$ |

**반환값.** $n$개의 파일을 하나로 병합할 때까지 두 인접 그룹을 반복 병합하는 최소 총 비용. 병합 비용은 두 그룹 크기의 합이다.

### 엣지케이스

1. **$n = 1$** — 병합할 파일이 없으므로 비용 `0` 반환.
2. **$n = 2$** — 두 파일을 한 번만 병합. `freq[0] + freq[1]` 반환.
3. **모든 `freq[i] = 0`** — 모든 병합 비용이 0이므로 `0` 반환.
4. **$\text{freq}[i]$가 매우 클 때** — 총 비용이 $n \times \max(\text{freq}) \times n$ 수준까지 커질 수 있으므로(`dp` 값이 $O(n \cdot \text{sum})$) JavaScript `number` 범위(최대 $2^{53}$) 안에 있는지 확인한다.

## 핵심 아이디어

**핵심 아이디어**: "구간 DP에서 최적 분할점이 인접 구간의 최적점 사이에 갇힌다 — 이 단조성 하나로 $O(n^3)$을 $O(n^2)$으로 줄인다."

구간 $[i, j]$를 병합하는 문제에서, 비용 함수가 사각 부등식과 단조성을 동시에 만족하면 $opt[i][j-1] \leq opt[i][j] \leq opt[i+1][j]$가 항상 성립한다. 각 구간 길이 레벨에서 탐색 범위들이 서로 겹치지 않고 이어지는(telescoping) 구조 덕분에, 한 레벨의 전체 탐색 비용이 $O(n)$으로 줄어 전체가 $O(n^2)$이 된다.

**풀이 구조**
1. 누적합으로 구간 합 $S(i,j)$를 $O(1)$에 계산할 수 있도록 전처리한다.
2. 구간 길이 2부터 $n$까지 오름차순으로 처리하며, 각 구간 $[i,j]$에 대해 $k \in [opt[i][j-1], opt[i+1][j]]$ 범위에서 최솟값을 탐색한다.
3. 최적 분할점 $opt[i][j]$를 기록해 이후 더 긴 구간의 탐색 범위 계산에 활용한다.

**조건**: 비용 함수 $S(i,j)$가 사각 부등식과 단조성 조건을 모두 만족해야 한다. 대표적인 예는 구간 합(누적합) 형태의 비용이다.

**대표 예시**: 파일 병합 최소 비용
$n$개의 파일을 두 인접 그룹씩 반복 병합해 하나로 합칠 때, 각 병합 비용은 두 그룹 크기의 합이다. $S(i,j) = \sum \text{freq}[t]$는 사각 부등식과 단조성을 만족하므로 Knuth 최적화가 적용된다. $n=5000$에서 naive $O(n^3) \approx 1.25 \times 10^{11}$을 $O(n^2) \approx 2.5 \times 10^7$로 줄인다.

**언제 쓰나**
구간 DP 점화식이 $dp[i][j] = \min_k(dp[i][k] + dp[k+1][j]) + S(i,j)$ 형태이고, 비용 함수 $S$가 사각 부등식과 단조성을 만족한다고 확인된 경우에 적용한다.

---

### 원형 아이디어와 naive 접근

파일 병합 순서를 결정하는 가장 단순한 방법은 모든 병합 순서를 시도하는 것이다. $n$개의 파일을 하나로 합치는 방법은 이진 트리 구조로 나타낼 수 있고, 이는 카탈란 수만큼 많아 완전 탐색이 지수 시간이 걸린다.

다음 단계로 자연스럽게 떠오르는 생각이 구간 DP다. "구간 $[i, j]$를 하나의 그룹으로 합치는 최솟값"을 점화식으로 재사용하는 방식이다. 그러나 이 점화식의 분할점 $k$ 탐색에 $O(n)$이 들어 전체 $O(n^3)$이 폭발 지점이 된다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1.** 비용 함수 $S(i,j) = \sum_{t=i}^{j} \text{freq}[t]$ (누적합)은 사각 부등식(Quadrangle Inequality)을 만족한다. $a \leq b \leq c \leq d$에 대해 $S(a,c) + S(b,d) \leq S(a,d) + S(b,c)$가 성립하는데, 이는 두 구간이 겹치면 합이 더 작다는 단순한 사실에서 비롯된다.
- **관찰 2.** $S(i,j)$는 추가로 단조성 조건 $S(b,c) \leq S(a,d)$ (for $a \leq b \leq c \leq d$)도 만족한다. 이 두 조건이 함께 성립할 때 Knuth's Optimization의 최적점 단조성이 유도된다.
- **관찰 3.** 최적점 단조성 $opt[i][j-1] \leq opt[i][j] \leq opt[i+1][j]$를 이용하면, 구간 길이 $\ell = j - i$가 고정된 레벨에서 모든 $(i, j)$ 쌍의 $k$ 탐색 범위 합이 telescoping 구조로 정확히 $O(n)$이 된다.

### 관찰을 형식화: 상태/구조 정의

**상태 정의:**

$$dp[i][j] = \text{구간 } [i, j] \text{를 하나의 그룹으로 병합할 때의 최소 총 비용}$$

$$opt[i][j] = dp[i][j] \text{를 달성하는 최적 분할점 } k \text{ (} i \leq k < j \text{)}$$

기저: $dp[i][i] = 0$, $opt[i][i] = i$.

이 형태여야 하는 이유: 부분 구간 $[i,k]$와 $[k+1,j]$를 독립적으로 병합한 뒤 두 그룹을 하나로 합칠 때 비용이 $S(i,j)$로 고정된다. 이 구조가 하위 구간의 결과를 상위 구간에 재사용 가능하게 한다.

**최적점 단조성이 성립하는 이유:**

QI와 단조성 두 조건을 결합하면 다음이 성립한다. 분할점 $k$가 $[i,j-1]$의 최적이라면, $j$를 한 칸 늘렸을 때($[i,j]$) 최적점은 $k$ 이상이어야 한다. 왜냐하면 $S$가 QI를 만족하면 "더 넓은 구간에서는 더 오른쪽 분할이 유리"한 방향으로 최적점이 이동하기 때문이다. 마찬가지로 $i$를 한 칸 좁히면($[i+1,j]$) 최적점은 $k$ 이하여야 한다.

### 점화식 또는 핵심 연산

**점화식 (최적점 범위를 좁힌 버전):**

$$dp[i][j] = \min_{k \in [opt[i][j-1],\, opt[i+1][j]]} \bigl( dp[i][k] + dp[k+1][j] \bigr) + S(i,j)$$

각 항의 의미:
- $dp[i][k]$: 좌측 구간 $[i,k]$를 최적으로 병합한 비용.
- $dp[k+1][j]$: 우측 구간 $[k+1,j]$를 최적으로 병합한 비용.
- $S(i,j)$: 두 그룹($[i,k]$와 $[k+1,j]$)을 최종적으로 하나로 합칠 때 드는 비용(두 그룹 크기의 합).

**탐색 범위 합산이 $O(n^2)$인 이유:**

구간 길이 $\ell = j - i$가 고정된 레벨을 생각하면, $k$의 탐색 범위는 $[opt[i][j-1], opt[i+1][j]]$다. 이를 같은 레벨의 모든 $(i, j)$에 대해 합산하면 인접 구간 사이에서 범위가 공유(telescoping)되어 총 합이 $O(n)$이 된다. 레벨 수가 $O(n)$이므로 전체 $O(n^2)$.

### 정당성 — 왜 이것이 옳은가

**귀납적 정당성.** 구간 길이 $\ell$에 대한 귀납법으로 증명한다.

- 기저: $\ell = 1$이면 $dp[i][i] = 0$, $opt[i][i] = i$ — 성립.
- 귀납: 길이 $< \ell$의 모든 구간에서 $dp$와 $opt$가 올바르다고 가정한다. 길이 $\ell$에서 탐색 범위 $[opt[i][j-1], opt[i+1][j]]$가 유효한 최적점을 포함함을 QI와 단조성으로 보장한다(관찰 1, 2). 따라서 최솟값 탐색이 올바른 결과를 낸다.

**까다로운 케이스.**

1. 경계 처리: $opt[i][j-1]$은 길이 $\ell - 1$ 구간의 결과이므로 구간 길이가 1씩 증가하는 순서(`for len in 2..n`)로 처리해야 미리 계산되어 있다. 순서가 바뀌면 미초기화 값을 읽는다.
2. $i + 1 > j - 1$인 경우(길이 2 구간): $opt[i+1][j]$는 $opt[i+1][i+1] = i+1$이고 $opt[i][j-1] = opt[i][i] = i$이므로 탐색 범위가 $[i, i+1]$. 이는 정상이다.
3. $\text{freq}[i] = 0$인 경우: $S(i,j) = 0$이면 $dp[i][j]$가 0이 될 수도 있어 최적점이 임의의 위치가 되는데, 단조성 조건 자체는 유지된다.

### 구현 디테일과 최적화

- **누적합 전처리**: $S(i,j) = \text{prefix}[j+1] - \text{prefix}[i]$ ($O(1)$ 계산)로 구간 합을 미리 구축한다.
- **구간 길이 순서**: 반드시 구간 길이를 2부터 $n$까지 오름차순으로 처리해 짧은 구간 결과가 먼저 채워지도록 한다.
- **INF 초기화**: `dp[i][j]`의 초기값을 충분히 크게 설정한다. `Number.MAX_SAFE_INTEGER / 2`를 사용하면 덧셈 중 오버플로를 피할 수 있다.
- **opt 초기값 선택**: `opt[i][j]`의 초기 탐색 하한을 `opt[i][j-1]`로 설정한다. 단조성에 의해 이 값이 실제 최적점의 하한임이 보장된다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function knuthOptimization(freq):
    n = |freq|
    if n == 1: return 0              // 기저: 파일 1개, 병합 없음

    // 누적합 구축
    prefix[0] = 0
    for i in 0..n-1:
        prefix[i+1] = prefix[i] + freq[i]

    S(i, j) = prefix[j+1] - prefix[i]   // 구간 합 O(1)

    // dp, opt 초기화
    for i in 0..n-1:
        dp[i][i]  = 0                // 원소 하나, 병합 비용 없음
        opt[i][i] = i                // 최적 분할점 = 자기 자신

    // 구간 길이 2 ~ n 순서로 처리
    for len in 2..n:
        for i in 0..n-len:           // i: 구간 시작
            j = i + len - 1          // j: 구간 끝 (불변식: j - i = len - 1)
            dp[i][j]  = INF
            // 탐색 범위: [opt[i][j-1], opt[i+1][j]]
            // 불변식: 이 범위 안에 실제 최적점이 반드시 존재
            lo_k = opt[i][j-1]
            hi_k = (j < n-1) ? opt[i+1][j] : j-1   // 경계 처리

            for k in lo_k .. min(hi_k, j-1):
                val = dp[i][k] + dp[k+1][j] + S(i, j)
                if val < dp[i][j]:
                    dp[i][j]  = val  // 더 작은 비용 갱신
                    opt[i][j] = k    // 최적점 갱신

    return dp[0][n-1]
```

**핵심 불변식:** 구간 길이 $\ell$ 처리 시 모든 길이 $< \ell$의 구간에 대해 `dp`와 `opt`가 확정되어 있으며, $opt[i][j-1] \leq opt[i][j] \leq opt[i+1][j]$ (Knuth 단조성)이 항상 성립한다.
