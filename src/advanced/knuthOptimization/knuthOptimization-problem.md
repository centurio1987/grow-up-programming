# Knuth's Optimization — 최적 파일 병합

## 중요도 · 난이도

| 항목 | 값 |
|------|-----|
| 중요도 | ★ 하 — 특정 분야·고급 |
| 난이도 | 고급 |

## 함수 인터페이스

```ts
export function knuthOptimization(freq: number[]): number;
```

## 제약 조건

- $1 \leq n \leq 5000$ (여기서 $n$ 은 `freq` 의 길이)
- $\text{freq}[i] \geq 0$
- 반환값은 $\geq 0$

## 문제 상세

$n$ 개의 인접한 파일 크기 $\text{freq}[0..n-1]$ 가 주어진다. 인접한 두 그룹을 병합할 때의 비용은 두 그룹의 크기 합이며, 모든 파일이 하나로 합쳐질 때까지 반복한다. 가능한 모든 병합 순서 중 **총 비용의 최솟값**을 반환하라.

누적합을 $S(i, j) = \sum_{t=i}^{j} \text{freq}[t]$ 라 하면, 구간 $[i, j]$ 를 모두 병합하는 최소 비용 $dp[i][j]$ 는 다음과 같다.

$$dp[i][j] = \min_{i \leq k < j} \left( dp[i][k] + dp[k+1][j] \right) + S(i, j)$$

기저:

$$dp[i][i] = 0$$

답은 $dp[0][n-1]$ 이다.

## 예시

| `freq` | 반환값 | 설명 |
| --- | --- | --- |
| `[10]` | `0` | 병합 불필요 |
| `[10, 20]` | `30` | 한 번 병합: $10 + 20 = 30$ |
| `[1, 2, 3]` | `9` | $(1+2)=3$, 이후 $(3+3)=6$, 총 $9$ |
