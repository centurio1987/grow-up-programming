# SuffixArray (접미사 배열) 해설

## 성능 목표 예측

| 연산 | 시간복잡도 | 비고 |
|------|-----------|------|
| constructor | O(n² log n) | naive 정렬; SA-IS 알고리즘으로 O(n) 가능 |
| search | O(m log n) | m = 패턴 길이, 이진 탐색 |
| lcp | O(m) | m = LCP 길이, 문자 직접 비교 |
| longestRepeatedSubstring | O(n²) | 인접 쌍 LCP 최댓값 |
| suffixes | O(n) | 배열 조회 |

공간복잡도: O(n) — 접미사 배열(SA) 자체는 n개의 정수

---

## 목표 함수

| 메서드 | 입력 | 출력 | 설명 |
|--------|------|------|------|
| `constructor(s)` | 문자열 | — | 접미사 배열 구성 |
| `search(pattern)` | 문자열 | number[] | 패턴 등장 위치 (정렬됨) |
| `lcp(i, j)` | 정수 두 개 | number | SA[i], SA[j] 접미사의 LCP 길이 |
| `longestRepeatedSubstring()` | — | string | 두 번 이상 등장하는 가장 긴 부분 문자열 |
| `suffixes()` | — | string[] | 정렬된 접미사 문자열 배열 |

---

## 핵심 아이디어

### 원형 아이디어와 naive 접근

문자열에서 패턴을 찾는 가장 단순한 방법은 텍스트를 처음부터 끝까지 스캔하는 것이다. 패턴 길이 m, 텍스트 길이 n이면 O(nm) 시간이 걸린다. 패턴이 수천 개라면 O(k·nm)으로 현실적으로 불가능해진다.

### 어떤 관찰이 돌파구가 되는가

**핵심 관찰**: 패턴 `p`가 텍스트 `s`에 위치 `i`에서 등장한다는 것은, 접미사 `s[i..]`가 패턴 `p`를 접두사로 가진다는 것과 동치다.

접미사들을 사전순 정렬하면, `p`를 접두사로 가지는 접미사들은 정렬된 배열에서 연속적인 구간을 형성한다. 이 구간을 이진 탐색으로 O(m log n)에 찾을 수 있다.

### 관찰을 형식화

접미사 배열 SA를 구성한 뒤:
1. 이진 탐색으로 `p`가 접두사인 접미사의 구간 `[lo, hi)` 탐색
2. `compareTo(suffix[SA[mid]], p)`: SA[mid] 위치의 접미사가 `p`보다 사전순으로 작은지 비교
3. `[lo, hi)` 범위의 SA 값이 곧 패턴의 등장 위치

### 핵심 연산

**구성**:
```
모든 접미사 인덱스 [0, 1, ..., n-1]을 생성
s[i..]를 기준으로 사전순 정렬
→ SA[k] = k번째로 작은 접미사의 시작 인덱스
```

**이진 탐색 (search)**:
```
lo = 0, hi = n
// 패턴 p가 접두사인 첫 번째 접미사 위치 찾기 (lower bound)
while lo < hi:
  mid = (lo + hi) / 2
  suffix = s.slice(SA[mid], SA[mid] + p.length)
  if suffix < p: lo = mid + 1
  else: hi = mid

left = lo

// upper bound
hi = n
while lo < hi:
  mid = (lo + hi) / 2
  suffix = s.slice(SA[mid], SA[mid] + p.length)
  if suffix <= p: lo = mid + 1  // p를 접두사로 가지는 접미사들도 포함
  else: hi = mid

right = lo
return SA[left..right]
```

### 정당성

접미사들이 정렬되어 있으면 패턴 `p`를 접두사로 가지는 접미사들은 `p` 이상이고 `p` + "최대 문자"(예: `p\xff`) 미만인 구간에 연속으로 놓인다. 이진 탐색의 lower/upper bound로 이 구간을 정확히 찾는다.

### 구현 디테일과 최적화

- **접두사 비교**: 패턴 길이만큼만 슬라이스해서 비교 — `s.slice(SA[mid], SA[mid] + p.length)`
- **경계 처리**: 접미사가 패턴보다 짧은 경우 자동으로 작게 취급됨 (lexicographic)
- **빈 문자열**: 빈 패턴 → 모든 위치 반환 또는 빈 배열 (구현 선택)
- **LCP 배열 (선택적 최적화)**: Kasai 알고리즘으로 O(n)에 인접 LCP 전처리 가능 → `longestRepeatedSubstring`을 O(n)으로 개선

## 수도 코드와 Activity Diagram

### 의사코드

```
class SuffixArray:
  s: string
  sa: int[]  // 정렬된 접미사 인덱스

  constructor(s):
    this.s = s
    n = s.length
    sa = [0, 1, ..., n-1]
    sa.sort((i, j) => compare(s[i..], s[j..]))
    // compare: 두 접미사를 사전순 비교

  search(pattern):
    if s.length == 0 or pattern.length == 0: return []
    lo = lowerBound(pattern)
    hi = upperBound(pattern)
    return sa[lo..hi].sort()

  lowerBound(pattern):
    lo, hi = 0, sa.length
    while lo < hi:
      mid = (lo + hi) >> 1
      suffix = s.slice(sa[mid], sa[mid] + pattern.length)
      if suffix < pattern: lo = mid + 1
      else: hi = mid
    return lo

  upperBound(pattern):
    lo, hi = 0, sa.length
    while lo < hi:
      mid = (lo + hi) >> 1
      suffix = s.slice(sa[mid], sa[mid] + pattern.length)
      if suffix <= pattern: lo = mid + 1
      else: hi = mid
    return lo

  lcp(i, j):
    a = sa[i], b = sa[j]
    count = 0
    while a + count < s.length and b + count < s.length:
      if s[a + count] != s[b + count]: break
      count++
    return count

  longestRepeatedSubstring():
    best = ""
    for k in 0 .. sa.length - 2:
      len = lcp(k, k + 1)
      if len > best.length:
        best = s.slice(sa[k], sa[k] + len)
    return best

  suffixes():
    return sa.map(i => s.slice(i))
```
