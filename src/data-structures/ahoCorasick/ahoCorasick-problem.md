# AhoCorasick (아호-코라식)

## 한 줄 요약
> Trie와 실패 링크를 결합하여 텍스트를 한 번만 스캔하는 것으로 수천 개의 패턴을 동시에 검색하는 오토마톤을 구현하라.

## 스토리

사이버 보안 회사에서 바이러스 탐지 엔진을 개발하고 있다. 악성코드 데이터베이스에 바이러스 시그니처가 10만 개 이상 등록되어 있고, 매 초마다 수백 MB의 네트워크 패킷을 검사해야 한다.

단순히 각 패턴에 KMP를 적용하면 패턴 수 k, 텍스트 길이 n, 패턴 평균 길이 m에 대해 O(k·n) 시간이 걸린다. 10만 개 패턴에 1MB 텍스트라면 초당 처리가 불가능하다.

**아호-코라식(Aho-Corasick) 알고리즘**은 모든 패턴을 하나의 **유한 상태 오토마톤**으로 컴파일한다. 텍스트를 한 번만 스캔하면서 동시에 모든 패턴의 등장 여부를 확인한다. 검색 시간이 O(텍스트 길이 + 총 매칭 수)로, 패턴 수에 무관하다.

실제로 Snort(네트워크 침입 탐지 시스템), 리눅스 커널의 `netfilter`, 안티바이러스 소프트웨어에 이 알고리즘이 적용되어 있다.

## 함수 인터페이스

```ts
export class AhoCorasick {
  constructor(patterns: string[])                        // O(총 패턴 길이 합)
  search(text: string): Map<string, number[]>            // O(텍스트 길이 + 총 매칭 수)
  // 반환값: Map<패턴, 시작 인덱스 배열(오름차순)>
}
```

## 제약 조건

- 시간 제한: 1초, 메모리 제한: 256 MB
- 패턴 수: 최대 1,000개
- 패턴 총 길이 합: 최대 10,000
- 텍스트 길이: 최대 100,000
- 결과 Map은 모든 패턴을 키로 포함 (매칭 없으면 빈 배열)
- 각 패턴의 결과 배열은 오름차순 정렬

## 문제 상세

### 3단계 구성

**1단계: Trie 구성 (Goto 함수)**

모든 패턴을 Trie에 삽입한다. 각 노드는 문자 → 자식 노드 맵을 가진다. 패턴의 마지막 문자 노드에 출력(output) 표시를 한다.

**2단계: 실패 링크(Failure Link) 구성**

BFS로 루트에서 시작하여 각 노드의 실패 링크를 계산한다.

- 루트의 자식: 실패 링크 = 루트
- 노드 v의 문자 c로의 자식 u의 실패 링크 = `goto(failure[v], c)`
  - `goto(failure[v], c)`가 없으면 재귀적으로 실패 링크를 따라가며 탐색
  - 루트까지 도달하면 루트

실패 링크는 "현재 패턴 매칭이 실패했을 때 다음으로 시도할 위치"를 가리킨다. KMP의 실패 함수를 트라이 전체로 확장한 개념이다.

**3단계: 출력 링크(Output Link) 구성**

실패 링크를 따라 도달한 노드가 패턴의 끝이라면, 그 패턴도 현재 위치에서 매칭된다. `output` 집합을 실패 링크 체인을 통해 합산한다.

### 검색

텍스트를 왼쪽에서 오른쪽으로 스캔하며 현재 상태(노드)를 유지한다.

- 각 문자에 대해 `goto` 함수를 호출
- `goto` 실패 시 `failure` 링크를 따라 재시도
- 현재 노드의 `output` 집합에 있는 모든 패턴의 결과에 현재 위치를 기록

## 예시

```ts
const ac = new AhoCorasick(["he", "she", "his", "hers"]);

ac.search("ahishers");
// Map {
//   "he"   => [4],
//   "she"  => [3],
//   "his"  => [1],
//   "hers" => [4]
// }
// 텍스트: a h i s h e r s
// 인덱스: 0 1 2 3 4 5 6 7
// "his" : a[h]i → a h[i]s → 인덱스 1에서 시작 (h=1, i=2, s=3)
// "she" : 인덱스 3 (s=3, h=4, e=5)
// "he"  : 인덱스 4 (h=4, e=5)
// "hers": 인덱스 4 (h=4, e=5, r=6, s=7)
```
