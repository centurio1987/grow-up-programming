# Mo's Algorithm — 구간 distinct 카운트

## 한 줄 요약

> `mosAlgorithm`은 배열과 구간 질의 목록을 받아, 각 구간에서 서로 다른 원소의 개수를 순서대로 반환한다.

## 스토리

음악 스트리밍 서비스의 분석팀은 재생 기록 배열을 갖고 있다. 각 항목은 그 순간 재생된 곡의 ID이다. 기획팀은 "특정 기간(인덱스 범위) 동안 몇 가지 다른 곡이 재생됐는가?"라는 질의를 대량으로 쏟아낸다.

질의 하나하나를 독립적으로 처리하면 너무 느리다. 분석팀은 수만 개의 질의를 한꺼번에 전달하고, 원래 질의 순서 그대로 결과를 돌려받길 원한다.

배열과 오프라인 구간 질의 목록을 받아, 각 구간 내 서로 다른 원소의 수를 원래 질의 순서에 맞게 반환하는 함수를 구현하라.

## 함수 인터페이스

```ts
export function mosAlgorithm(
  arr: number[],
  queries: [number, number][],
): number[];
```

- `arr` — 정수로 이루어진 배열 (길이 $n$)
- `queries` — 구간 질의 목록. 각 항목 `[l, r]`은 $0$-기반 폐구간 (길이 $q$)
- 반환 — 각 구간 `[l, r]`에서 서로 다른 정수의 개수를 담은 배열. 순서는 `queries`와 동일

## 제약 조건

- $1 \leq n \leq 10^4$
- $0 \leq q \leq 10^4$
- $0 \leq l \leq r < n$
- $A[i] \in \mathbb{Z}$ (음수, 0, 양수 모두 가능)
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

각 질의 $[l_i, r_i]$에 대한 답은 다음과 같다:

$$\text{result}[i] = |\{\, A[k] \mid l_i \leq k \leq r_i \,\}|$$

`queries`가 빈 배열이면 빈 배열을 반환한다.

반환 배열의 길이는 `queries`의 길이와 같아야 한다. 결과 순서는 내부에서 질의를 어떤 순서로 처리하든 관계없이 `queries`의 원래 순서를 따른다.

## 예시

```ts
const arr = [1, 2, 1, 3, 2];

mosAlgorithm(arr, [[0, 2]]);     // [2] — {1, 2}: 2가지
mosAlgorithm(arr, [[1, 4]]);     // [3] — {2, 1, 3}: 3가지
mosAlgorithm(arr, [[3, 3]]);     // [1] — {3}: 단일 원소

// 여러 질의, 입력 순서 보존
mosAlgorithm(arr, [
  [0, 4],  // {1, 2, 3} → 3
  [0, 2],  // {1, 2}   → 2
  [3, 4],  // {3, 2}   → 2
]);
// [3, 2, 2]

// 빈 질의
mosAlgorithm([1, 2, 3], []);     // []

// 모든 원소가 같은 경우
mosAlgorithm([5, 5, 5, 5], [[0, 3], [1, 2]]); // [1, 1]
```
