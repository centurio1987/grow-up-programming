# RollingHash (롤링 해시 / Rabin-Karp) 해설

## 성능 목표 예측

| 연산 | 시간복잡도 | 비고 |
|------|-----------|------|
| hash | O(m) | m = 문자열 길이 |
| search | O(n + m) 평균 | n = 텍스트 길이, 해시 충돌 없을 때 |
| search | O(nm) 최악 | 모든 위치에서 충돌 발생 시 |

공간복잡도: O(1) — 추가 자료구조 불필요 (결과 배열 제외)

---

## 목표 함수

| 메서드 | 입력 | 출력 | 설명 |
|--------|------|------|------|
| `constructor(base, mod)` | 정수 두 개 | — | 해시 파라미터 초기화 |
| `hash(s)` | 문자열 | number | 다항식 해시값 계산 |
| `search(text, pattern)` | 문자열 두 개 | number[] | Rabin-Karp 패턴 매칭 |

---

## 핵심 아이디어

### 원형 아이디어와 naive 접근

길이 n의 텍스트에서 길이 m의 패턴을 찾을 때, 모든 시작 위치 `i`에 대해 `text[i..i+m-1] == pattern`을 비교하면 O(nm) 시간이 걸린다.

### 어떤 관찰이 돌파구가 되는가

**핵심 관찰**: 두 문자열이 같으면 해시값도 같다. 즉, 해시값이 다르면 두 문자열은 반드시 다르다. 해시값 비교는 O(1)이므로, 모든 윈도우의 해시를 빠르게 계산할 수 있다면 대부분의 위치를 O(1)에 배제할 수 있다.

문제: n-m+1개 윈도우 각각의 해시를 O(m)에 계산하면 여전히 O(nm)이다.

**돌파**: 인접한 두 윈도우의 해시는 한 글자만 다르다. 이 관계를 수식으로 표현하면 O(1)에 이전 해시에서 다음 해시를 유도할 수 있다.

### 관찰을 형식화

다항식 해시:
```
H(s[i..i+m-1]) = Σ_{k=0}^{m-1} s[i+k] * base^(m-1-k)  (mod p)
```

윈도우를 오른쪽으로 한 칸 이동:
```
H(s[i+1..i+m]) = (H(s[i..i+m-1]) - s[i] * base^(m-1)) * base + s[i+m]  (mod p)
```

- 왼쪽 끝 문자 `s[i]`의 기여분을 빼고
- 전체를 `base`배 하여 자릿값 이동
- 오른쪽 끝 새 문자 `s[i+m]`을 더함

### 핵심 연산

```
search(text, pattern):
  n = text.length, m = pattern.length
  if m > n: return []

  // base^(m-1) 미리 계산 (O(m))
  highPow = 1
  for i in 1..m-1: highPow = highPow * base % mod

  // 패턴 해시와 첫 윈도우 해시 (O(m))
  patHash = hash(pattern)
  winHash = hash(text[0..m-1])

  result = []
  for i in 0 .. n-m:
    if winHash == patHash:
      if text[i..i+m-1] == pattern:  // 충돌 검증
        result.push(i)
    if i < n - m:
      // O(1) 해시 갱신
      winHash = (winHash - text[i] * highPow) % mod
      winHash = (winHash * base + text[i + m]) % mod
      winHash = (winHash + mod) % mod  // 음수 방지

  return result
```

### 정당성

`winHash == patHash`이고 실제 문자열도 같으면 등장 위치가 맞다. `winHash != patHash`이면 반드시 다른 문자열이므로 건너뛰어도 안전하다. 해시 충돌(해시가 같지만 문자열이 다른 경우)은 문자열 직접 비교로 걸러낸다.

### 구현 디테일과 최적화

- **음수 방지**: JavaScript 모듈로 연산이 음수를 반환할 수 있으므로 `(x % mod + mod) % mod` 패턴 사용
- **대형 숫자**: `base * charCode` 값이 안전 정수 범위를 넘을 수 있음. BigInt 또는 float 부동소수점 허용 범위(약 2^53)를 주의
- **이중 해시**: false positive를 줄이기 위해 두 개의 (base, mod) 쌍으로 해시를 계산
- **charCode 기반**: `s.charCodeAt(i)` 사용, 'a'를 1로 매핑하거나 그대로 사용 (일관성만 유지하면 됨)

## 수도 코드와 Activity Diagram

### 의사코드

```
class RollingHash:
  base, mod

  constructor(base=31, mod=1_000_000_007):
    this.base = base
    this.mod = mod

  hash(s):
    h = 0
    for ch in s:
      h = (h * base + charCode(ch)) % mod
    return h

  search(text, pattern):
    n = text.length, m = pattern.length
    if m == 0 or m > n: return []

    highPow = 1
    for i in 1..m-1: highPow = highPow * base % mod

    patHash = hash(pattern)
    winHash = hash(text[0..m-1])
    result = []

    for i in 0..n-m:
      if winHash == patHash:
        if text.slice(i, i+m) == pattern:
          result.push(i)
      if i < n - m:
        winHash = (winHash - charCode(text[i]) * highPow % mod + mod) % mod
        winHash = (winHash * base + charCode(text[i + m])) % mod

    return result
```
