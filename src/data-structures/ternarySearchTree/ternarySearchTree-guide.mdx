# TernarySearchTree (삼진 탐색 트리) 해설

## 성능 목표 예측

| 연산 | 시간복잡도 | 비고 |
|------|-----------|------|
| insert | O(m) | m = 단어 길이 |
| search | O(m) | |
| startsWith | O(m) | |
| delete | O(m) | |
| wordsWithPrefix | O(m + k) | k = 결과 총 문자 수 |
| size | O(1) | 카운터 유지 |

공간복잡도: O(총 문자 수) — 일반 Trie의 O(알파벳 수 × 총 문자 수)보다 훨씬 효율적

---

## 목표 함수

| 메서드 | 입력 | 출력 | 설명 |
|--------|------|------|------|
| `insert(word)` | 문자열 | void | 트리에 단어 삽입 |
| `search(word)` | 문자열 | boolean | 정확히 일치하는 단어 존재 여부 |
| `startsWith(prefix)` | 문자열 | boolean | 해당 접두사로 시작하는 단어 존재 여부 |
| `delete(word)` | 문자열 | boolean | 단어 삭제, 성공 여부 반환 |
| `wordsWithPrefix(prefix)` | 문자열 | string[] | 접두사로 시작하는 모든 단어 |
| `size()` | — | number | 저장된 단어 수 |

## 핵심 아이디어

### 원형 아이디어와 naive 접근

가장 단순한 접근은 배열에 모든 단어를 저장하고 탐색 시 순차 비교하는 것이다. 단어 수를 n, 단어 길이를 m이라 하면 O(nm) 시간이 걸려 실용적이지 않다.

일반 Trie는 각 노드마다 알파벳 크기(26)만큼의 자식 배열을 유지한다. 공통 접두사 공유로 O(m) 탐색이 가능하지만, 모든 노드가 크기 26인 배열을 갖기 때문에 희소한 단어 집합에서는 메모리 낭비가 심하다.

### 어떤 관찰이 돌파구가 되는가

Trie의 각 노드를 수평 방향으로 "이진 탐색 트리"처럼 구성할 수 있다는 관찰이다. 같은 깊이(같은 문자 위치)에서 서로 다른 문자들을 하나의 BST로 연결하면, 노드는 세 자식만 가지면 된다.

- **왼쪽 자식**: 현재 노드 문자보다 **작은** 문자 처리
- **가운데 자식**: 현재 노드 문자와 **같은** 경우 → 다음 문자 레벨로 이동
- **오른쪽 자식**: 현재 노드 문자보다 **큰** 문자 처리

### 관찰을 형식화

단어 `w[0..m-1]`를 삽입할 때, 각 위치 `i`에서 현재 노드의 `char`와 `w[i]`를 비교:

```
compare(w[i], node.char):
  < 0  → node = node.left    (w[i] 인덱스 유지)
  > 0  → node = node.right   (w[i] 인덱스 유지)
  = 0  → node = node.mid, i++ (다음 문자로 진행)
```

`i == m`(단어를 다 소비)이면 `node.isEnd = true`.

### 핵심 연산

**삽입 (재귀 버전)**:
```
insert(node, word, i):
  if i >= word.length: return node (isEnd 표시는 호출자에서 처리)
  ch = word[i]
  if node == null: node = new TSTNode(ch)
  if ch < node.char:
    node.left = insert(node.left, word, i)
  elif ch > node.char:
    node.right = insert(node.right, word, i)
  else:
    if i == word.length - 1:
      node.isEnd = true
    else:
      node.mid = insert(node.mid, word, i + 1)
  return node
```

**검색**:
```
search(node, word, i):
  if node == null: return false
  ch = word[i]
  if ch < node.char: return search(node.left, word, i)
  if ch > node.char: return search(node.right, word, i)
  // ch == node.char
  if i == word.length - 1: return node.isEnd
  return search(node.mid, word, i + 1)
```

**wordsWithPrefix**:
접두사 탐색으로 접두사의 마지막 문자에 해당하는 노드를 찾은 뒤, 그 노드부터 DFS로 모든 `isEnd` 노드까지 경로를 수집한다.

### 정당성

왼쪽/오른쪽 이동은 문자 인덱스를 소비하지 않는다. 따라서 단어의 각 문자는 반드시 가운데 링크를 통해 한 번씩만 소비된다. 총 O(m) 번의 노드 방문으로 단어를 처리할 수 있다.

### 구현 디테일과 최적화

- **중복 삽입 방지**: `isEnd`가 이미 true이면 `_size` 증가 없이 반환
- **빈 문자열 처리**: `word.length === 0`이면 루트에 `isEnd` 표시 (별도 플래그 또는 더미 노드)
- **삭제 후 노드 정리**: 단순 구현은 `isEnd = false`만 변경. 노드 제거는 선택 사항 (메모리 최적화)
- **반복(iterative) 구현**: 재귀 깊이가 O(m × log n)까지 깊어질 수 있으므로 매우 긴 단어가 있을 때는 반복 방식 선호

## 수도 코드와 Activity Diagram

### 의사코드

```
class TSTNode:
  char, left, mid, right, isEnd

class TernarySearchTree:
  root = null
  _size = 0

  insert(word):
    root = _insert(root, word, 0, isNew=ref false)
    if isNew: _size++

  _insert(node, word, i, isNew):
    if i > word.length: return node
    ch = word[i]  // 빈 문자열: i=0, length=0 처리 필요
    if node == null: node = TSTNode(ch)
    if ch < node.char:
      node.left = _insert(node.left, word, i, isNew)
    elif ch > node.char:
      node.right = _insert(node.right, word, i, isNew)
    else:
      if i == word.length - 1:
        if not node.isEnd: isNew = true
        node.isEnd = true
      else:
        node.mid = _insert(node.mid, word, i+1, isNew)
    return node

  search(word):
    return _search(root, word, 0)

  _search(node, word, i):
    if node == null: return false
    ch = word[i]
    if ch < node.char: return _search(node.left, word, i)
    if ch > node.char: return _search(node.right, word, i)
    if i == word.length - 1: return node.isEnd
    return _search(node.mid, word, i+1)

  startsWith(prefix):
    node = _getNode(root, prefix, 0)
    return node != null

  wordsWithPrefix(prefix):
    node = _getNode(root, prefix, 0)
    if node == null: return []
    result = []
    if node.isEnd: result.push(prefix)
    _collect(node.mid, prefix, result)
    return result

  _collect(node, current, result):
    if node == null: return
    _collect(node.left, current, result)
    if node.isEnd: result.push(current + node.char)
    _collect(node.mid, current + node.char, result)
    _collect(node.right, current, result)
```
