# MonotonicQueue (단조 큐)

## 한 줄 요약
> 슬라이딩 윈도우 최댓값/최솟값을 O(n)에 구하는 단조 감소/증가 덱(Deque) 자료구조

## 스토리

기상청 실시간 모니터링 시스템을 개발 중인 당신은 초당 수만 개의 기온 데이터를 처리해야 합니다.
사용자가 "최근 k분 동안의 최고 기온"과 "최근 k분 동안의 최저 기온"을 매 분마다 요청합니다.
새로운 데이터가 들어올 때마다 k개의 원소를 선형 탐색한다면 O(nk)로 n=10^5, k=10^3에서 10억 번 연산이 필요합니다.

단조 큐(Monotonic Queue)는 양쪽에서 삽입/삭제가 가능한 덱(Deque)을 이용해 윈도우 내 최댓값 후보를 관리합니다.
새 원소가 들어올 때 덱 뒤쪽에서 현재 원소보다 작은 값을 모두 제거하므로, 덱의 앞쪽에는 항상 현재 윈도우의 최댓값이 위치합니다.
윈도우가 슬라이드될 때 만료된 원소는 덱 앞쪽에서 제거합니다.

각 원소는 덱에 최대 한 번 push되고 최대 한 번 pop되므로 전체 시간복잡도는 O(n)입니다.

## 함수 인터페이스

```ts
export class MonotonicQueue {
  slidingWindowMax(nums: number[], k: number): number[]
  slidingWindowMin(nums: number[], k: number): number[]
}
```

| 메서드 | 설명 | 반환값 |
|--------|------|--------|
| `slidingWindowMax(nums, k)` | 크기 k인 슬라이딩 윈도우의 최댓값 배열 | 길이 `nums.length - k + 1` |
| `slidingWindowMin(nums, k)` | 크기 k인 슬라이딩 윈도우의 최솟값 배열 | 길이 `nums.length - k + 1` |

## 제약 조건

- `1 <= k <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`
- 결과 배열 길이는 정확히 `nums.length - k + 1`이어야 한다
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

**slidingWindowMax**
`nums` 배열의 인덱스 `0`부터 시작하여 크기 `k`인 윈도우를 오른쪽으로 한 칸씩 슬라이드하며,
각 위치에서 윈도우 내 최댓값을 구해 배열로 반환하라.

윈도우의 위치:
- 윈도우 0: `nums[0..k-1]`
- 윈도우 1: `nums[1..k]`
- ...
- 윈도우 n-k: `nums[n-k..n-1]`

**slidingWindowMin**
동일한 슬라이딩 윈도우 방식으로, 각 윈도우 내 최솟값을 반환하라.

## 예시

```ts
const mq = new MonotonicQueue();

// slidingWindowMax
mq.slidingWindowMax([1, 3, -1, -3, 5, 3, 6, 7], 3);
// => [3, 3, 5, 5, 6, 7]
// 설명:
// 윈도우 [1,3,-1]   → 최댓값 3
// 윈도우 [3,-1,-3]  → 최댓값 3
// 윈도우 [-1,-3,5]  → 최댓값 5
// 윈도우 [-3,5,3]   → 최댓값 5
// 윈도우 [5,3,6]    → 최댓값 6
// 윈도우 [3,6,7]    → 최댓값 7

// slidingWindowMin
mq.slidingWindowMin([1, 3, -1, -3, 5, 3, 6, 7], 3);
// => [-1, -3, -3, -3, 3, 3]

// 기온 모니터링 (최근 3분 최고 기온)
mq.slidingWindowMax([20, 22, 18, 25, 24, 21, 23, 19], 3);
// => [22, 25, 25, 25, 24, 23]
```
