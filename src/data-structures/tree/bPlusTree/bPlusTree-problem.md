# BPlusTree (B+ 트리)

## 한 줄 요약
> 내부 노드는 라우팅 키만, 리프 노드는 실제 데이터를 보유하며 리프끼리 연결 리스트로 이어진 B+ 트리를 구현하라.

## 스토리

데이터베이스 설계자 지수는 수천만 건의 주문 데이터에서 "2024년 1월 주문 전체 조회"와 같은 범위 질의를 빠르게 처리해야 한다. B-트리는 각 노드에 실제 데이터가 흩어져 있어서 범위 스캔 시 트리 전체를 방문해야 한다.

B+ 트리는 **실제 데이터를 오직 리프 노드에만** 저장한다. 내부 노드는 탐색을 위한 라우팅 키 역할만 한다. 그리고 리프 노드들을 연결 리스트로 연결하여 범위 질의 시 리프를 선형 순회하는 것만으로 결과를 얻을 수 있다. 범위 k개를 찾는 데 O(k + log n)이면 충분하다.

MySQL InnoDB의 클러스터드 인덱스, PostgreSQL의 인덱스, SQLite의 B-tree 모두 B+ 트리 기반이다. 현대 관계형 데이터베이스에서 가장 널리 쓰이는 인덱스 자료구조다.

## 함수 인터페이스

```ts
export class BPlusTree<T> {
  constructor(t: number = 3, comparator?: (a: T, b: T) => number)
  // 리프만 데이터 보유, 내부 노드는 라우팅 키만 저장
  // 리프끼리 연결 리스트로 연결 → 범위 질의 O(k + log n)
  insert(value: T): void
  delete(value: T): boolean
  has(value: T): boolean
  range(low: T, high: T): T[]   // O(k + log n) — low <= v <= high인 모든 값
  size(): number
  inOrder(): T[]
}
```

## 제약 조건

- $n \leq 10^4$, $t \geq 2$
- 시간 제한: 1초, 메모리 제한: 256 MB
- 중복 값은 한 번만 저장한다 (집합 의미론)
- `range(low, high)`: low 이상 high 이하인 값을 정렬된 순서로 반환
- 모든 리프는 같은 깊이에 위치해야 한다

## 문제 상세

### 노드 구조

B-트리와 달리 두 종류의 노드가 구분된다.

```
// 리프 노드: 실제 데이터 저장 + 다음 리프 포인터
BPlusLeafNode<T> {
  keys: T[]                       // 실제 데이터
  next: BPlusLeafNode<T> | undefined  // 다음 리프로의 연결
}

// 내부 노드: 라우팅 키만 저장
BPlusInternalNode<T> {
  keys: T[]                                    // 라우팅 키
  children: (BPlusInternalNode<T> | BPlusLeafNode<T>)[]
}
```

### 삽입 규칙

1. 루트부터 탐색하여 올바른 리프 노드를 찾는다.
2. 리프에 키를 정렬 삽입한다.
3. 리프가 가득 차면 (2t-1 초과) 분할:
   - 절반은 새 리프 노드로 이동
   - 새 리프의 **첫 번째 키(복사본)** 를 부모 내부 노드에 삽입 (B-트리와 다름: 올라가는 게 아니라 복사)
   - 연결 리스트 포인터 갱신
4. 내부 노드가 가득 차면 B-트리와 동일하게 분할 (중간 키는 올라가고 삭제)

### 삭제 규칙

1. 리프 노드에서 키를 찾아 삭제한다.
2. 내부 노드에 올라가 있는 라우팅 키는 삭제하지 않아도 되는 경우가 있다 (리프에 키가 남아있으면 라우팅 키는 그냥 두어도 됨).
3. 리프가 언더플로우(t-1 미만)가 되면:
   - **재분배**: 형제 리프에 t개 이상 키가 있으면 빌린다. 부모 라우팅 키를 업데이트한다.
   - **병합**: 형제 리프와 병합한다. 부모의 해당 라우팅 키를 삭제한다.
4. 내부 노드에서도 언더플로우가 발생하면 B-트리와 동일하게 처리한다.

### B-트리와 B+ 트리의 핵심 차이

| 구분 | B-트리 | B+ 트리 |
|------|--------|---------|
| 데이터 위치 | 모든 노드 | 리프만 |
| 내부 노드 | 키 + 데이터 | 라우팅 키만 |
| 리프 연결 | 없음 | 연결 리스트 |
| 범위 질의 | 트리 순회 필요 | 리프 선형 순회 O(k+log n) |
| 삭제 후 내부 노드 | 키 제거 | 라우팅 키 잔류 가능 |

## 예시

```ts
const tree = new BPlusTree<number>(3);

for (let i = 1; i <= 20; i++) {
  tree.insert(i);
}

console.log(tree.has(10));         // true
console.log(tree.has(99));         // false
console.log(tree.size());          // 20
console.log(tree.range(5, 10));    // [5, 6, 7, 8, 9, 10]
console.log(tree.inOrder());       // [1, 2, 3, ..., 20]

tree.delete(7);
console.log(tree.range(5, 10));    // [5, 6, 8, 9, 10]
```
