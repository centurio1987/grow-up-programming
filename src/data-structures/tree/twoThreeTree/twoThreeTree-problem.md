# TwoThreeTree (2-3 트리)

## 한 줄 요약
> 각 내부 노드가 1~2개의 키와 2~3개의 자식을 가지며, 항상 완전 균형을 유지하는 탐색 트리를 구현하라.

## 스토리

데이터베이스 엔지니어인 윤정은 초기 인덱스 프로토타입을 설계하고 있다. BST(이진 탐색 트리)는 최악의 경우 O(n) 높이가 되어 성능을 보장할 수 없다. AVL 트리나 레드-블랙 트리는 복잡하다. 윤정은 보다 단순한 구조로 균형을 **자동으로** 유지하는 방법이 필요하다.

2-3 트리는 각 노드에 키를 1~2개 허용함으로써 삽입/삭제 시 발생하는 불균형을 **분할(split)** 과 **병합(merge)** 을 통해 위로 전파하여 해소한다. 트리의 높이는 항상 O(log n)을 보장하고, 모든 리프는 동일한 깊이에 존재한다. 이 구조는 이후 B-트리, B+ 트리로 일반화되어 현대 데이터베이스 인덱스의 기반이 된다.

2-3 트리는 B-트리의 최소 차수 t=2인 특수 케이스다. 학습 목적에서 B-트리보다 단순하며, 균형 트리의 핵심 원리인 오버플로우 전파와 언더플로우 처리를 직접 익히기에 적합하다.

## 함수 인터페이스

```ts
export class TwoThreeTree<T> {
  constructor(comparator?: (a: T, b: T) => number)
  insert(value: T): void      // O(log n)
  delete(value: T): boolean   // O(log n)
  has(value: T): boolean      // O(log n)
  size(): number
  inOrder(): T[]              // O(n) 정렬된 배열
}
```

## 제약 조건

- $n \leq 10^4$
- 시간 제한: 1초, 메모리 제한: 256 MB
- 중복 값은 한 번만 저장한다 (집합 의미론)
- `comparator`를 제공하지 않으면 기본 비교(`<`, `>`)를 사용한다

## 문제 상세

### 노드 구조

2-3 트리의 노드는 두 가지 유형이 있다.

- **2-노드**: 키 1개, 자식 2개 (왼쪽 자식 < 키 < 오른쪽 자식)
- **3-노드**: 키 2개 (`k1 < k2`), 자식 3개 (왼쪽 < k1 < 중간 < k2 < 오른쪽)
- **리프 노드**: 자식 없음, 키 1~2개

모든 리프는 동일한 깊이(완전 균형)에 위치해야 한다.

### 삽입 규칙

1. 탐색으로 삽입 위치(리프)를 찾는다.
2. 리프에 키를 추가한다.
3. 노드에 키가 3개(오버플로우)가 되면:
   - 중간 키를 부모로 올린다.
   - 나머지 키들로 두 개의 노드를 만든다.
   - 부모에서도 오버플로우가 발생하면 위로 전파한다.
4. 루트에서 오버플로우가 발생하면 새 루트를 만들어 트리 높이가 1 증가한다.

### 삭제 규칙

1. 탐색으로 삭제할 키를 찾는다.
2. 내부 노드에 있다면 중위 후속자(in-order successor)로 대체 후 리프에서 삭제한다.
3. 리프에서 삭제 후 키가 0개(언더플로우)가 되면:
   - **재분배**: 형제 노드에 키가 2개면 부모를 통해 빌린다.
   - **병합**: 형제 노드에 키가 1개면 부모 키를 가져와 병합한다.
4. 병합으로 인해 부모에서도 언더플로우가 발생하면 위로 전파한다.
5. 루트의 키가 0개가 되면(자식이 1개) 해당 자식이 새 루트가 된다.

### 균형 조건

- 모든 리프는 같은 깊이에 있어야 한다.
- 비루트 노드는 최소 1개, 최대 2개의 키를 가진다.
- 루트는 1개 이상의 키를 가진다 (트리가 비어있지 않은 경우).

## 예시

```ts
const tree = new TwoThreeTree<number>();

tree.insert(5);
tree.insert(3);
tree.insert(7);
tree.insert(1);
tree.insert(4);

console.log(tree.has(3));     // true
console.log(tree.has(6));     // false
console.log(tree.size());     // 5
console.log(tree.inOrder());  // [1, 3, 4, 5, 7]

tree.delete(3);
console.log(tree.has(3));     // false
console.log(tree.inOrder());  // [1, 4, 5, 7]
```
