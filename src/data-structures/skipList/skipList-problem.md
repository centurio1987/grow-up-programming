# Skip List (스킵 리스트)

## 한 줄 요약
> 확률적 다단계 연결 리스트로 O(log n) 기대 시간에 삽입·삭제·검색을 지원하는 정수 키 저장소를 구현하라.

## 스토리

대규모 온라인 게임 회사에서 수백만 명의 유저 점수를 실시간으로 관리하는 리더보드 시스템을 구축해야 한다. 매 초 수천 건의 점수 갱신(삽입/삭제)이 발생하고, 동시에 특정 유저의 순위 조회(검색)도 끊임없이 들어온다.

처음에는 정렬 배열로 구현했지만 삽입 시 O(n) 이동 비용이 병목이 되었다. 레드블랙 트리나 AVL 트리는 O(log n)을 보장하지만 회전 로직이 복잡해 구현 실수가 잦았고, 동시성 환경에서 락 경합도 심각했다. 팀의 시니어 엔지니어가 스킵 리스트를 제안한다.

스킵 리스트는 여러 레벨의 연결 리스트를 계층적으로 쌓은 자료구조다. 새 키를 삽입할 때 동전을 던지듯 확률(p=0.5)로 상위 레벨에 노드를 추가한다. 이 단순한 규칙만으로 균형 트리에 준하는 O(log n) 기대 복잡도를 얻으면서도, 구현이 훨씬 간결하고 lock-free 확장도 자연스럽다. 이제 리더보드의 심장부가 될 스킵 리스트를 완성하라.

## 함수 인터페이스

```ts
export class SkipList {
  insert(key: number): void    // O(log n) 기대; 중복 키는 무시
  search(key: number): boolean // O(log n) 기대
  delete(key: number): void    // O(log n) 기대; 없는 키는 무시
  toArray(): number[]          // 레벨 0 연결 리스트를 오름차순 배열로 반환, O(n)
}
// MAX_LEVEL = 16, 승격 확률 p = 0.5
```

| 메서드 | 설명 | 반환 |
|--------|------|------|
| `insert(key)` | 키를 스킵 리스트에 삽입. 이미 존재하면 무시 | `void` |
| `search(key)` | 키 존재 여부 확인 | `boolean` |
| `delete(key)` | 키 제거. 없으면 무시 | `void` |
| `toArray()` | 저장된 모든 키를 오름차순으로 반환 | `number[]` |

## 제약 조건

- 키는 정수 범위 $-2^{31} \le key \le 2^{31}-1$
- 최대 레벨: $\text{MAX\_LEVEL} = 16$
- 승격 확률: $p = 0.5$
- 저장 가능한 키의 개수: $n \le 10^6$
- 시간복잡도 기대값
  - `insert` / `search` / `delete`: $O(\log n)$
  - `toArray`: $O(n)$

## 문제 상세

스킵 리스트는 맨 아래(레벨 0)에 모든 키를 정렬 순서로 보관하는 연결 리스트를 두고, 위쪽 레벨들은 아래 레벨의 부분 집합을 "고속 도로"처럼 연결한다.

**헤드 노드**: 실제 키를 갖지 않는 파수꾼(sentinel) 노드로 모든 레벨의 시작점이 된다. 키 값은 음의 무한대로 취급한다.

**레벨 결정 (`randomLevel`)**: 새 노드를 삽입할 때 1부터 시작해 확률 p=0.5로 레벨을 하나씩 올린다. MAX_LEVEL을 초과하면 중단한다.

**삽입 (`insert`)**:
1. 현재 최고 레벨에서 시작해 삽입 위치 직전 노드들을 `update[]` 배열에 기록하며 내려온다.
2. 레벨 0에서 키 중복을 확인하고, 있으면 무시한다.
3. `randomLevel()`로 새 노드의 레벨을 정한다.
4. 각 레벨에서 `update[i]` 뒤에 새 노드를 삽입한다.

**검색 (`search`)**:
1. 현재 최고 레벨에서 시작해 `next[level].key < key`이면 앞으로 이동한다.
2. 레벨 0에 도달했을 때 다음 노드의 키가 `key`이면 `true`, 아니면 `false`.

**삭제 (`delete`)**:
1. `insert`와 동일하게 `update[]` 배열을 구성한다.
2. 레벨 0에서 해당 키가 존재하는지 확인하고, 없으면 무시한다.
3. 각 레벨에서 `update[i].next[i]`가 삭제 대상이면 포인터를 건너뛴다.
4. 최고 레벨이 빈 경우 `currentLevel`을 줄인다.

## 예시

```ts
const sl = new SkipList();

sl.insert(3);
sl.insert(1);
sl.insert(5);
sl.insert(3); // 중복 무시

console.log(sl.toArray());   // [1, 3, 5]
console.log(sl.search(3));   // true
console.log(sl.search(2));   // false

sl.delete(1);
console.log(sl.toArray());   // [3, 5]

sl.delete(99); // 없는 키 무시
console.log(sl.toArray());   // [3, 5]
```
