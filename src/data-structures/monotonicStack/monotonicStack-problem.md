# MonotonicStack (단조 스택)

## 한 줄 요약
> 배열의 각 원소에 대해 "다음/이전 더 큰·작은 원소 인덱스"를 O(n)에 한 번에 구하는 자료구조

## 스토리

주식 트레이딩 플랫폼을 운영 중인 당신은 수십만 개의 주가 기록을 분석해야 합니다.
사용자들은 "오늘 이후 처음으로 오늘보다 비싼 날이 언제인가?"를 실시간으로 묻습니다.
O(n²)의 이중 반복으로는 수십만 개 데이터를 처리할 수 없습니다.

단조 스택(Monotonic Stack)은 스택이 항상 단조 증가 또는 단조 감소 상태를 유지하도록 원소를 관리합니다.
새로운 원소가 스택 상단보다 클 때 스택에서 원소를 꺼내는 순간, 꺼내진 원소의 "다음 더 큰 원소"가 결정됩니다.
이 과정에서 각 원소는 스택에 최대 한 번 push되고 한 번 pop되므로, 전체 시간복잡도는 O(n)입니다.

하나의 배열을 왼쪽→오른쪽, 또는 오른쪽→왼쪽으로 순회하면서 스택 불변식을 유지함으로써
nextGreater, prevGreater, nextSmaller 세 가지 질의를 모두 O(n)에 해결할 수 있습니다.

## 함수 인터페이스

```ts
export class MonotonicStack {
  nextGreater(arr: number[]): number[]
  prevGreater(arr: number[]): number[]
  nextSmaller(arr: number[]): number[]
}
```

| 메서드 | 설명 | 반환값 |
|--------|------|--------|
| `nextGreater(arr)` | 각 인덱스 i에 대해 i보다 오른쪽에 있는 arr[j] > arr[i]인 최소 j | 없으면 -1 |
| `prevGreater(arr)` | 각 인덱스 i에 대해 i보다 왼쪽에 있는 arr[j] > arr[i]인 최대 j | 없으면 -1 |
| `nextSmaller(arr)` | 각 인덱스 i에 대해 i보다 오른쪽에 있는 arr[j] < arr[i]인 최소 j | 없으면 -1 |

## 제약 조건

- `0 <= arr.length <= 10^5`
- `-10^9 <= arr[i] <= 10^9`
- 동일한 값은 "더 크다/작다" 조건을 만족하지 않는다 (strict inequality)
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

**nextGreater**
배열 `arr`에서 각 인덱스 `i`에 대해, `i`의 오른쪽(`j > i`)에 존재하는 원소 중
`arr[j] > arr[i]`를 만족하는 가장 작은 인덱스 `j`를 구하라.
만약 그런 `j`가 없다면 -1을 반환한다.

**prevGreater**
배열 `arr`에서 각 인덱스 `i`에 대해, `i`의 왼쪽(`j < i`)에 존재하는 원소 중
`arr[j] > arr[i]`를 만족하는 가장 큰 인덱스 `j`를 구하라.
만약 그런 `j`가 없다면 -1을 반환한다.

**nextSmaller**
배열 `arr`에서 각 인덱스 `i`에 대해, `i`의 오른쪽(`j > i`)에 존재하는 원소 중
`arr[j] < arr[i]`를 만족하는 가장 작은 인덱스 `j`를 구하라.
만약 그런 `j`가 없다면 -1을 반환한다.

## 예시

```ts
const ms = new MonotonicStack();

// nextGreater
ms.nextGreater([2, 1, 2, 4, 3]);
// => [3, 2, 3, -1, -1]
// 설명: arr[0]=2의 다음 더 큰 원소는 arr[3]=4 (인덱스 3)
//       arr[1]=1의 다음 더 큰 원소는 arr[2]=2 (인덱스 2)
//       arr[4]=3 의 오른쪽에 더 큰 원소 없음 -> -1

// prevGreater
ms.prevGreater([2, 1, 2, 4, 3]);
// => [-1, 0, -1, -1, 3]
// 설명: arr[1]=1의 이전 더 큰 원소는 arr[0]=2 (인덱스 0)
//       arr[4]=3의 이전 더 큰 원소는 arr[3]=4 (인덱스 3)

// nextSmaller
ms.nextSmaller([3, 1, 2, 4, 0]);
// => [1, 4, 4, 4, -1]
// 설명: arr[0]=3의 다음 더 작은 원소는 arr[1]=1 (인덱스 1)
//       arr[1]=1의 다음 더 작은 원소는 arr[4]=0 (인덱스 4)

// 주식 가격 시나리오
ms.nextGreater([73, 74, 75, 71, 69, 72, 76, 73]);
// => [2, 2, 6, 5, 5, 6, -1, -1]
// 설명: 73원짜리 주식(인덱스 0)보다 처음으로 비싼 날은 인덱스 2(75원)
```
