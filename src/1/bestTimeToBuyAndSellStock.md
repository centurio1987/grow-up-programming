# bestTimeToBuyAndSellStock 분석 보고서

## 1. 접근 방법의 방향성

문제는 $0 \leq i \leq j < N$에서 $prices[j] - prices[i]$의 최댓값을 구하는 것이다. 단순한 이중 루프는 $O(N^2)$이지만, 본 솔루션은 좌측 최솟값 추적을 통한 1-pass 알고리즘으로 $O(N)$에 해결한다.

핵심 관찰:

$$
\max_{0 \leq i \leq j < N}(prices[j] - prices[i]) = \max_{0 \leq j < N}\bigg(prices[j] - \min_{0 \leq i \leq j} prices[i]\bigg)
$$

즉, 각 매도일 $j$를 고정하면 최적의 매수일은 "$j$까지의 최저가가 발생한 날"이다. 따라서 배열을 한 번 훑으면서 (a) 현재까지의 최저 매수가 $minP$와 (b) 현재까지의 최대 이익 $maxPrice$를 동시에 갱신하면 된다.

방향성은 정확하다. 표준적이고 가장 일반적으로 권장되는 1-pass 솔루션이다.

## 2. 정확성 분석

`bun test src/1/bestTimeToBuyAndSellStock.test.ts` 실행 결과:

```
 12 pass
 0 fail
 13 expect() calls
Ran 12 tests across 1 file. [49.00ms]
```

검증된 케이스:

- **기본**: `[7,1,5,3,6,4] → 5`, `[1,2,3,4,5] → 4`, `[2,4,1,7] → 6`
- **엣지**: 내림차순 `[7,6,4,3,1] → 0`, 모두 같음 `[5,5,5,5] → 0`, 최저가가 마지막 `[5,4,3,2,1] → 0`, 0 포함 `[0,10] → 10`
- **바운더리**: `N=1 → 0`, `N=2` 상승/하락, 최댓값 차이 `[0,10000] → 10000`
- **성능**: $N = 100{,}000$ 단조증가 배열 100ms 이내 통과

`if (_prices.length < 2) return 0` 가드로 $N=1$ 케이스와 빈 배열을 안전하게 처리한다. 거래 불가능 시 0을 반환하는 사양도 `maxPrice = 0` 초기값으로 자연스럽게 충족한다.

`else if` 구조의 정합성:
- `_prices[i] < minP`: 새 최저가이므로 갱신만 한다. 이 시점에서의 이익은 음수이므로 갱신 후 즉시 다음 반복으로 가도 무방하다.
- `_prices[i] >= minP`: 매도 후보로서 `_prices[i] - minP`가 기존 `maxPrice`보다 크면 갱신한다.

`_prices[i] == minP`인 경우 `else` 분기로 들어가지만 이익 계산값이 0이고 `maxPrice >= 0`이므로 갱신이 일어나지 않아 정확성에 문제가 없다.

## 3. 최적화 분석

### 3.1. 시간/공간 복잡도

- **시간복잡도**: $O(N)$ — 배열을 정확히 한 번 순회한다.
- **공간복잡도**: $O(1)$ — 보조 변수 `minP`, `maxPrice`만 사용한다.

이 문제의 하한은 $\Omega(N)$이다(모든 입력 원소를 최소 한 번은 읽어야 한다). 따라서 점근적으로 더 줄일 여지는 없다.

### 3.2. 상수 인자 측면

현재 구현은 이미 다음과 같이 잘 다듬어져 있다:

1. **1-pass**: 한 번의 루프로 최저가와 최대이익을 동시에 계산한다.
2. **`else if`**: 새 최저가 갱신 시 이익 갱신을 건너뛰어 분기를 절약한다.
3. **인덱스 직접 접근**: `_prices[i]!`로 비-널 단언을 사용해 `noUncheckedIndexedAccess`의 오버헤드를 회피한다.

추가 미세 최적화 후보:

- `for...of` 대신 인덱스 루프를 사용하고 있어 V8/JSC의 hidden class 측면에서 이미 빠른 편이다.
- `_prices.length`를 지역 변수에 캐시하면 미세하게 빠를 수 있으나 V8 JIT이 자동으로 최적화하므로 실측 차이는 무시할 수준이다.

### 3.3. 알고리즘 원형 — Kadane's Algorithm으로부터의 유도

본 솔루션은 **Kadane's algorithm**(최대 부분합)의 변형으로 볼 수 있다. 원형에서 어떻게 유도되는지 단계별로 기술한다.

**원형: Kadane's Algorithm**

배열 $A$가 주어졌을 때 연속 부분배열의 최대합을 $O(N)$에 구한다:

$$
\text{maxSubarraySum}(A) = \max_{0 \leq l \leq r < N} \sum_{k=l}^{r} A[k]
$$

상태 변수 $f(j)$를 "$j$에서 끝나는 부분배열의 최대합"으로 정의하면:

$$
f(j) = \max(A[j],\ f(j-1) + A[j])
$$

답은 $\max_j f(j)$이다.

**유도 1단계: 차분 배열로의 변환**

주식 문제의 이익 $prices[j] - prices[i]$ ($i \leq j$)는 다음과 같이 차분의 합으로 표현된다:

$$
prices[j] - prices[i] = \sum_{k=i+1}^{j} (prices[k] - prices[k-1])
$$

차분 배열 $D[k] = prices[k] - prices[k-1]$ ($1 \leq k < N$)을 정의하면, 최대 이익은 다음과 같다:

$$
\text{maxProfit}(prices) = \max\bigg(0,\ \max_{1 \leq l \leq r < N} \sum_{k=l}^{r} D[k]\bigg)
$$

즉, **차분 배열 $D$에 대한 최대 부분합 문제**로 환원된다. 거래하지 않는 선택지가 있으므로 $0$과의 최댓값을 취한다.

**유도 2단계: Kadane 적용**

$D$에 Kadane을 적용하면:

$$
f(j) = \max(D[j],\ f(j-1) + D[j])
$$

여기서 $f(j)$는 "$j$일에 매도했을 때, 적절한 매수일을 선택한 이익"으로 해석된다.

**유도 3단계: 최저 매수가 추적 형태로의 재해석**

$f(j) = prices[j] - prices[i^*]$ 꼴로 풀어 쓰면, $i^*$는 "$j$까지의 최저가가 발생한 인덱스"가 된다. 따라서:

$$
f(j) = prices[j] - \min_{0 \leq i \leq j} prices[i]
$$

이 형태가 본 솔루션이 직접 구현하는 점화식이다. 변수 $minP$가 $\min_{0 \leq i \leq j} prices[i]$를, $maxPrice$가 $\max_j f(j)$를 추적한다.

**정리**

Kadane은 일반적인 부분합 문제이고, 주식 문제는 그 차분 형태이다. 차분의 부분합 = 양 끝 값의 차라는 망원합(telescoping) 성질 덕분에, 차분 배열을 명시적으로 만들 필요 없이 "최저가 추적" 형태로 단순화된다.

## 4. 결론

- 정확성: 12개 테스트 모두 통과. 가드, `else if` 분기, 초기값 모두 정확하게 처리된다.
- 최적화: 시간 $O(N)$, 공간 $O(1)$로 점근 최적. 상수 인자 측면에서도 일반적인 1-pass 솔루션의 모범적 형태이다.
- 개선 여지: 알고리즘 수준의 개선은 없다. 코드 가독성 측면에서 `_prices` 파라미터 이름의 언더스코어 접두사가 "미사용" 관습과 충돌하므로 `prices`로 두는 편이 명료할 수 있다(미세 사항).
