# editDistance — 편집 거리 (Levenshtein Distance)

## 성능 목표 예측

| 항목 | 값 |
|------|-----|
| 문자열 길이 | $0 \leq n = |s| \leq 1{,}000$, $0 \leq m = |t| \leq 1{,}000$ |
| 연산 비용 | 삽입, 삭제, 교체 각 $1$ |

**naive 접근의 문제점**: 순수 재귀로 $s$를 $t$로 변환하는 모든 편집 경로를 탐색하면, 중복 부분 문제가 지수적으로 발생한다. 같은 $(i, j)$ 쌍이 수없이 재계산되어 최악 $O(3^{\max(n,m)})$이 된다.

**목표 복잡도**: 시간 $O(nm)$, 공간 $O(nm)$ 또는 롤링으로 $O(\min(n,m))$. $n \cdot m = 10^6$이므로 충분하다.

**공간 복잡도**: 2D DP 테이블은 $O(nm)$. 점화식이 이전 행만 참조하므로 행 롤링으로 $O(m)$까지 줄일 수 있다.

---

## 목표 함수

```ts
function editDistance(s: string, t: string): number
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `s` | 시작 문자열 | $0 \leq |s| \leq 1{,}000$ |
| `t` | 목표 문자열 | $0 \leq |t| \leq 1{,}000$ |

**반환값**: $s$를 $t$로 변환하는 데 필요한 최소 편집 횟수 (삽입/삭제/교체 각 비용 $1$).

**엣지케이스**:

| 입력 | 기대 출력 | 이유 |
|------|-----------|------|
| `s=""`, `t="abc"` | `3` | 빈 문자열에서 3번 삽입 |
| `s="abc"`, `t=""` | `3` | 3번 삭제 |
| `s="abc"`, `t="abc"` | `0` | 동일 문자열 |
| `s="a"`, `t="b"` | `1` | 교체 1번 |

## 핵심 아이디어

### 원형 아이디어와 naive 접근

$s$를 $t$로 바꾸는 문제를 재귀로 분해한다. $s[0..i-1]$과 $t[0..j-1]$ 사이의 편집 거리를 $f(i, j)$라 하면:

```
f(i, j):
    if i == 0: return j   // t의 j개 문자를 모두 삽입
    if j == 0: return i   // s의 i개 문자를 모두 삭제
    if s[i-1] == t[j-1]: return f(i-1, j-1)
    return 1 + min(f(i-1, j), f(i, j-1), f(i-1, j-1))
```

이 재귀는 $(n+1)(m+1)$개의 고유한 부분 문제를 갖지만, 메모이제이션 없이는 같은 $(i, j)$를 지수적으로 재계산한다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1**: $f(i, j)$의 값은 $f(i-1, j)$, $f(i, j-1)$, $f(i-1, j-1)$ 세 값에만 의존한다 — 중복 부분 문제가 명확하다.
- **관찰 2**: 부분 문제의 수가 $(n+1)(m+1) = O(nm)$으로 다항식이다. 메모이제이션(하향식) 또는 테이블 채우기(상향식)로 $O(nm)$에 해결 가능하다.
- **관찰 3**: 마지막 연산을 기준으로 세 가지 경우만 고려하면 된다 — 삭제, 삽입, 교체. 이 세 가지가 완전하고 중복되지 않으므로 최적 부분 구조가 성립한다.

### 관찰을 형식화: 상태/구조 정의

상태 $dp[i][j]$를 다음과 같이 정의한다.

$$dp[i][j] = s[0..i-1] \text{을 } t[0..j-1] \text{로 변환하는 최소 편집 횟수}$$

이 정의가 왜 이 형태여야 하는가: "접두어 쌍"을 상태로 잡으면 부분 문제들이 방향성 있는 순서(왼쪽 위 → 오른쪽 아래)로 정렬되어 순환 의존성이 없다. "임의의 부분 문자열"로 확장하면 상태 수가 $O(n^2 m^2)$으로 폭발한다.

초기 조건:

$$dp[0][j] = j \quad (0 \leq j \leq m)$$
$$dp[i][0] = i \quad (0 \leq i \leq n)$$

- $dp[0][j] = j$: 빈 $s$에서 $t[0..j-1]$을 만들려면 $j$번 삽입해야 한다.
- $dp[i][0] = i$: $s[0..i-1]$을 빈 $t$로 만들려면 $i$번 삭제해야 한다.

### 점화식 또는 핵심 연산

$1 \leq i \leq n$, $1 \leq j \leq m$에 대해:

$$dp[i][j] = \begin{cases} dp[i-1][j-1] & \text{if } s[i-1] = t[j-1] \\ 1 + \min\bigl(dp[i-1][j],\; dp[i][j-1],\; dp[i-1][j-1]\bigr) & \text{otherwise} \end{cases}$$

각 항의 의미:
- $dp[i-1][j-1]$ (문자 일치): $s[i-1]$과 $t[j-1]$이 같으므로 추가 비용 없이 이전 상태에서 이어받는다.
- $dp[i-1][j] + 1$ (삭제): $s[i-1]$을 삭제하고 $s[0..i-2]$를 $t[0..j-1]$로 변환한다.
- $dp[i][j-1] + 1$ (삽입): $t[j-1]$을 삽입해 $s[0..i-1]$을 $t[0..j-2]$로 변환한 결과에 이어 붙인다.
- $dp[i-1][j-1] + 1$ (교체): $s[i-1]$을 $t[j-1]$로 교체하고 이전 상태에서 이어받는다.

### 정당성 — 왜 이것이 옳은가

최적 편집 시퀀스에서 마지막 연산은 반드시 삭제, 삽입, 교체, 또는 일치(비용 없음) 중 하나다. 각 경우 나머지 부분은 더 작은 부분 문제가 되고, 귀납적으로 $dp$ 값이 최적임이 보장된다.

문자 일치 케이스에서 "교체 비용 0"으로 처리하는 이유: 두 문자가 같을 때 교체를 하면 비용이 $1 + dp[i-1][j-1]$이 되어 $dp[i-1][j-1]$보다 크므로, $\min$ 연산에서 항상 선택되지 않는다. 따라서 일치 케이스를 별도로 처리하는 것이 효율적이다.

빈 문자열 케이스: $dp[0][j] = j$, $dp[i][0] = i$는 점화식의 경계 조건으로, 귀납의 기저(base case)이다.

### 구현 디테일과 최적화

- **공간 최적화**: 점화식이 $dp[i-1][*]$ 행과 $dp[i][j-1]$ 셀만 참조한다. 따라서 이전 행 하나만 유지하면 $O(m)$ 공간으로 해결 가능하다. 단, $dp[i-1][j-1]$ 값은 갱신 전에 미리 저장해야 한다.
- **루프 순서**: $i$ (행)가 바깥, $j$ (열)가 안쪽으로 순회해야 한다. $dp[i][j]$가 $dp[i][j-1]$을 참조하므로 $j$가 왼쪽에서 오른쪽으로 증가해야 한다.
- **함정**: $dp[0][*]$ 초기화를 빠뜨리면 $j=1$인 셀에서 $dp[0][j-1]$이 초기화되지 않아 잘못된 결과가 나온다.
- **함정**: 인덱스 오프셋 — $s[i-1]$과 $t[j-1]$이 문자 인덱스이고, $dp$ 인덱스는 1-based임을 혼동하지 않아야 한다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function editDistance(s, t):
    n ← len(s),  m ← len(t)
    dp ← (n+1) × (m+1) 크기의 행렬    // 불변식: dp[i][j] = s[0..i-1] → t[0..j-1] 최소 편집 수

    for i from 0 to n: dp[i][0] ← i   // 불변식: i번 삭제
    for j from 0 to m: dp[0][j] ← j   // 불변식: j번 삽입

    for i from 1 to n:
        for j from 1 to m:
            if s[i-1] == t[j-1]:
                dp[i][j] ← dp[i-1][j-1]                              // 일치: 비용 0
            else:
                dp[i][j] ← 1 + min(dp[i-1][j],   // 삭제
                                    dp[i][j-1],   // 삽입
                                    dp[i-1][j-1]) // 교체

    return dp[n][m]
```

**핵심 불변식**: $dp[i][j]$ 셀이 채워진 시점에 $dp[i][j] = s[0..i-1]$을 $t[0..j-1]$로 변환하는 최소 편집 횟수이며, $dp[i][j] \leq \max(i, j)$ (전부 삽입 또는 삭제)가 항상 성립한다.
