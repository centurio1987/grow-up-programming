# houseRobber — 인접 불가 최대합

## 성능 목표 예측

| 항목 | 값 |
|------|-----|
| 입력 크기 | $0 \leq N \leq 100{,}000$ |
| 원소 범위 | $0 \leq nums[i] \leq 10{,}000$ |

**naive 접근의 문제점**: 인접 제약을 지키는 모든 부분집합을 열거하면 $O(2^N)$이다. $N = 10^5$에서는 불가능하다.

**목표 복잡도**: 시간 $O(N)$, 공간 $O(1)$. 선형 DP 후 롤링 변수로 공간을 줄인다.

**공간 복잡도**: DP 배열 전체를 저장하면 $O(N)$이지만, 점화식이 이전 두 값만 참조하므로 변수 두 개로 $O(1)$이 가능하다.

---

## 목표 함수

```ts
function houseRobber(nums: number[]): number
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `nums` | 각 집의 현금 배열 | $0 \leq N \leq 100{,}000$, $0 \leq nums[i] \leq 10{,}000$ |

**반환값**: 인접한 집을 동시에 털지 않으면서 얻을 수 있는 최대 금액. $N = 0$이면 $0$.

**엣지케이스**:

| 입력 | 기대 출력 | 이유 |
|------|-----------|------|
| `[]` | `0` | 집 없음 |
| `[5]` | `5` | 집 하나 |
| `[1, 2]` | `2` | 둘 중 하나만 선택 가능 |
| `[0, 0, 0]` | `0` | 모든 집이 빈집 |

## 핵심 아이디어

### 원형 아이디어와 naive 접근

인접 제약을 지키는 모든 부분집합을 열거해 합이 최대인 것을 찾는다.

```
best = 0
for each subset S of {0, 1, ..., N-1}:
    if S에 인접한 두 원소가 없으면:
        best = max(best, sum(nums[i] for i in S))
```

부분집합의 수가 $2^N$이므로 $N = 100$에서도 우주의 나이를 초과한다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1**: $i$번째 집을 선택할지 여부는 $i-1$번째 집의 선택 여부에만 의존한다. 이전 모든 선택 내역을 알 필요가 없다 — 최적 부분 구조가 성립한다.
- **관찰 2**: $i$번째 집에서의 최적 결정은 "집 $i$를 털기($i-2$까지의 최적합 + $nums[i]$)"와 "집 $i$를 건너뛰기($i-1$까지의 최적합)" 중 큰 값이다. 이 두 경우가 완전하다.
- **관찰 3**: $dp[i]$는 $dp[i-1]$과 $dp[i-2]$만 참조하므로, 배열 전체를 저장하지 않고 변수 두 개로 $O(1)$ 공간에서 처리할 수 있다.

### 관찰을 형식화: 상태/구조 정의

상태 $dp[i]$를 다음과 같이 정의한다.

$$dp[i] = \text{집 } 0, 1, \ldots, i \text{ 중 인접 제약을 지키면서 얻을 수 있는 최대 금액}$$

초기 조건:

$$dp[0] = nums[0]$$
$$dp[1] = \max(nums[0], nums[1])$$

이 정의가 왜 이 형태여야 하는가: "집 $0 \ldots i$의 최적합"으로 정의하면 부분 문제들이 크기 순으로 정렬되어 순환 의존성이 없다. "$i$에서 끝나는 최대합"으로 정의하면 $i$를 반드시 선택해야 하는 제약이 생겨 $i$를 건너뛰는 최적해를 놓친다.

### 점화식 또는 핵심 연산

$i = 2, 3, \ldots, N-1$에 대해:

$$dp[i] = \max\bigl(dp[i-1],\; dp[i-2] + nums[i]\bigr)$$

결과: $dp[N-1]$

- $dp[i-1]$: 집 $i$를 건너뛰고 $i-1$까지의 최적합을 그대로 유지
- $dp[i-2] + nums[i]$: 집 $i$를 털고, 인접 제약에 의해 $i-1$은 건너뛰었으므로 $i-2$까지의 최적합에 더함

### 정당성 — 왜 이것이 옳은가

귀납적으로 증명한다. 기저: $dp[0] = nums[0]$ (집이 하나이면 무조건 선택), $dp[1] = \max(nums[0], nums[1])$ (두 집 중 하나 선택).

귀납 가정: $dp[0], \ldots, dp[i-1]$이 각각 해당 범위의 최적합이라고 가정한다. 집 $i$에 대해 최적 선택은:
- 집 $i$를 선택: $i-1$은 제외되므로 $dp[i-2] + nums[i]$
- 집 $i$를 미선택: $dp[i-1]$

이 두 경우 외에 다른 가능성은 없으므로 $dp[i] = \max(dp[i-1], dp[i-2] + nums[i])$가 최적이다.

음수 원소가 없으므로 ($nums[i] \geq 0$) $dp$ 값은 단조 비감소한다. 즉, $dp[i] \geq dp[i-1]$이 항상 성립한다. 따라서 "아무것도 안 털기"가 절대 최적이 되지 않으며 $dp[N-1] \geq 0$이 보장된다.

### 구현 디테일과 최적화

- 롤링 변수: `prev2 = dp[i-2]`, `prev1 = dp[i-1]`만 유지한다. 루프 후 `prev2 ← prev1`, `prev1 ← curr`으로 갱신한다.
- **함정**: `prev2`와 `prev1` 갱신 순서가 중요하다. `prev1`을 먼저 `prev2`에 대입한 뒤 `curr`을 `prev1`에 대입해야 한다. 순서를 바꾸면 `prev2`가 `curr`을 가리켜 두 단계 전이 아닌 한 단계 전을 참조하는 오류가 발생한다.
- $N = 0$: 빈 배열 즉시 $0$ 반환.
- $N = 1$: 루프 진입 없이 `prev1 = dp[0] = nums[0]` 반환.
- $N = 2$: 루프가 한 번 돌아 `prev1 = max(nums[0], nums[1])`을 확인하고 반환.

## 수도 코드와 Activity Diagram

### 의사코드

```
function houseRobber(nums):
    if len(nums) == 0: return 0
    if len(nums) == 1: return nums[0]

    prev2 ← nums[0]                       // 불변식: dp[i-2]
    prev1 ← max(nums[0], nums[1])         // 불변식: dp[i-1]

    for i from 2 to N-1:
        curr  ← max(prev1, prev2 + nums[i])   // dp[i]
        prev2 ← prev1                          // 슬라이딩: i-2 → i-1
        prev1 ← curr                           // 슬라이딩: i-1 → i

    return prev1                               // dp[N-1]
```

### Activity Diagram

```mermaid
flowchart TD
    A([시작]) --> B{N == 0?}
    B -- Yes --> C([return 0])
    B -- No --> D{N == 1?}
    D -- Yes --> E([return nums[0]])
    D -- No --> F["prev2 = nums[0] / prev1 = max(nums[0], nums[1])"]
    F --> G{i < N?}
    G -- Yes --> H["curr = max(prev1, prev2 + nums[i])"]
    H --> I["prev2 = prev1 / prev1 = curr"]
    I --> J[i++]
    J --> G
    G -- No --> K([return prev1])
```

**핵심 불변식**: 루프 변수 $i$ 진입 시점에 `prev2` $= dp[i-2]$, `prev1` $= dp[i-1]$이며, 루프 종료 후 `prev1` $= dp[N-1]$이 전체 최적합이다.

## 복잡도 분석 심화

| 접근 방식 | 시간 | 공간 | 비고 |
|-----------|------|------|------|
| 완전 탐색 (2^N 부분집합) | $O(2^N)$ | $O(1)$ | 불가 |
| DP 배열 전체 저장 | $O(N)$ | $O(N)$ | 가능하지만 낭비 |
| 롤링 변수 2개 | $O(N)$ | $O(1)$ | 최적 |

**변형 1 — 원형 집 배열**: 첫 번째 집과 마지막 집이 인접한 경우. 두 번 분리 실행한다: (a) 집 $0$을 포함하고 집 $N-1$을 제외, (b) 집 $0$을 제외하고 집 $N-1$을 포함. 두 결과의 최댓값이 답이다.

**변형 2 — 인접 3채 불가**: 인접한 두 집 뿐 아니라 세 집 연속 선택도 불가한 경우. 상태를 "$i-1$과 $i-2$의 선택 여부"로 확장해 $O(N)$ DP를 적용한다.

**변형 3 — 일반 그래프에서 최대 독립 집합**: House Robber는 경로 그래프에서 최대 독립 집합 문제이다. 트리로 확장하면 루트에서 DFS DP ($O(N)$)로 해결 가능하다. 일반 그래프에서는 NP-hard이다.

**수치 예시로 귀납 확인**:

$nums = [2, 7, 9, 3, 1]$에서:

| $i$ | `nums[i]` | `prev2` | `prev1` | `curr` |
|-----|-----------|---------|---------|--------|
| 0 | 2 | — | — | init: `prev2=2, prev1=max(2,7)=7` |
| 2 | 9 | 2 | 7 | `max(7, 2+9)=11` → `prev2=7, prev1=11` |
| 3 | 3 | 7 | 11 | `max(11, 7+3)=11` → `prev2=11, prev1=11` |
| 4 | 1 | 11 | 11 | `max(11, 11+1)=12` → `prev2=11, prev1=12` |

결과: $12$ (집 $0, 2, 4$를 선택: $2+9+1=12$).
