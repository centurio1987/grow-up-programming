# knapsack01 — 0/1 배낭 문제 해설

## 성능 목표 예측

| 제약 | 값 |
|------|-----|
| 물건 수 $n$ | $1 \leq n \leq 100$ |
| 배낭 용량 $W$ | $1 \leq W \leq 10^4$ |
| 물건 무게 $w_i$ | $1 \leq w_i \leq 10^4$ |
| 물건 가치 $v_i$ | $1 \leq v_i \leq 10^4$ |

**Naive 접근의 복잡도 분석**

각 물건을 담거나 담지 않는 두 가지 선택이 있으므로, 완전 탐색의 경우의 수는 $2^n$이다. $n = 100$이면 $2^{100} \approx 10^{30}$가지로, 현재까지 알려진 어떤 컴퓨터도 이를 열거할 수 없다. 이 폭발적 증가가 naive 접근의 핵심 문제다.

메모이제이션이 없는 재귀조차도 동일한 `(물건 인덱스, 남은 용량)` 쌍이 여러 경로에서 반복 등장해 지수 시간이 된다.

**목표 복잡도**

$$O(n \cdot W) = O(100 \times 10^4) = O(10^6)$$

상태 공간 크기가 $n \times W$이고 각 셀을 $O(1)$에 채우므로 목표 복잡도를 달성한다.

**공간 복잡도**

- 2D 테이블 $O(n \times W)$는 $100 \times 10^4 = 10^6$ 원소 (약 4MB for int32) — 허용 가능
- 롤링 배열로 $O(W) = O(10^4)$ 원소까지 줄일 수 있다.

## 목표 함수

```ts
function knapsack01(weights: number[], values: number[], W: number): number
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `weights` | 각 물건의 무게 배열 | 길이 $n$, $1 \leq w_i \leq 10^4$ |
| `values` | 각 물건의 가치 배열 | 길이 $n$, $1 \leq v_i \leq 10^4$ |
| `W` | 배낭 최대 용량 | $1 \leq W \leq 10^4$ |
| 반환값 | 얻을 수 있는 최대 가치 | $\geq 0$ |

**엣지케이스 목록**

| 입력 | 기대 출력 | 이유 |
|------|-----------|------|
| `weights=[], values=[], W=100` | `0` | 물건이 없으므로 가치 0 |
| `weights=[5], values=[10], W=3` | `0` | 물건 무게(5)가 용량(3)을 초과 |
| `weights=[1,1,1], values=[1,2,3], W=2` | `5` | 가치 2+3인 두 번째·세 번째 물건 선택 |
| `weights=[1]*100, values=[10000]*100, W=10000` | `1000000` | 최대 입력: 100개 모두 담고 용량 여유 |

## 핵심 아이디어

**핵심 아이디어**: "각 물건을 한 번씩만 쓸 수 있으므로, 1D 배열을 큰 용량부터 내려가며 갱신해 중복 사용을 막는다."

배낭 문제의 핵심은 각 물건이 최대 1번만 담길 수 있다는 0/1 제약이다. 2D DP를 1D로 압축할 때 배열을 작은 인덱스부터 올리면 같은 물건이 여러 번 담기는 무한 배낭이 되어버린다. 큰 용량부터 내려가며 갱신하면 물건 $i$를 처리하는 동안 이미 갱신된 값을 참조하는 일이 없으므로 0/1 제약이 유지된다.

**풀이 구조**
1. `dp[0..W] = 0`으로 초기화한다.
2. 각 물건 `i`에 대해 바깥 루프를 돈다.
3. 안쪽 루프는 `w = W`부터 `weights[i]`까지 내림차순으로 순회한다.
4. `dp[w] = max(dp[w], dp[w - weights[i]] + values[i])`로 갱신한다.
5. `dp[W]`를 반환한다.

**조건**: 각 아이템을 최대 1번만 선택할 수 있고(0/1 제약), 아이템마다 무게와 가치가 있을 때 가치의 합을 최대화하는 문제에 적용 가능하다.

**대표 예시**: `weights=[2,3,4], values=[3,4,5], W=5`
물건이 3개이고 배낭 용량이 5일 때, 물건 0(무게 2, 가치 3)과 물건 1(무게 3, 가치 4)을 선택하면 총 무게 5에 가치 7이 된다. 큰 용량부터 내림차순으로 갱신하면 물건 0을 처리할 때 `dp[2]=3`, `dp[3]=3`, `dp[4]=3`, `dp[5]=3`이 되고, 물건 1 처리 후 `dp[5] = max(3, dp[2]+4) = 7`이 올바르게 계산된다.

**언제 쓰나**
"N개의 아이템 중 일부를 선택해 용량 W 이내에서 가치를 최대화하라"는 형태이고, 아이템을 중복 없이 각 1번씩만 쓸 수 있을 때 이 패턴을 사용한다.

### 원형 아이디어와 naive 접근

각 물건에 대해 "담는다" / "담지 않는다"를 선택하는 재귀를 생각해 보자.

```
function solve(i, remainW):
    if i == n or remainW == 0: return 0
    // 물건 i를 담지 않는 경우
    skip = solve(i + 1, remainW)
    // 물건 i를 담는 경우 (무게가 허용될 때만)
    take = 0
    if weights[i] <= remainW:
        take = values[i] + solve(i + 1, remainW - weights[i])
    return max(skip, take)
```

이 재귀는 두 가지 문제를 가진다.

1. **지수 시간**: 선택 트리의 높이가 $n$이고 분기가 2개이므로 $O(2^n)$.
2. **중복 부분문제**: `solve(3, 50)`이 서로 다른 경로에서 여러 번 호출된다. 물건 0~2를 담은 방식이 달라도 `(i=3, remainW=50)` 상태의 최적값은 동일하다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1 — 최적 부분구조**: $i$번째 물건부터 $n-1$번째 물건까지만 고려하고 용량이 $w$일 때의 최적값은, $i+1$번째 물건부터의 최적값으로 표현된다. 이 구조가 성립하면 DP를 적용할 수 있다.
- **관찰 2 — 중복 부분문제**: 상태 `(물건 인덱스 i, 남은 용량 w)`는 $n \times (W+1)$가지밖에 없다. 재귀 트리는 $O(2^n)$이지만, 실제로 다른 상태는 $O(n \cdot W)$개뿐이다. 메모이제이션 또는 바텀업 DP로 각 상태를 한 번만 계산한다.
- **관찰 3 — 0/1 제약**: 각 물건은 최대 1번만 사용한다. 이 제약을 유지하려면 1D 롤링 배열에서 갱신 방향이 중요해진다(무한 배낭과의 차이).

### 관찰을 형식화: 상태/구조 정의

$$dp[i][w] = \text{"물건 } 1 \ldots i \text{까지만 고려하고 배낭 용량이 } w \text{일 때 얻을 수 있는 최대 가치"}$$

초기 조건:

$$dp[0][w] = 0 \quad \forall w \in [0, W]$$

물건을 0개 고려했을 때 가치는 항상 0이다.

왜 이 2D 정의여야 하는가? 상태에서 "물건 인덱스"를 포함하지 않으면 물건 $i$를 "아직 처리하지 않은 물건과의 상호작용"을 표현할 수 없다. 반면 이 2D 정의는 "물건 $i$를 포함할지 말지"의 두 선택을 이전 행($i-1$)의 값으로 표현할 수 있게 해 준다.

### 점화식 또는 핵심 연산

물건 $i$를 고려할 때 두 가지 경우를 분기한다.

$$dp[i][w] = \begin{cases} dp[i-1][w] & \text{if } w_i > w \quad \text{(물건 } i \text{를 담을 수 없음)} \\ \max\!\bigl(dp[i-1][w],\; dp[i-1][w - w_i] + v_i\bigr) & \text{otherwise} \end{cases}$$

각 항의 의미:
- $dp[i-1][w]$: 물건 $i$를 **담지 않는** 경우 — 이전 상태 그대로
- $dp[i-1][w - w_i] + v_i$: 물건 $i$를 **담는** 경우 — 물건 $i$의 무게만큼 용량을 비우고 이전 물건들로 최적화한 결과에 $v_i$를 추가
- $\max(\cdot)$: 두 선택 중 더 나은 것을 취함

결과: $dp[n][W]$

### 정당성 — 왜 이것이 옳은가

귀납으로 검증한다. $dp[0][w] = 0$은 자명하게 올바르다. $dp[i-1][\cdot]$이 물건 $1 \ldots i-1$까지의 최적값이라고 가정하면, 물건 $i$에 대해 "담지 않는다"와 "담는다" 두 경우의 최적을 비교하면 $dp[i][w]$가 결정된다. 0/1 제약(각 물건을 최대 1번)이 보장되는 이유는 "담는" 경우 참조하는 $dp[i-1][w - w_i]$가 물건 $i$를 포함하지 않은 값이기 때문이다.

특수 케이스: $v_i$가 음수인 경우는 문제 제약상 발생하지 않는다($v_i \geq 1$). 하지만 $w_i = W$인 물건 하나만 있고 그것을 담으면 용량이 정확히 $W$가 되는 경우도 정상적으로 처리된다.

### 구현 디테일과 최적화

**2D → 1D 롤링 배열**: $dp[i][w]$를 계산할 때 $dp[i-1][\cdot]$만 필요하므로, 1D 배열 하나로 "in-place" 갱신이 가능하다. 단, 반드시 $w$를 **큰 값에서 작은 값으로** 순회해야 한다.

왜 큰 값부터인가? 물건 $i$를 "담는" 경우 $dp[w - w_i]$를 참조한다. $w$를 작은 값부터 갱신하면 $dp[w - w_i]$가 이미 물건 $i$를 반영한 값으로 덮어써진 뒤이므로, 물건 $i$를 여러 번 담는 효과(= 무한 배낭)가 발생한다. 큰 값부터 갱신하면 $dp[w - w_i]$는 아직 물건 $i$가 반영되지 않은 값이므로 0/1 제약이 유지된다.

**흔한 함정**:
- 1D 배열에서 $w$를 오름차순으로 순회하면 무한 배낭이 된다.
- `values.length !== weights.length`인 경우를 사전에 검사해야 한다.
- $W = 0$이면 모든 물건을 담을 수 없으므로 0을 반환해야 한다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function knapsack01(weights, values, W):
    n = weights.length
    dp = 크기 (W+1)인 배열, 0으로 초기화
    // 불변식: dp[w]는 "지금까지 처리한 물건들만 사용해 용량 w에서 얻는 최대 가치"

    for i = 0 to n-1:                           // 물건 순서대로 처리
        for w = W downto weights[i]:             // 큰 용량부터 순회 (0/1 제약 유지)
            // 물건 i를 담는 경우 vs 담지 않는 경우
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
        // 루프 불변식: 물건 0..i까지 고려 완료

    return dp[W]
```

**핵심 불변식**: 바깥 루프 $i$가 완료된 시점에 $dp[w]$는 "물건 $0 \ldots i$까지만 고려하고 배낭 용량이 $w$일 때 얻을 수 있는 최대 가치"이다. 안쪽 루프가 큰 $w$부터 감소하므로, 물건 $i$가 같은 행에서 자기 자신을 참조하는 일은 발생하지 않는다.
