# palindromePartitioningMinCut — 팰린드롬 분할 최소 컷 해설

## 성능 목표 예측

| 제약 | 값 |
|------|-----|
| 문자열 길이 $n = |s|$ | $1 \leq n \leq 2000$ |
| 문자 종류 | 영문 소문자 26가지 |

**Naive 접근의 복잡도 분석**

가장 단순한 접근은 모든 분할 위치의 집합을 완전 탐색하는 것이다. 길이 $n$인 문자열에는 $n-1$개의 분할 가능 위치가 있으므로, 모든 분할 패턴의 수는 $2^{n-1}$가지이다. $n = 2000$이면 $2^{1999}$가지 — 완전히 불가능하다.

분할 탐색을 재귀로 개선해도, 각 분할 후보에서 팰린드롬 여부를 $O(n)$에 판정하면 전체적으로 $O(n^3)$이 된다. $n = 2000$이면 $8 \times 10^9$ 연산으로 시간 초과가 된다.

**목표 복잡도**

$$O(n^2) = O(2000^2) = O(4 \times 10^6)$$

두 단계 DP — 팰린드롬 판정 테이블을 $O(n^2)$에 사전 구성하고, 최소 컷을 $O(n^2)$에 계산한다.

**공간 복잡도**

$$O(n^2) = O(4 \times 10^6)$$

$n \times n$ 팰린드롬 판정 테이블이 필요하다. 부울 배열이므로 약 4MB (비트 최적화 시 더 줄어든다). 최소 컷 배열은 $O(n)$만 추가로 필요하다.

## 핵심 아이디어

### 원형 아이디어와 naive 접근

문자열 $s[0 \ldots n-1]$의 모든 분할 지점을 재귀로 탐색한다.

```
function minCuts(start):
    if start == n: return -1   // 더 이상 나눌 필요 없음 (이전 부분 문자열 완료)
    best = Infinity
    for end = start to n-1:
        if isPalindrome(s[start..end]):
            best = min(best, 1 + minCuts(end + 1))
    return best

answer = minCuts(0) - 1   // 부분 문자열 수에서 1을 뺀 컷 수
```

`isPalindrome(s[start..end])` 판정에 $O(n)$이 걸리고, 재귀 트리의 크기가 $O(2^n)$이므로 전체 $O(n \cdot 2^n)$이다. 더 나아가 중복 부분문제를 메모이제이션해도 상태가 $O(n)$, 각 상태에서 팰린드롬 판정에 $O(n^2)$가 걸려 전체 $O(n^3)$이 된다. $n = 2000$에서 여전히 느리다.

두 가지 낭비가 있다. 첫째, 팰린드롬 판정을 반복 수행한다. 둘째, 재귀 트리에서 같은 시작 위치가 여러 경로에 등장한다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1 — 팰린드롬 판정의 재사용**: 구간 $[i, j]$가 팰린드롬인지를 매번 $O(n)$에 판정하는 대신, 모든 구간에 대한 판정을 $O(n^2)$에 사전 계산해 테이블에 저장하면 이후 $O(1)$ 조회가 가능하다.
- **관찰 2 — 1D DP로 축소**: "문자열 $s[0 \ldots i]$의 최소 컷"은 $s[0 \ldots j-1]$의 최소 컷에 1을 더한 값으로 표현할 수 있다(단, $s[j \ldots i]$가 팰린드롬인 경우). 이 관찰로 2D 재귀를 1D DP로 단순화할 수 있다.
- **관찰 3 — 팰린드롬의 구조적 성질**: 구간 $[i, j]$가 팰린드롬이면 $s[i] = s[j]$이고 안쪽 $[i+1, j-1]$도 팰린드롬이다. 이 점화식으로 짧은 구간에서 긴 구간으로 팰린드롬 테이블을 채울 수 있다.

### 관찰을 형식화: 상태/구조 정의

**1단계 — 팰린드롬 테이블**:

$$isPal[i][j] = \begin{cases} \text{true} & \text{if } s[i \ldots j] \text{가 팰린드롬} \\ \text{false} & \text{otherwise} \end{cases}$$

**2단계 — 최소 컷 배열**:

$$cuts[i] = \text{"} s[0 \ldots i] \text{를 모두 팰린드롬 부분 문자열로 분할할 때의 최소 컷 횟수"}$$

$cuts$는 1D 배열이다. 왜 1D로 충분한가? $cuts[i]$는 접두사 $s[0 \ldots i]$만의 최소 컷을 나타낸다. 어떤 분할이든 반드시 접두사를 단계적으로 확장하는 구조이므로, 시작 위치를 0으로 고정하면 끝 위치 $i$만으로 상태를 표현할 수 있다.

### 점화식 또는 핵심 연산

**팰린드롬 테이블 전이**:

$$isPal[i][j] = (s[i] = s[j]) \wedge \bigl(j - i \leq 2 \,\vee\, isPal[i+1][j-1]\bigr)$$

각 항의 의미:
- $(s[i] = s[j])$: 양 끝 문자가 같아야 한다.
- $j - i \leq 2$: 길이 1 또는 2인 경우, 양 끝이 같으면 자동으로 팰린드롬이다.
- $isPal[i+1][j-1]$: 안쪽 부분도 팰린드롬이어야 한다.

**최소 컷 전이**:

$$cuts[i] = \begin{cases} 0 & \text{if } isPal[0][i] \;\; (s[0 \ldots i] \text{ 전체가 팰린드롬)} \\ \displaystyle\min_{1 \leq j \leq i,\; isPal[j][i]} \bigl(cuts[j-1] + 1\bigr) & \text{otherwise} \end{cases}$$

각 항의 의미:
- $isPal[0][i]$: 전체를 컷 없이 그대로 쓸 수 있는 최선의 경우
- $cuts[j-1]$: $s[0 \ldots j-1]$의 최소 컷 수
- $+1$: $s[j-1]$과 $s[j]$ 사이에 컷 한 번 추가
- $isPal[j][i]$: $s[j \ldots i]$가 팰린드롬인 분할만 유효

결과: $cuts[n-1]$

### 정당성 — 왜 이것이 옳은가

**팰린드롬 테이블**의 정확성: 길이 1($i = j$)에서는 자명하게 팰린드롬이다. 길이 2에서는 양 끝 문자 비교만 필요하다. 길이 $> 2$에서는 $isPal[i+1][j-1]$이 먼저 계산되어 있으므로 귀납이 성립한다. 구간 길이 순서로 채우면 의존성이 항상 먼저 해결된다.

**최소 컷 배열**의 정확성: $cuts[0]$은 $s[0]$ 혼자이므로 0이다. $cuts[i]$를 계산할 때 참조하는 $cuts[j-1]$ ($j-1 < i$)은 이미 계산된 값이다. $isPal[0][i]$이면 컷이 0번이고, 그렇지 않으면 유효한 팰린드롬 suffix $s[j \ldots i]$에 대해 $cuts[j-1] + 1$의 최솟값을 취한다. 모든 가능한 마지막 팰린드롬 조각을 탐색하므로 최소가 보장된다.

특수 케이스:
- 길이 2인 구간에서 $isPal[i+1][j-1]$은 $isPal[i+1][i]$ — 길이 0 구간이다. 길이 0 구간은 항상 팰린드롬으로 처리해야 한다(조건 $j - i \leq 2$로 처리).
- 모든 문자가 같은 경우: `isPal[0][n-1] = true`이므로 $cuts[n-1] = 0$이 된다.

### 구현 디테일과 최적화

**팰린드롬 테이블 채우기 순서**: 짧은 구간부터 채워야 긴 구간이 참조할 때 값이 준비되어 있다. `for len = 1 to n` → `for i = 0 to n-len` → `j = i + len - 1` 순서로 처리한다. 또는 `isPal[i][i] = true`, `isPal[i][i+1] = (s[i]==s[i+1])`로 기저 케이스를 처리한 뒤 `for len = 3 to n`으로 나머지를 채운다.

**흔한 함정**:
- `cuts[i]`의 초기값을 $+\infty$로 설정하지 않으면 `min` 연산이 깨진다.
- $j = 0$인 경우(즉, $isPal[0][i]$인 경우)를 분리하지 않으면 `cuts[-1]`을 참조하게 된다. $j \geq 1$인 경우만 `cuts[j-1]`을 참조하도록 분기해야 한다.
- `isPal[i+1][j-1]`에서 $i+1 > j-1$인 경우(길이 0 또는 1 구간)를 `true`로 처리해야 한다. 조건 `j - i <= 2`를 추가하거나 기저 케이스를 명시적으로 채운다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function palindromePartitioningMinCut(s):
    n = s.length
    isPal = n × n 배열, false로 초기화

    // 1단계: 팰린드롬 판정 테이블 구성
    // 불변식: isPal[i][j]는 s[i..j]가 팰린드롬인지 여부
    for i = 0 to n-1:
        isPal[i][i] = true              // 길이 1
    for i = 0 to n-2:
        isPal[i][i+1] = (s[i] == s[i+1])  // 길이 2

    for len = 3 to n:                   // 길이 3 이상
        for i = 0 to n - len:
            j = i + len - 1
            isPal[i][j] = (s[i] == s[j]) && isPal[i+1][j-1]

    // 2단계: 최소 컷 배열 구성
    cuts = 크기 n인 배열, +Infinity로 초기화
    // 불변식: cuts[i]는 s[0..i]의 최소 팰린드롬 분할 컷 수

    for i = 0 to n-1:
        if isPal[0][i]:                 // 전체가 팰린드롬
            cuts[i] = 0
        else:
            for j = 1 to i:
                if isPal[j][i]:         // s[j..i]가 팰린드롬인 경우
                    cuts[i] = min(cuts[i], cuts[j-1] + 1)

    return cuts[n-1]
```
