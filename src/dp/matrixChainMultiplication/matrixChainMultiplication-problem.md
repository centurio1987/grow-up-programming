# 행렬 체인 곱셈 최소 비용

## 한 줄 요약

> 함수는 행렬 체인의 차원 배열을 받아, 모든 행렬을 곱할 때 필요한 최소 스칼라 곱셈 횟수를 반환한다.

## 스토리

그래픽스 엔진을 개발하는 채린은 여러 변환 행렬을 순서대로 곱해 최종 좌표를 계산하는 코드를 최적화하고 있다. 수학적으로 결합 법칙이 성립하므로 곱셈 순서를 어떻게 묶느냐에 따라 실제 연산 횟수가 크게 달라진다는 사실을 알게 됐다.

채린은 같은 결과를 내면서도 연산 횟수를 가능한 한 줄이고 싶다. 행렬이 많아질수록 경우의 수도 폭발적으로 늘어나, 눈으로 최적 배열을 찾기는 불가능하다.

두 행렬 $p \times q$와 $q \times r$을 곱하는 비용은 $p \cdot q \cdot r$번의 스칼라 곱셈이다.

## 함수 인터페이스

```ts
export function matrixChainMultiplication(dims: number[]): number;
```

- `dims` — 길이 $n + 1$의 양의 정수 배열. $i$번째 행렬의 크기는 $dims[i-1] \times dims[i]$이다.
- 반환 — 행렬 체인 전체를 곱할 때 필요한 최소 스칼라 곱셈 횟수

## 제약 조건

- $1 \leq |dims| \leq 101$ (즉 행렬 수 $n \leq 100$)
- $1 \leq dims[i] \leq 500$
- 행렬은 주어진 순서대로 곱해야 하며, 순서 자체는 바꿀 수 없다.
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

$n$개의 행렬 $A_1, A_2, \ldots, A_n$이 주어지고, 이를 반드시 이 순서로 곱해야 한다. 결합 방식(어느 구간을 먼저 곱할지)은 자유롭게 선택할 수 있다.

행렬 $A_i$의 크기는 $dims[i-1] \times dims[i]$이다.

$|dims| = 1$ (행렬이 없음) 또는 $|dims| = 2$ (행렬 1개)인 경우 곱셈이 발생하지 않으므로 $0$을 반환한다.

## 예시

```ts
matrixChainMultiplication([10, 20, 30]);
// 6000 — 행렬이 두 개(10×20, 20×30)이므로 결합 방식이 한 가지: 10·20·30 = 6000

matrixChainMultiplication([10, 30, 5, 60]);
// 4500 — 세 가지 결합 중 (A1A2)A3 방식이 10·30·5 + 10·5·60 = 1500 + 3000 = 4500으로 최소

matrixChainMultiplication([10, 20, 30, 40, 30]);
// 30000 — 최적 결합 순서를 선택했을 때의 최솟값

matrixChainMultiplication([10, 20]);
// 0 — 행렬이 한 개뿐이므로 곱셈 없음

matrixChainMultiplication([5, 5, 5, 5]);
// 250 — 세 행렬 모두 5×5이므로 어떤 순서로 묶어도 5·5·5 × 2 = 250
```
