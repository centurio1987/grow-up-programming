# treeMaxIndependentSet — 트리 최대 가중 독립집합 해설

## 성능 목표 예측

| 제약 | 값 |
|------|-----|
| 노드 수 $n$ | $1 \leq n \leq 10^5$ |
| 가중치 $weights[i]$ | $-10^4 \leq weights[i] \leq 10^4$ |
| 간선 수 | $n - 1$ (트리 구조) |

**Naive 접근의 복잡도 분석**

그래프 최대 독립집합은 NP-완전이지만, 트리는 구조가 단순해 다항 시간에 해결 가능하다. 그러나 naive하게 모든 독립집합을 열거하면 $2^n$가지 부분집합을 검사해야 한다. $n = 10^5$이면 이는 천문학적 수이다.

일반 그래프에서의 접근(역추적)도 트리에는 과도하다. 트리의 부모-자식 구조를 활용하면 훨씬 효율적으로 처리할 수 있다.

**목표 복잡도**

$$O(n)$$

각 노드를 정확히 한 번 방문하는 후위 순회 DFS로 해결한다. 각 노드에서 자식들의 DP 값을 합산하는 작업이 간선 수에 비례하므로, 전체 작업량은 $O(n)$이다.

**공간 복잡도**

$$O(n)$$

인접 리스트, DP 배열(각 노드당 2개 값), 방문 스택이 모두 $O(n)$이다. $n = 10^5$에서 재귀 DFS는 스택 오버플로 위험이 있으므로 반복 DFS를 권장한다.

---

## 목표 함수

```ts
function treeMaxIndependentSet(
  n: number,
  edges: [number, number][],
  weights: number[]
): number
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `n` | 노드 수 | $1 \leq n \leq 10^5$ |
| `edges` | 무방향 간선 목록 | 길이 $n-1$, 트리 보장 |
| `weights` | 노드 가중치 배열 | 길이 $n$, $-10^4 \leq weights[i] \leq 10^4$ |
| 반환값 | 독립집합의 최대 가중치 합 | 정수 |

**엣지케이스 목록**

| 입력 | 기대 출력 | 이유 |
|------|-----------|------|
| `n=1, edges=[], weights=[5]` | `5` | 노드 하나: 그 자체가 독립집합 |
| `n=2, edges=[[0,1]], weights=[3,-1]` | `3` | 노드 0만 선택 (인접하므로 둘 다 불가) |
| `n=3, edges=[[0,1],[1,2]], weights=[-1,-1,-1]` | `0` | 모든 가중치 음수 → 빈 집합 선택 (합=0) |
| `n=4, edges=[[0,1],[0,2],[0,3]], weights=[1,10,10,10]` | `30` | 노드 0과 인접한 1,2,3은 선택 불가. 노드 1+2+3=30 vs 노드 0=1 |

## 핵심 아이디어

**핵심 아이디어**: "트리를 후위 순회하며 각 노드에 대해 '선택/미선택' 두 가지 값만 유지하면 전체 최적이 자동으로 결정된다."

일반 그래프에서의 최대 독립집합은 NP-완전이지만, 트리는 계층 구조 덕분에 $O(n)$에 해결된다. 트리에서 노드 $v$를 선택하면 인접한 자식들만 선택 불가능해지고, 서로 다른 서브트리 사이에는 간선이 없으므로 독립적으로 최적화할 수 있다. 이 서브트리 독립성이 핵심으로, 리프에서 루트 방향으로 후위 순회하며 두 값(`dp[v][0]`, `dp[v][1]`)을 올라보내면 된다.

**풀이 구조**
1. 무방향 간선 목록으로 인접 리스트를 구성한다.
2. 루트 0에서 후위 순회(DFS)를 시작한다.
3. 각 노드 `v`에 대해 `dp[v][0] = 0`, `dp[v][1] = weights[v]`로 초기화한다.
4. 자식 `u` 각각에 대해 `dp[v][0] += max(dp[u][0], dp[u][1])`, `dp[v][1] += dp[u][0]`를 합산한다.
5. `max(dp[0][0], dp[0][1])`을 반환한다.

**조건**: 트리(사이클 없는 연결 그래프) 구조에서 인접한 두 노드를 동시에 선택할 수 없는 독립집합 최대화 문제에 적용 가능하다.

**대표 예시**: 별 모양 트리 `n=4, edges=[[0,1],[0,2],[0,3]], weights=[1,10,10,10]`
루트 0을 선택하면 1,2,3을 모두 포기해야 하므로 가치 1. 루트 0을 선택 안 하면 1,2,3 모두 자유롭게 선택 가능해 10+10+10=30. `dp[0][0]=30`, `dp[0][1]=1`이 되어 최댓값 30을 반환한다.

**언제 쓰나**
"트리에서 인접 노드를 동시에 선택할 수 없는 조건 하에 가중치 합을 최대화하라"거나 "트리의 계층적 상호 배제 조건이 있는 최적화 문제"에서 이 트리 DP 패턴을 사용한다.

### 원형 아이디어와 naive 접근

모든 부분집합 $S \subseteq \{0, 1, \ldots, n-1\}$에 대해 "독립집합인지" 확인하고, 독립집합이면 가중치 합을 계산해 최댓값을 갱신한다.

```
best = 0
for each subset S of {0..n-1}:
    if isIndependent(S):    // 인접한 두 노드가 모두 S에 속하지 않는지 확인
        best = max(best, sum of weights[v] for v in S)
return best
```

부분집합의 수는 $2^n$이고 독립집합 확인에 $O(n)$이 걸려 총 $O(n \cdot 2^n)$이다. $n = 10^5$에서 불가능하다.

핵심 낭비는 트리의 **계층 구조를 무시**하는 점이다. 트리에서는 노드 $v$가 선택되면 그 부모와 자식만 배제하면 된다. 다른 노드들은 $v$의 선택 여부와 무관하게 서로 최적으로 선택할 수 있다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1 — 트리의 부모-자식 관계가 유일하다**: 임의의 노드 $v$를 루트로 두면, 트리가 계층 구조를 가진다. $v$가 선택되면 $v$의 부모와 자식들은 선택 불가다. 반대로 $v$가 선택되지 않으면 자식들은 독립적으로 최적화할 수 있다.
- **관찰 2 — 서브트리 독립성**: 노드 $v$를 기준으로 서로 다른 자식의 서브트리는 간선으로 연결되어 있지 않다(트리 정의상). 따라서 자식 $u_1$의 서브트리와 자식 $u_2$의 서브트리 내 선택이 서로에게 영향을 주지 않는다. 각 서브트리를 독립적으로 최적화한 결과를 단순히 합산하면 전체 최적이 된다.
- **관찰 3 — 두 가지 선택의 완전성**: 각 노드 $v$에 대해 "선택한다"와 "선택하지 않는다" 두 경우만 존재하며, 이 두 경우가 서브트리 내 모든 가능성을 망라한다.

### 관찰을 형식화: 상태/구조 정의

각 노드 $v$에 대해 두 값을 정의한다.

$$dp[v][0] = \text{"노드 } v \text{를 선택하지 않을 때, } v \text{의 서브트리에서 얻는 최대 가중치 합"}$$

$$dp[v][1] = \text{"노드 } v \text{를 선택할 때, } v \text{의 서브트리에서 얻는 최대 가중치 합"}$$

이 정의가 적절한 이유: $v$의 선택 여부가 부모와 자식에 영향을 미치므로, 서브트리 값을 반환할 때 "루트를 선택했는가?"라는 정보를 함께 전달해야 한다. 이 2가지 값이면 부모 노드가 자신의 상태에 따라 자식 값을 올바르게 선택할 수 있다.

### 점화식 또는 핵심 연산

$v$의 자식 집합을 $C(v)$라 할 때:

**$v$를 선택하지 않는 경우** ($dp[v][0]$):

$$dp[v][0] = \sum_{u \in C(v)} \max\!\bigl(dp[u][0],\; dp[u][1]\bigr)$$

각 항의 의미:
- 자식 $u$는 선택해도 되고 안 해도 된다 — 더 큰 값을 선택
- 모든 자식의 기여를 합산

**$v$를 선택하는 경우** ($dp[v][1]$):

$$dp[v][1] = weights[v] + \sum_{u \in C(v)} dp[u][0]$$

각 항의 의미:
- $weights[v]$: $v$ 자체의 가중치
- $dp[u][0]$: $v$가 선택되므로 인접한 자식 $u$는 반드시 선택 불가

결과 (루트를 0으로 설정):

$$\text{treeMaxIndependentSet}(n, edges, weights) = \max\!\bigl(dp[0][0],\; dp[0][1]\bigr)$$

### 정당성 — 왜 이것이 옳은가

귀납적으로 리프(잎) 노드부터 검증한다. 리프 노드 $v$는 $C(v) = \emptyset$이므로 $dp[v][0] = 0$, $dp[v][1] = weights[v]$다. 이는 "선택 안 하면 가중치 0, 선택하면 본인 가중치"로 자명히 올바르다.

내부 노드 $v$에서, 귀납 가정으로 모든 자식 $u \in C(v)$의 $dp[u][0]$과 $dp[u][1]$이 각자의 서브트리에서 정확한 최대 가중치 합이라고 하자. $v$를 선택하지 않으면 각 자식이 자유롭게 선택 가능하므로 $\max(dp[u][0], dp[u][1])$의 합이 최적이다. $v$를 선택하면 모든 자식이 배제되므로 $dp[u][0]$만 사용 가능하다. 두 경우를 모두 계산해 $\max$를 취하므로 전체 최적이 보장된다.

특수 케이스: 음수 가중치가 있으면 선택하지 않는 것이 유리할 수 있다. $dp[v][0]$와 $dp[v][1]$ 중 더 큰 값을 취하므로 자동으로 처리된다. $n = 1$인 단일 노드는 $dp[0][0] = 0$, $dp[0][1] = weights[0]$이며, 음수 가중치이면 $dp[0][0] = 0$이 선택된다.

### 구현 디테일과 최적화

**재귀 DFS vs 반복 DFS**: $n \leq 10^5$에서 재귀 DFS는 스택 깊이가 최대 $n$에 달할 수 있어 스택 오버플로 위험이 있다. 명시적 스택을 사용한 반복 DFS를 권장한다.

**반복 DFS 구현 패턴 — ENTER/EXIT 두 단계**: 반복 DFS에서 후위 순회(자식 처리 후 부모 처리)를 구현하려면 각 노드를 두 번 스택에 넣는 방식을 사용한다. 처음 ENTER 단계에서 초기화하고 자식들을 스택에 푸시한다. EXIT 단계에서 자식들의 결과를 합산한다.

**부모 추적**: 무방향 간선 목록으로부터 인접 리스트를 구성하면 부모 방향과 자식 방향이 구분되지 않는다. DFS 중 현재 노드의 이전 방문 노드(부모)를 별도로 기억해 자식과 부모를 구분한다.

**흔한 함정**:
- EXIT 단계에서 자식 $u$를 순회할 때 부모 방향($u == parent$)을 제외하지 않으면, 부모를 자식으로 오인해 순환이 발생한다.
- `dp[v][0]`을 초기화하지 않고 누산하면 이전 값이 남아 잘못된 결과가 나온다. 반드시 ENTER 단계에서 `dp[v][0] = 0`, `dp[v][1] = weights[v]`로 초기화한다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function treeMaxIndependentSet(n, edges, weights):
    // 인접 리스트 구성
    adj = 크기 n인 빈 리스트 배열
    for (u, v) in edges:
        adj[u].push(v)
        adj[v].push(u)

    dp = n × 2 배열, 0으로 초기화
    // 불변식: dp[v][0]은 v 미선택 시 v 서브트리 최대 가중치 합
    // 불변식: dp[v][1]은 v 선택 시 v 서브트리 최대 가중치 합

    // 반복 DFS (스택 오버플로 방지)
    parent = 크기 n인 배열, -1로 초기화
    stack = [(노드=0, phase=ENTER)]

    while stack not empty:
        (v, phase) = stack.pop()

        if phase == ENTER:
            dp[v][0] = 0
            dp[v][1] = weights[v]
            stack.push((v, EXIT))                // EXIT 단계 예약
            for u in adj[v]:
                if u != parent[v]:               // 자식만 처리 (부모 제외)
                    parent[u] = v
                    stack.push((u, ENTER))

        else:  // EXIT (후위 처리)
            for u in adj[v]:
                if u != parent[v]:               // 자식 u의 값 합산
                    dp[v][0] += max(dp[u][0], dp[u][1])
                    dp[v][1] += dp[u][0]

    return max(dp[0][0], dp[0][1])
```
