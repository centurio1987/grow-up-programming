# subsetSum — 부분집합 합 해설

## 성능 목표 예측

| 제약 | 값 |
|------|-----|
| 배열 크기 $n$ | $1 \leq n \leq 1000$ |
| 목표 합 $target$ | $0 \leq target \leq 10^4$ |
| 원소 값 $nums[i]$ | $0 \leq nums[i] \leq 10^4$ |

**Naive 접근의 복잡도 분석**

부분집합 합 문제의 가장 단순한 접근은 모든 부분집합을 열거하는 것이다. $n$개 원소에서 부분집합의 수는 $2^n$이다. $n = 1000$이면 $2^{1000} \approx 10^{301}$가지 — 우주의 나이보다도 훨씬 긴 시간이 걸린다. 이것이 naive 접근의 핵심 한계다.

재귀적으로 구현해도, 메모이제이션 없이는 같은 `(인덱스, 남은 합)` 쌍이 반복 등장해 지수 시간이 유지된다.

**목표 복잡도**

$$O(n \cdot target) = O(1000 \times 10^4) = O(10^7)$$

상태 공간이 `(원소 인덱스, 가능한 합)`의 $n \times (target+1)$가지이고, 각 상태를 $O(1)$에 처리하므로 목표를 달성한다.

**공간 복잡도**

- 2D 테이블 $O(n \times target)$ 대신 **1D 롤링 배열** $O(target)$으로 충분하다.
- $target \leq 10^4$이면 배열 크기 10,001 — 매우 작다.

---

## 목표 함수

```ts
function subsetSum(nums: number[], target: number): boolean
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `nums` | 비음의 정수 배열 | $1 \leq n \leq 1000$, $0 \leq nums[i] \leq 10^4$ |
| `target` | 목표 합 | $0 \leq target \leq 10^4$ |
| 반환값 | 합이 $target$인 부분집합이 존재하면 `true` | 부울 |

**엣지케이스 목록**

| 입력 | 기대 출력 | 이유 |
|------|-----------|------|
| `nums=[1,2,3], target=0` | `true` | 빈 부분집합의 합이 0 |
| `nums=[2,4,6], target=5` | `false` | 모든 원소가 짝수; 5는 홀수라 불가 |
| `nums=[10000], target=10000` | `true` | 단일 원소가 target과 일치 |
| `nums=[0,0,0], target=0` | `true` | 0을 포함한 부분집합 존재 |

## 핵심 아이디어

**핵심 아이디어**: "어떤 부분집합을 골랐는지 기억하는 대신, 달성 가능한 합의 집합만 비트맵으로 추적한다."

부분집합은 $2^n$개나 되지만, "합 $x$가 달성 가능한가?"라는 불리언 정보는 `target+1`개의 비트로 충분하다. 새 원소 $v$를 추가하면 기존에 달성 가능했던 모든 합 $x$에 대해 $x+v$도 달성 가능해진다. 이 "합 집합의 이동"을 0/1 배낭과 같은 방식(큰 인덱스부터 내림차순 갱신)으로 구현하면 각 원소를 최대 1번만 사용하는 조건이 자동으로 지켜진다.

**풀이 구조**
1. `dp[0] = true`로 초기화하고 나머지는 `false`로 설정한다.
2. 각 원소 `v`에 대해 바깥 루프를 돈다.
3. 안쪽 루프는 `x = target`부터 `v`까지 내림차순으로 순회한다.
4. `dp[x] = dp[x] || dp[x - v]`로 갱신한다.
5. `dp[target]`을 반환한다.

**조건**: 배열의 원소를 각각 0번 또는 1번 선택(0/1 제약)해 합이 정확히 `target`이 되는 부분집합이 존재하는지 판별할 때 적용 가능하다.

**대표 예시**: `nums=[3,1,4], target=4`
초기: `dp = [T,F,F,F,F]`. 원소 3 처리 후: `dp = [T,F,F,T,F]`. 원소 1 처리 후(내림차순): `dp = [T,T,F,T,T]`. 원소 4 처리 후: `dp = [T,T,F,T,T]`. `dp[4] = true`이므로 부분집합 {4} 또는 {3,1}이 존재한다.

**언제 쓰나**
"N개의 수 중 일부를 골라 합이 정확히 K가 되는가?"를 묻는 문제, 또는 전체 합의 절반이 가능한지 묻는 파티션 문제에서 이 패턴을 사용한다.

### 원형 아이디어와 naive 접근

$n$개의 원소 각각에 대해 "포함한다" / "포함하지 않는다"를 결정하는 재귀를 생각해 보자.

```
function solve(index, remaining):
    if remaining == 0: return true      // 합 달성
    if index == n or remaining < 0: return false
    // 원소 index를 포함하는 경우
    if solve(index + 1, remaining - nums[index]): return true
    // 원소 index를 포함하지 않는 경우
    return solve(index + 1, remaining)
```

이 재귀는 두 가지 문제가 있다.

1. **지수 탐색**: 트리의 깊이가 $n$이고 분기가 2개이므로 $O(2^n)$.
2. **중복 부분문제**: `solve(3, 50)`은 원소 0~2를 선택한 방식이 달라도 `(index=3, remaining=50)` 상태가 같으면 결과가 동일하다. 메모이제이션 없이 중복 계산이 발생한다.

이 구조를 관찰하면, 상태는 `(index, remaining)`의 $n \times (target+1)$가지이므로 DP로 $O(n \times target)$에 해결할 수 있다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1 — "도달 가능한 합의 집합"만 추적하면 충분하다**: 부분집합의 구성 자체가 아니라, "합 $x$가 달성 가능한가"라는 불리언 정보만 유지하면 된다. 이 정보는 $target + 1$개의 비트로 표현할 수 있다.
- **관찰 2 — 원소 추가 시 집합의 변화**: 새 원소 $v$를 고려할 때, 기존에 합 $x$가 달성 가능했다면 원소 $v$를 추가해 합 $x + v$도 달성 가능해진다. 즉, "달성 가능한 합의 집합"에 $v$를 더한 이동이 발생한다.
- **관찰 3 — 0/1 제약**: 각 원소는 최대 1번만 사용할 수 있다. 갱신 방향이 이 제약에 결정적 영향을 미친다. 큰 값에서 작은 값 방향으로 순회하면 원소를 여러 번 사용하는 상황을 방지한다.

### 관찰을 형식화: 상태/구조 정의

$$dp[x] = \text{"지금까지 처리한 원소들의 부분집합으로 합 } x \text{를 달성할 수 있는가?"}$$

초기 조건:

$$dp[0] = \text{true}, \quad dp[x] = \text{false} \;\; (x > 0)$$

$dp[0] = \text{true}$의 의미: 빈 부분집합(원소를 하나도 선택하지 않음)의 합은 0이므로, 어떤 원소도 처리하기 전에 합 0은 항상 달성 가능하다.

왜 1D 정의로 충분한가? 2D로 정의하면 $dp[i][x]$는 "원소 $0 \ldots i-1$까지 고려했을 때 합 $x$ 달성 가능 여부"이다. 갱신 시 $dp[i][x] = dp[i-1][x] \vee dp[i-1][x - nums[i-1]]$이 되어, 행 $i$는 행 $i-1$에만 의존한다. 이 의존성 구조 덕분에 1D in-place 갱신이 가능하다.

### 점화식 또는 핵심 연산

원소 $v$를 처리할 때, 합 $x$를 **큰 값부터 작은 값으로** ($x = target$ 내려서 $x = v$까지) 순회하며:

$$dp[x] \gets dp[x] \vee dp[x - v] \quad (x \geq v)$$

각 항의 의미:
- $dp[x]$: 원소 $v$ 이전 상태에서 합 $x$가 달성 가능했는가?
- $dp[x - v]$: 원소 $v$ 이전 상태에서 합 $x - v$가 달성 가능했는가? (원소 $v$를 추가하면 합 $x$가 됨)
- $dp[x] \gets dp[x] \vee dp[x - v]$: 두 경우 중 하나라도 참이면 합 $x$ 달성 가능

결과: $dp[target]$

### 정당성 — 왜 이것이 옳은가

귀납으로 검증한다. 초기에 $dp[0] = \text{true}$이고 나머지는 $\text{false}$이다. 원소 집합 $\{nums[0], \ldots, nums[k-1]\}$까지 처리를 마쳤을 때 $dp[x]$가 해당 집합의 부분집합으로 합 $x$를 달성 가능한지를 정확히 나타낸다고 가정하자. 원소 $nums[k]$를 처리할 때 $x$를 큰 값부터 갱신하면:

$dp[x - nums[k]]$는 아직 $nums[k]$가 반영되지 않은 이전 상태(= 귀납 가정에서의 값)이다. 따라서 $dp[x] \vee dp[x - nums[k]]$는 "$nums[k]$를 포함하지 않고 합 $x$" 또는 "$nums[k]$를 포함하고 합 $x - nums[k]$"를 통해 합 $x$를 달성하는 두 경우를 정확히 처리한다.

큰 값부터 순회하는 이유를 다시 확인한다. 만약 작은 $x$부터 순회한다면, $dp[v]$를 갱신할 때 $dp[0]$(이미 $\text{true}$)를 참조해 $dp[v] = \text{true}$가 되고, 이어 $dp[2v]$를 갱신할 때 이미 갱신된 $dp[v]$를 참조해 $dp[2v] = \text{true}$가 되는 식으로 원소 $v$가 무한 번 사용되는 효과가 나타난다. 큰 값부터 순회하면 이 연쇄가 일어나지 않는다.

### 구현 디테일과 최적화

**조기 종료**: $dp[target] = \text{true}$가 되는 순간 즉시 `true`를 반환할 수 있다. 이후 원소 처리가 불필요하다.

**원소 $v > target$인 경우**: 이 원소는 $target$을 만드는 데 기여할 수 없으므로 건너뛰어도 된다.

**원소 $v = 0$인 경우**: 0을 추가해도 합이 변하지 않으므로 $dp$ 배열이 바뀌지 않는다. 갱신 루프에서 $x - v = x$가 되어 $dp[x] \gets dp[x] \vee dp[x]$가 되므로, 결과적으로 아무 변화도 없다. 별도 처리 없이 자동으로 올바르게 동작한다.

**비트셋 최적화**: 부울 배열 $dp$를 비트셋으로 표현하면 64개씩 병렬 처리할 수 있어 $O(n \cdot target / 64)$로 줄어든다. $n = 1000$, $target = 10^4$에서 이미 충분히 빠르므로 일반적으로 불필요하다.

**흔한 함정**: 큰 값부터 순회해야 한다는 점을 잊고 오름차순으로 작성하면, 같은 원소를 여러 번 선택하는 무한 배낭이 되어 틀린 답이 나온다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function subsetSum(nums, target):
    dp = 크기 (target+1)인 부울 배열, false로 초기화
    dp[0] = true
    // 불변식: dp[x]는 "현재까지 처리한 원소들의 부분집합으로 합 x 달성 가능 여부"

    for v in nums:                              // 각 원소를 순서대로 처리
        if v > target: continue                 // 기여 불가 원소 생략
        for x = target downto v:               // 큰 값부터 순회 (0/1 제약 유지)
            dp[x] = dp[x] || dp[x - v]         // 원소 v 포함 여부 고려
        if dp[target]: return true              // 조기 종료 (선택적 최적화)

    return dp[target]
```

```mermaid

    B --> C{원소 v 순회\n바깥 루프}
    C -- 모든 원소 완료 --> G([dp[target] 반환])
    C -- 원소 v 선택 --> H{"v > target?"}
    H -- Yes --> C
    H -- No --> D{x = target downto v\n안쪽 루프 감소 방향}

    E -- Yes --> F([true 반환])
    E -- No --> C
    D -- x 처리 중 --> I["dp[x] = dp[x] || dp[x - v]"]
    I --> D
```

**핵심 불변식**: 원소 $v$의 처리가 완료된 시점에 $dp[x]$는 "배열 내 $v$까지(포함)의 원소들 중 일부를 골라 합 $x$를 달성할 수 있는가?"를 정확히 나타낸다. 갱신이 큰 $x$부터 내려가므로 각 원소는 정확히 한 번만 사용된다.
