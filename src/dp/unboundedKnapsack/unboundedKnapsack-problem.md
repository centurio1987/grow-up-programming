# 동전 교환 최소 개수

## 한 줄 요약

> 함수는 동전 종류 배열과 목표 금액을 받아, 그 금액을 만드는 데 필요한 최소 동전 개수를 반환하며, 불가능하면 -1을 반환한다.

## 스토리

환전소 직원 민지는 고객에게 거스름돈을 줄 때 항상 가장 적은 수의 동전을 건네고 싶다. 각 동전은 충분히 많이 보유하고 있어 같은 액면가를 여러 번 쓸 수 있다.

민지가 다루는 동전 종류와 잔돈 금액은 매번 다르다. 어떤 조합으로도 정확히 그 금액을 만들 수 없으면 고객에게 불가능하다고 안내해야 한다.

## 함수 인터페이스

```ts
export function unboundedKnapsack(coins: number[], amount: number): number;
```

- `coins` — 사용할 수 있는 동전의 액면가 목록. 각 동전은 무한히 사용 가능하다.
- `amount` — 만들어야 할 목표 금액 (0 이상의 정수)
- 반환 — `amount`를 만드는 데 필요한 최소 동전 개수. 불가능하면 `-1`. `amount`가 0이면 `0`.

## 제약 조건

- $1 \leq |coins| \leq 100$
- $1 \leq coins[i] \leq 10^4$
- $0 \leq amount \leq 10^4$
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

각 동전은 몇 번이든 사용할 수 있다 (무한 공급).

`amount`가 $0$이면 동전이 필요 없으므로 $0$을 반환한다.

주어진 동전들로 `amount`를 정확히 만들 수 없으면 $-1$을 반환한다.

`coins`가 비어 있고 `amount`가 $0$이면 $0$, `amount`가 $0$보다 크면 $-1$을 반환한다.

## 예시

```ts
unboundedKnapsack([1, 2, 5], 11);
// 3 — 5 + 5 + 1 = 11, 동전 3개

unboundedKnapsack([2], 3);
// -1 — 짝수 동전으로 홀수 금액을 만들 수 없음

unboundedKnapsack([1], 0);
// 0 — 목표 금액이 0이므로 동전 불필요

unboundedKnapsack([1, 3, 4], 6);
// 2 — 3 + 3 = 6, 동전 2개 (탐욕적으로 4+1+1=3개보다 적음)

unboundedKnapsack([5, 10], 3);
// -1 — 모든 동전이 목표 금액보다 큼
```
