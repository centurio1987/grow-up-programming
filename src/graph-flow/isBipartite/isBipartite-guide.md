# isBipartite 해설

## 성능 목표 예측

| 항목 | 값 |
|------|----|
| 정점 V | $1 \leq V \leq 10^5$ |
| 간선 E | $0 \leq E \leq 2 \times 10^5$ |
| 그래프 연결성 | 비연결 그래프 허용 |

**naive 접근의 문제점:**
모든 정점 쌍 $(u, v)$에 대해 경로를 탐색해 홀수 사이클 여부를 따지는 방법은
$O(V^2)$ 또는 $O(V \cdot (V + E))$에 달한다.
$V = 10^5$이면 $O(V^2) = 10^{10}$ → 시간 초과.

**목표 복잡도:** $O(V + E)$ 시간, $O(V + E)$ 공간.

**근거:** BFS/DFS는 각 정점과 간선을 한 번씩만 방문한다.
$V + E \leq 3 \times 10^5$이므로 단순 선형 탐색으로 충분히 통과 가능하다.

**공간 복잡도:** 인접 리스트 $O(V + E)$, 색상 배열 $O(V)$. 2D 구조가 필요 없으므로 공간 오버헤드가 없다.

---

## 목표 함수

```ts
isBipartite(n: number, edges: [number, number][]): boolean
```

| 매개변수 | 의미 | 제약 |
|----------|------|------|
| `n` | 정점 개수 $V$ (인덱스: $0 \ldots n-1$) | $1 \leq n \leq 10^5$ |
| `edges` | 무방향 간선 `[u, v]` 목록 | $0 \leq E \leq 2 \times 10^5$ |
| 반환값 | 이분 그래프이면 `true`, 아니면 `false` | — |

**엣지케이스:**

1. **빈 입력 / 간선 없음:** 간선이 없으면 모든 정점이 독립적이므로 $A = \{0, 1, \ldots, n-1\}$, $B = \emptyset$으로 두면 조건을 만족한다. → `true`
2. **단일 정점 ($n = 1$):** 인접 정점이 없으므로 충돌 없음. → `true`
3. **자기 루프 `[u, u]`:** 색상 충돌이 즉시 발생한다. `color[u] == color[u]` → `false`
4. **비연결 그래프:** 각 컴포넌트를 독립적으로 판정해야 한다. 어느 한 컴포넌트라도 이분이 아니면 전체 `false`.
5. **홀수 사이클:** 삼각형(3-사이클)이 있으면 `false`. 짝수 사이클(4-사이클 등)은 이분 그래프 조건과 양립 가능하다.

## 핵심 아이디어

**핵심 아이디어**: "그래프를 두 색으로 칠할 수 있으면 이분 그래프다 — BFS 레이어 구조가 2-착색을 자동으로 수행한다."

이분 그래프의 판별은 홀수 사이클 열거 없이, 정점을 색 0과 색 1로 교대로 칠하며 인접 정점과 색이 충돌하는지만 확인하면 된다. BFS를 시작 정점에서 출발하면 짝수 레이어는 색 0, 홀수 레이어는 색 1로 자연스럽게 대응되고, 이미 착색된 이웃과 색이 같으면 홀수 사이클이 존재함을 의미해 즉시 `false`를 반환할 수 있다. 비연결 그래프도 모든 컴포넌트를 순회하는 외부 루프로 처리하면 된다.

**풀이 구조**
1. 무방향 간선을 양방향 인접 리스트로 구성한다.
2. `color` 배열을 `-1`(미착색)로 초기화한다.
3. 미착색 정점마다 BFS를 시작해 색 0을 부여한다.
4. BFS 중 이웃이 미착색이면 반대 색을 부여하고 큐에 삽입한다.
5. 이미 착색된 이웃의 색이 자신과 같으면 즉시 `false`를 반환한다.
6. 모든 컴포넌트를 처리하고 나면 `true`를 반환한다.

**조건**: 무방향 그래프여야 하며, 비연결 그래프도 처리할 수 있도록 모든 정점을 순회해야 한다. 자기 루프(`[u, u]`)는 즉시 `false`를 유발한다.

**대표 예시**: 학생 둘씩 짝지어 프로젝트 팀 배정
"서로 친한 학생끼리는 다른 팀"이라는 조건이 있을 때, 친밀 관계 그래프가 이분 그래프인지 확인하면 된다. BFS 착색 과정에서 충돌이 발생하면 어떤 분배 방식으로도 조건을 만족할 수 없음을 $O(V+E)$에 판별한다.

**언제 쓰나**
"그래프가 이분 그래프인가?", "홀수 사이클이 존재하는가?", "정점을 두 집합으로 나눌 수 있는가?" 같은 문제에서 사용한다. 이분 그래프 매칭 문제의 전제 조건 검증에도 자주 활용된다.

### 원형 아이디어와 naive 접근

문제를 가장 단순하게 생각하면: 그래프의 모든 사이클을 열거하고, 각 사이클의 길이를 계산해 홀수 사이클이 있는지 확인한다.

```
naive:
  for each cycle C in G:
    if len(C) is odd: return false
  return true
```

사이클 열거는 DFS로 가능하지만, 사이클 수는 최대 $O(E - V + 1)$개이고
각 사이클의 검사 비용은 경로 길이에 비례하므로 전체 비용이 $O(V \cdot E)$ 이상으로 폭발한다.
$V = 10^5$, $E = 2 \times 10^5$이면 $O(V \cdot E) = 2 \times 10^{10}$ → 시간 초과.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1:** 이분 그래프의 동치 조건은 "홀수 사이클 없음"이다.
  즉, 그래프가 이분 그래프 $\Leftrightarrow$ 정점을 두 색(0, 1)으로 칠해 모든 간선이 서로 다른 색 사이를 잇도록 만들 수 있다.

- **관찰 2:** 색칠이 가능한지 판단하는 것은 사이클 열거보다 훨씬 쉽다.
  시작 정점에 색 0을 주고, 이웃은 색 1, 그 이웃은 다시 색 0으로 전파하면 된다.
  이 과정에서 모순(인접 정점이 같은 색)이 발생하는지만 확인하면 충분하다.

- **관찰 3:** BFS의 레이어 구조가 2-착색과 동일하다.
  소스에서 짝수 거리인 정점은 색 0, 홀수 거리인 정점은 색 1로 대응된다.
  따라서 BFS 과정에서 자동으로 2-착색이 수행된다.

### 관찰을 형식화: 상태/구조 정의

상태 변수: `color[v]` ∈ {−1, 0, 1}

- `-1`: 미착색 (아직 방문하지 않은 정점)
- `0`: 집합 A에 속함
- `1`: 집합 B에 속함

이 정의가 2값(0/1)인 이유: 이분 그래프는 정확히 두 집합으로의 분할이므로,
상태가 3값 이상이면 불필요하게 복잡해지고 모순 감지 조건도 불명확해진다.

인접 리스트 `adj[v]`: 무방향 간선 `[u, v]`를 `adj[u]`와 `adj[v]` 양쪽에 등록해 탐색 방향성을 제거한다.

### 점화식 또는 핵심 연산

BFS 과정에서의 착색 규칙:

$$
\text{color}[v] = 1 - \text{color}[u] \quad \text{(v가 미착색이고, u에서 v로 간선이 존재할 때)}
$$

모순 감지 조건:

$$
\text{conflict}(u, v) = [\text{color}[v] \neq -1 \;\land\; \text{color}[v] = \text{color}[u]]
$$

- 우항 $1 - \text{color}[u]$: $u$의 반대 색. 인접 정점은 반드시 다른 색이어야 한다.
- `color[v] != -1`: 이미 착색된 정점에서만 충돌이 발생할 수 있다.
- `color[v] == color[u]`: 같은 색이 인접하면 홀수 사이클이 존재함을 의미한다.

### 정당성 — 왜 이것이 옳은가

귀납적으로 증명한다. BFS 시작 시 `color[start] = 0`으로 초기화한다.
단계 $k$에서 레이어 $k$의 정점들이 모두 올바른 색을 가진다고 가정하면,
레이어 $k+1$의 정점은 `1 - color[parent]`를 배정받으므로 인접 간선이 서로 다른 색을 잇는다.
이 과정에서 이미 착색된 이웃의 색이 같으면 홀수 사이클 존재가 확정된다.

**까다로운 케이스:**
- 자기 루프: `adj[u]`에 `u`가 포함되면, `color[u] == color[u]`가 성립해 즉시 `false`를 반환한다.
- 비연결 그래프: 미착색 정점을 순회하는 외부 루프로 모든 컴포넌트를 처리하므로 누락 없이 판정된다.
- 짝수 사이클: BFS 레이어가 0, 1, 0, 1, ...로 교대하므로 4-사이클 등은 충돌을 일으키지 않는다.

### 구현 디테일과 최적화

- **비연결 그래프 처리:** `for start in 0..n-1` 루프에서 `color[start] == -1`인 경우에만 BFS를 시작한다. 이 루프의 총 비용은 $O(V + E)$ (각 정점·간선은 한 번씩 처리).

- **조기 종료:** 모순 발견 시 즉시 `false`를 반환한다. BFS 큐를 비우거나 다른 컴포넌트를 탐색할 필요가 없다.

- **함정 — 역방향 간선 등록 누락:** 무방향 간선 `[u, v]`를 `adj[u]`에만 등록하면 `v → u` 방향의 충돌을 탐지하지 못한다. 반드시 양쪽에 등록해야 한다.

- **함정 — 비연결 그래프에서 시작 정점만 탐색:** BFS를 정점 0에서만 시작하면 다른 컴포넌트를 판정하지 못한다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function isBipartite(n, edges):
    // 1. 인접 리스트 구성 (무방향)
    adj[0..n-1] = []
    for (u, v) in edges:
        adj[u].push(v)
        adj[v].push(u)           // 불변식: adj는 무방향 그래프를 대칭으로 표현

    // 2. 초기화
    color[0..n-1] = -1           // 불변식: color[v] ∈ {-1, 0, 1}

    // 3. 모든 컴포넌트 탐색
    for start in 0..n-1:
        if color[start] != -1: continue   // 불변식: 이미 착색된 컴포넌트 스킵

        color[start] = 0
        queue = [start]

        while queue is not empty:
            u = dequeue(queue)             // 불변식: color[u]는 이미 확정됨

            for v in adj[u]:
                if color[v] == -1:
                    color[v] = 1 - color[u]   // 불변식: 인접 정점은 반대 색
                    enqueue(queue, v)
                else if color[v] == color[u]:
                    return false               // 홀수 사이클 → 이분 불가

    return true

// 핵심 불변식:
//   BFS 진행 중, 이미 착색된 정점 v에 대해
//   color[v] != color[u] → 현재까지 이분 성질 보존
//   color[v] == color[u] → 홀수 사이클 존재 → false
```
