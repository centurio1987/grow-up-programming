# 최소 컷 (Minimum s-t Cut)

## 한 줄 요약

> 유량 네트워크의 정점 수 `n`, 방향 간선 목록 `edges`, 소스 `source`, 싱크 `sink` 를 받아, 소스와 싱크를 분리하는 간선 부분집합의 **용량 합이 최소인 컷**의 값을 `{ cut }` 형태로 반환한다.

## 스토리

보안팀이 데이터센터 네트워크를 점검한다. 외부 침입자(소스)가 내부 핵심 서버(싱크)에 접근하지 못하도록 일부 통신 링크를 차단하려 한다. 각 링크마다 차단 비용(용량)이 다르다.

목표는 침입자가 핵심 서버에 도달하는 모든 경로를 막으면서 **차단 비용의 합을 최소로** 줄이는 것이다. 어떤 링크를 골라 끊어야 가장 적은 비용으로 완벽하게 막을 수 있을까?

침입자와 서버가 처음부터 연결된 경로가 없다면 아무 링크도 끊을 필요가 없다.

## 함수 인터페이스

```ts
export function minCut(
  n: number,
  edges: [number, number, number][],
  source: number,
  sink: number,
): { cut: number };
```

- `n` — 정점의 개수. 정점 번호는 $0 \ldots n-1$.
- `edges` — 방향 가중치 간선 목록. 각 원소 `[u, v, c]` 는 정점 `u` 에서 `v` 로 향하는 용량 `c` 의 간선이다.
- `source` — 소스 정점 번호.
- `sink` — 싱크 정점 번호.
- 반환 — `{ cut }`. 최소 s-t 컷의 용량 합.

## 제약 조건

- $2 \leq n \leq 500$
- $0 \leq E \leq 10^4$ (여기서 $E$ 는 간선의 수)
- $0 \leq c(u, v) \leq 10^6$
- $0 \leq \text{source},\, \text{sink} < n,\;\; \text{source} \neq \text{sink}$
- 정점 번호는 $0 \ldots n-1$.
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

유량 네트워크 $G = (V, E, c)$ 와 소스 $s$, 싱크 $t$ 에 대해, **s-t 컷** $(S, T)$ 은 다음 조건을 만족하는 정점 분할이다.

$$S \cup T = V,\; S \cap T = \emptyset,\; s \in S,\; t \in T$$

컷 용량:

$$c(S, T) = \sum_{\substack{(u, v) \in E \\ u \in S,\, v \in T}} c(u, v)$$

최소 컷:

$$\text{minCut}(G, s, t) = \min_{(S, T) \text{ s-t cut}} c(S, T)$$

- 소스에서 싱크로 향하는 경로가 없으면 최소 컷 용량은 `0` 이다.
- 컷은 $S \to T$ 방향 간선만 포함하며, $T \to S$ 방향 간선은 포함하지 않는다.
- 평행 간선은 각 간선의 용량을 개별적으로 합산한다.
- 용량 `0` 인 간선도 컷 용량 계산에서 개수만큼 더해진다(값은 `0`이므로 합계에 영향 없음).

## 예시

```ts
minCut(
  4,
  [[0, 1, 3], [0, 2, 2], [1, 2, 1], [1, 3, 2], [2, 3, 3]],
  0, 3,
); // { cut: 5 } — 소스에서 싱크까지 흘릴 수 있는 최대 유량과 같음

minCut(3, [[0, 1, 10], [1, 2, 5]], 0, 2); // { cut: 5 } — 직렬 경로, 병목 용량이 곧 최소 컷

minCut(2, [[0, 1, 10]], 0, 1); // { cut: 10 } — 소스와 싱크를 잇는 단일 간선 하나가 유일한 컷

minCut(3, [[0, 1, 10]], 0, 2); // { cut: 0 } — 소스에서 싱크로 가는 경로 없음, 끊을 간선 없음

minCut(2, [], 0, 1); // { cut: 0 } — 간선 없음

minCut(4, [[0, 1, 100], [1, 2, 100], [2, 3, 1]], 0, 3); // { cut: 1 } — 싱크에 가장 가까운 병목 간선이 최소 컷
```
