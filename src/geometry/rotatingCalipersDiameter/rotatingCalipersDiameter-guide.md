# rotatingCalipersDiameter 해설 — 회전 캘리퍼스

## 성능 목표 예측

| 제약 항목 | 값 |
|-----------|-----|
| 점 수 $n$ | $2 \leq n \leq 10^5$ |
| 좌표 범위 | $-10^9 \leq x, y \leq 10^9$ (정수 권장) |

**naive 접근의 시간복잡도**

가장 단순한 접근은 모든 점 쌍의 거리를 비교하는 브루트 포스다.

$$T_{\text{naive}}(n) = \binom{n}{2} = O(n^2)$$

$n = 10^5$에서 $5 \times 10^9$ 비교가 필요하므로 제한 초과다. 개선 방향은 "지름이 반드시 특정한 점들에서만 발생한다"는 구조적 관찰을 이용해 탐색 공간을 줄이는 것이다.

**목표 복잡도**: $O(n \log n)$ — 볼록 껍질 구성 $O(n \log n)$ + 회전 캘리퍼스 $O(h) \leq O(n)$.

**공간 복잡도**: $O(n)$ — 볼록 껍질 배열이 최대 $n$개의 점을 보관.

**메모리 트레이드오프**: 볼록 껍질 배열 외에 추가 자료구조는 불필요하다. 두 포인터 $i, j$만으로 $O(1)$ 추가 공간.

---

## 목표 함수

```typescript
function rotatingCalipersDiameter(points: Point[]): number
```

| 파라미터 | 타입 | 의미 | 제약 |
|----------|------|------|------|
| `points` | `Point[]` | 2차원 점 배열 | $2 \leq n \leq 10^5$, 정수 좌표 권장 |

**반환값**: 가장 먼 두 점 사이의 제곱 거리($\sqrt{}$ 없이, $\geq 0$). 정수 좌표이면 정확한 정수 결과.

**엣지케이스**:
1. **$n = 2$**: 두 점의 제곱 거리를 직접 반환.
2. **모든 점 공선**: 볼록 껍질이 두 끝점만 가짐. 제곱 거리 반환.
3. **볼록 껍질 크기 $h = 1$**: 모든 점이 동일 → 제곱 거리 0 반환.
4. **정수 좌표**: 제곱 거리가 정수이므로 부동소수 오차 없음. `sqrt` 미적용이 이 때문.

## 핵심 아이디어

**핵심 아이디어**: "지름은 항상 볼록 껍질 위의 두 점에서 나온다 — 두 평행 지지선을 함께 회전시키면 모든 후보 쌍을 $O(h)$에 열거할 수 있다."

임의 내부 점은 지름에 기여하지 못하므로 먼저 볼록 껍질을 구한다. 그 다음 양쪽에서 껍질을 동시에 접하는 두 평행 지지선을 각도 0에서 $\pi$까지 회전시키면, 지름 후보인 antipodal pair가 순서대로 등장한다. 두 포인터가 각자의 지지선을 담당하며, 어느 쪽 변의 각도가 더 작은지 외적 부호로 결정해 그쪽 포인터만 전진시킨다.

**풀이 구조**
1. 볼록 껍질을 반시계 방향으로 구성한다.
2. 초기 antipodal 포인터 $j$를 찾는다(변 $h_0 \to h_1$에서 가장 먼 점).
3. 두 포인터 $i$, $j$로 현재 쌍의 제곱 거리를 기록하고 `max_dist2`를 갱신한다.
4. 두 변 벡터의 외적 부호로 $i$ 또는 $j$ 중 하나(또는 둘 다)를 전진시킨다.
5. 포인터 전진 합이 $k$에 도달하면 종료하고 `max_dist2`를 반환한다.

**조건**: 입력 점이 2개 이상이어야 하며, 볼록 껍질이 반시계 방향으로 정렬되어야 한다. 정수 좌표를 사용하면 `sqrt` 없이 정확한 정수 결과를 얻는다.

**대표 예시**: 레이더 범위 최대 거리 측정
$n$개의 기지국 좌표 중 가장 멀리 떨어진 두 기지국 간 거리를 구하는 문제다. 볼록 껍질 구성에 $O(n \log n)$, 캘리퍼스 회전에 $O(h)$면 충분하며, 브루트 포스 $O(n^2)$과 달리 점 수가 늘어나도 빠르게 동작한다.

**언제 쓰나**
"점 집합의 지름(가장 먼 두 점 거리)을 구하라"는 문제에서 $n$이 커서 $O(n^2)$이 느릴 때 사용한다. 볼록 껍질이 이미 구해진 상황이라면 캘리퍼스 단계만 $O(h)$로 처리할 수 있어 더욱 효율적이다.

### 원형 아이디어와 naive 접근

"지름"을 구하는 가장 단순한 시도는 모든 점 쌍 $O(n^2)$ 비교다. 개선을 위한 첫 번째 관찰은 "지름은 항상 볼록 껍질 위의 두 점에서 발생한다"는 것이다. 내부 점은 어떤 껍질 위 점보다 다른 점들에 더 가깝기 때문이다. 볼록 껍질을 먼저 구하면 $h$개의 점만 고려해도 된다. 그러나 껍질 크기가 $O(n)$인 경우 $O(h^2)$ 브루트 포스도 충분하지 않다. 폭발 지점은 "껍질 위의 점들 사이에서도 어떤 쌍이 지름인지 판단하는 기준이 없다"는 것이다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1**: 볼록 다각형의 지름은 항상 **antipodal pair** — 두 평행 지지선(supporting lines)에 각각 접하는 꼭짓점 쌍 — 에서 발생한다. 볼록 다각형에서 antipodal pair는 $O(h)$개뿐이다.
- **관찰 2**: 두 평행 지지선을 동시에 "회전"시키면(각도를 $0$에서 $\pi$까지), 모든 antipodal pair를 순서대로 방문할 수 있다. 두 포인터 $i, j$가 각각의 지지선을 나타낸다.
- **관찰 3**: 회전 단계에서 "어느 포인터를 먼저 전진시킬지"는 두 변의 외적 부호로 결정된다. 외적이 어느 변의 각도가 더 작은지 알려주기 때문이다.

### 관찰을 형식화: 상태/구조 정의

**볼록 다각형 $\mathcal{H}$**: 반시계 방향으로 정렬된 껍질 꼭짓점 $h_0, h_1, \ldots, h_{k-1}$.

**Antipodal pair**: 두 꼭짓점 $h_i, h_j$가 "각각의 평행 지지선에 접한다"는 것은, $h_j$가 변 $\overrightarrow{h_i h_{i+1}}$에 대해 가장 먼 점임을 의미한다.

**두 포인터**: $i$ = 한쪽 캘리퍼스 포인터, $j$ = 반대쪽 캘리퍼스 포인터. 항상 $i < j$ (모듈러 기준)를 유지하며 반시계 방향으로 전진한다.

이 형태여야 하는 이유: 볼록 다각형의 정렬 특성에 의해 두 포인터가 단조 전진하면서 모든 antipodal pair를 빠짐없이, 중복 없이 방문할 수 있다. 무작위 탐색 없이 선형 시간으로 처리된다.

**상태**: `max_dist2` = 지금까지 발견한 최대 제곱 거리. 단조 비감소.

### 점화식 또는 핵심 연산

**캘리퍼스 전진 방향 결정 (유도)**:

현재 캘리퍼스가 포인터 $i$, $j$에 있다. 두 변 벡터를 정의한다.

$$\vec{v}_i = h_{(i+1)\%k} - h_i, \quad \vec{v}_j = h_{(j+1)\%k} - h_j$$

두 벡터의 외적(z 성분):

$$c = \vec{v}_{i,x}\,\vec{v}_{j,y} - \vec{v}_{i,y}\,\vec{v}_{j,x}$$

- $c > 0$: $\vec{v}_i$의 각도가 $\vec{v}_j$보다 크다. $j$를 먼저 전진시켜야 지지선이 평행 상태에 더 가까워진다.
- $c < 0$: $\vec{v}_j$의 각도가 크다. $i$를 전진.
- $c = 0$: 두 변이 평행. 두 포인터 모두 전진. 이때 $(i, j+1)$과 $(i+1, j)$ 쌍도 추가로 확인.

**왜 외적이 전진 방향을 결정하는가**:

두 지지선이 현재 평행 상태에 있다고 하자. 다음 단계에서 지지선을 더 작은 각도만큼 회전시켜야 평행 상태를 유지할 수 있다. 작은 각도에 해당하는 변의 포인터를 전진시켜야 한다. 외적의 부호는 두 벡터 중 어느 것이 더 "앞에" 있는지(즉 더 작은 각도로 회전할지) 알려준다.

**제곱 거리 계산**:

$$\mathrm{dist}^2(a, b) = (a_x - b_x)^2 + (a_y - b_y)^2$$

**각 항의 의미**:
- $c > 0, j$ 전진: 오른쪽 캘리퍼스가 느리므로 전진.
- $c < 0, i$ 전진: 왼쪽 캘리퍼스가 느리므로 전진.
- $c = 0$: 두 변이 동시에 평행 → 네 꼭짓점 조합(antipodal rectangle)을 모두 검사.

### 정당성 — 왜 이것이 옳은가

**지름은 껍질 위**: 임의 내부 점 $p$와 껍질 위 점 $q$에 대해, $p$는 껍질 정점 $a, b$의 볼록 결합이므로 $\|p - q\| \leq \max(\|a - q\|, \|b - q\|)$이 된다. 따라서 내부 점은 지름에 기여하지 못한다.

**Antipodal pair가 지름**: 볼록 다각형의 지름 $(h_i, h_j)$에서 $h_j$는 변 $\overrightarrow{h_i h_{i+1}}$에 수직인 방향으로 가장 멀리 있다. 이는 $h_j$가 해당 방향의 지지선 위에 있음을 의미하고, 따라서 antipodal pair다.

**캘리퍼스가 모든 antipodal pair를 방문**: 두 포인터의 전진 합이 최대 $2k$번이다. 각 단계에서 최소 한 포인터가 전진하므로 루프는 $O(k)$에 종료된다. 볼록성에 의해 antipodal pair의 순서가 포인터 전진 순서와 일치함이 보장된다.

**까다로운 케이스**:
- **$c = 0$ (평행)**: 두 포인터가 동시에 전진할 때, $(i, j+1)$과 $(i+1, j)$ 쌍이 누락될 수 있다. 반드시 이 두 쌍도 `max_dist2`와 비교해야 한다.
- **볼록 껍질 크기 $k = 2$**: 선분 형태. $h_0$과 $h_1$만 비교하면 된다. 일반 루프로도 처리되지만, 초기 $j$ 설정이 $0$이 되는 케이스를 별도로 처리해야 한다.
- **초기 $j$ 설정**: $i = 0$일 때 $j$는 $h_0 \to h_1$ 변에 대해 가장 먼 점이어야 한다. 외적이 양수인 동안 $j$를 전진시켜 초기값을 찾는다.

### 구현 디테일과 최적화

- **제곱 거리 반환**: 부동소수 `sqrt`를 피하기 위해 제곱 거리를 반환. 정수 좌표에서 정확한 정수 결과.
- **포인터 종료 조건**: 두 포인터의 전진 합이 $k$를 초과하면 종료. 또는 한 포인터가 한 바퀴 돌면 종료.
- **convexHull 의존성**: 이 함수는 반시계 방향으로 정렬된 볼록 껍질을 입력으로 받는다. 볼록 껍질 구성에 `convexHull` 함수를 재활용한다.
- **함정**: 볼록 껍질이 이미 입력으로 주어졌을 때 다시 껍질을 구하면 $O(n \log n)$이 중복 발생한다. 이미 껍질이 있다면 캘리퍼스 단계만 수행한다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function dist2(a, b):
  dx = a.x - b.x; dy = a.y - b.y
  return dx*dx + dy*dy

function cross2d(v1, v2):
  // 두 2D 벡터의 외적 (z 성분)
  return v1.x * v2.y - v1.y * v2.x

function sub(a, b):
  return {x: a.x - b.x, y: a.y - b.y}

function rotatingCalipersDiameter(points):
  if len(points) == 2:
    return dist2(points[0], points[1])

  hull = convexHull(points)  // 반시계, 공선 제외
  k = len(hull)

  if k == 1: return 0
  if k == 2: return dist2(hull[0], hull[1])

  // 초기 j: hull[0]→hull[1] 변에 대해 가장 먼 antipodal 포인터
  j = 1
  while cross2d(sub(hull[1], hull[0]), sub(hull[(j+1)%k], hull[j])) > 0:
    j = (j + 1) % k

  max_dist2 = 0
  i = 0
  i0 = 0
  j0 = j  // 시작 j 기록

  // 불변식: i + (j - j0 + k) % k <= k (전진 합 <= k)
  while true:
    // 현재 antipodal pair 검사
    max_dist2 = max(max_dist2, dist2(hull[i], hull[j]))

    v_i = sub(hull[(i+1)%k], hull[i])
    v_j = sub(hull[(j+1)%k], hull[j])
    c = cross2d(v_i, v_j)

    if c > 0:
      j = (j + 1) % k  // j 전진
    elif c < 0:
      i = (i + 1) % k  // i 전진
    else:  // 평행: 추가 쌍도 확인
      max_dist2 = max(max_dist2,
                      dist2(hull[i], hull[(j+1)%k]),
                      dist2(hull[(i+1)%k], hull[j]))
      i = (i + 1) % k
      j = (j + 1) % k

    // 종료 조건: 한 바퀴 완료
    if i == i0 and j == j0: break
    // 또는 포인터 합이 k 이상이면 종료 (구현 방식에 따라 조정)

  return max_dist2
```

**핵심 불변식**: 두 포인터 $i$, $j$의 총 전진 횟수는 $2k$를 초과하지 않는다. 따라서 캘리퍼스 단계는 $O(k)$에 종료되고, 모든 antipodal pair를 정확히 한 번씩 방문한다. `max_dist2`는 단조 비감소이며, 최종 값이 점 집합의 지름의 제곱이다.
