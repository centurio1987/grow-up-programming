# crt — 중국인의 나머지 정리 해설

## 성능 목표 예측

| 항목 | 값 |
|------|-----|
| 입력 크기 | 연립식 $k$개, 모듈러 $m_i \leq 10^{18}$ |
| 시간 복잡도 | $O(k \log M)$ ($M = \text{lcm}(m_1, \ldots, m_k)$) |
| 공간 복잡도 | $O(1)$ (스트리밍 합병) |

**naive 접근의 한계.** $x \equiv r_1 \pmod{m_1}$에서 $x = r_1, r_1 + m_1, r_1 + 2m_1, \ldots$를 나열하며 나머지 조건을 일일이 확인하는 방법은 $O(M)$이다. $M = \text{lcm}(m_1, \ldots, m_k)$는 지수적으로 커질 수 있어 실용 불가능하다.

**목표 복잡도와 근거.** 두 합동식을 합치는 연산은 확장 유클리드 호제법 $O(\log \min(m_1, m_2))$가 1회 필요하다. $k$개 식을 쌍쌍이 합치면 총 $k - 1$번의 확장 유클리드 호출이며, 각 모듈러 크기는 최대 $M$이므로 $O(k \log M)$이다.

**공간 트레이드오프.** 누적 해 $(x_\text{cur}, M_\text{cur})$만 유지하므로 $O(1)$ 추가 공간이다. 입력 배열 자체 외에 추가 구조가 필요 없다.

---

## 목표 함수

```ts
function crt(
  remainders: bigint[],
  moduli: bigint[],
): { x: bigint; M: bigint } | null
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `remainders` | 잔차 배열 $[r_1, \ldots, r_k]$ | bigint 배열 |
| `moduli` | 모듈러 배열 $[m_1, \ldots, m_k]$ | 모두 $\geq 1$ |

**반환값**: 해 $\{ x, M \}$ (단, $0 \leq x < M$, $M = \text{lcm}(m_1, \ldots, m_k)$) 또는 모순이면 `null`.

**엣지케이스**:
1. `k = 0` → `{ x: 0n, M: 1n }` (빈 연립식의 관례적 해)
2. `k = 1, r=[3], m=[7]` → `{ x: 3n, M: 7n }`
3. 서로소 조건 위반 (예: $x \equiv 1 \pmod{4}$, $x \equiv 2 \pmod{6}$) → `null`
4. `m_i = 1n` → $x \equiv r \pmod{1}$은 항상 성립하므로 해가 존재함

## 핵심 아이디어

**핵심 아이디어**: "여러 개의 나머지 조건을 하나씩 대수적으로 합쳐 단 하나의 합동식으로 압축한다"

연립 합동식 $x \equiv r_1 \pmod{m_1},\ x \equiv r_2 \pmod{m_2},\ \ldots$를 풀 때, 모든 후보를 나열하면 $O(M)$으로 $M = \text{lcm}(m_1, \ldots, m_k)$이 폭발적으로 커진다. 대신 두 합동식을 한 번에 하나씩 확장 유클리드 호제법으로 합치면 $O(\log m)$에 새로운 단일 합동식을 만들 수 있고, 이를 $k$번 반복하면 전체 연립식이 $O(k \log M)$에 풀린다.

**풀이 구조**
1. 초기 상태 $(x_\text{cur}, M_\text{cur}) = (r_1 \bmod m_1, m_1)$ 설정
2. 각 새로운 합동식 $(r_i, m_i)$에 대해 merge 연산 수행
3. merge: $x = r_1 + m_1 t$로 매개변수화 후 $t$에 대한 선형 합동식을 확장 유클리드로 풀기
4. 해가 없으면(조건 불일치) null 반환, 있으면 새로운 $(x_\text{new}, \text{lcm}(m_1, m_2))$ 반환

**조건**: 각 $m_i \geq 1$이어야 하며, 해가 존재하려면 $\gcd(m_i, m_j) \mid (r_i - r_j)$가 모든 쌍에서 성립해야 한다. $m_i$들이 쌍쌍이 서로소이면 항상 유일한 해가 존재한다.

**대표 예시**: $x \equiv 2 \pmod{3}$, $x \equiv 3 \pmod{5}$ 연립 풀기
$x = 2 + 3t$로 놓으면 $2 + 3t \equiv 3 \pmod{5}$, 즉 $3t \equiv 1 \pmod{5}$. 확장 유클리드로 $3^{-1} \equiv 2 \pmod{5}$을 구하면 $t \equiv 2 \pmod{5}$. 따라서 $x = 2 + 3 \cdot 2 = 8 \equiv 8 \pmod{15}$. 검증: $8 = 2 + 3 \cdot 2$, $8 = 3 + 5 \cdot 1$, 두 조건 모두 만족.

**언제 쓰나**
"여러 조건을 동시에 만족하는 정수 $x$를 구하라"는 형태의 문제, 특히 여러 주기가 겹쳐야 하는 스케줄링 문제나 암호학적 복호화 문제에서 사용한다.

### 원형 아이디어와 naive 접근

연립 합동식을 푸는 가장 직접적인 방법은 첫 번째 조건의 해 후보 $x = r_1, r_1 + m_1, r_1 + 2m_1, \ldots$를 나열하며 나머지 조건을 하나씩 확인하는 것이다. 이는 $O(M)$이고 $M$이 기하급수적으로 커질 수 있어 폭발한다. "두 합동식을 하나로 대수적으로 합칠 수 없을까?"라는 질문이 돌파구이다.

### 어떤 관찰이 돌파구가 되는가

- **핵심 관찰 1**: $x \equiv r_1 \pmod{m_1}$의 해를 $x = r_1 + m_1 t$로 매개변수화하면, 두 번째 조건 $x \equiv r_2 \pmod{m_2}$는 $t$에 대한 선형 합동식 $m_1 t \equiv r_2 - r_1 \pmod{m_2}$로 변환된다.
- **핵심 관찰 2**: 선형 합동식 $at \equiv b \pmod{m}$은 $\gcd(a, m) \mid b$일 때만 해를 가지며, 확장 유클리드 호제법으로 $O(\log m)$에 풀 수 있다.
- **핵심 관찰 3**: 두 합동식을 합치면 새로운 단일 합동식 $x \equiv x_\text{new} \pmod{\text{lcm}(m_1, m_2)}$가 된다. 이 과정을 $k$번 반복하면 전체 연립식이 풀린다.

### 관찰을 형식화: 상태/구조 정의

상태를 $(x_\text{cur}, M_\text{cur})$ 쌍으로 정의한다. 의미: "지금까지 처리한 조건들의 유일한 해는 $x \equiv x_\text{cur} \pmod{M_\text{cur}}$이다."

불변식: 매 합병 단계 직후, $x_\text{cur}$은 지금까지 처리한 모든 합동식을 만족하며, $0 \leq x_\text{cur} < M_\text{cur}$이다.

이 형태여야 하는 이유: 단일 합동식으로 압축된 상태를 유지해야 새 조건과 합칠 수 있기 때문이다. $k$번 합병 후 결과가 전체 연립식의 해가 된다.

### 점화식 또는 핵심 연산

두 합동식을 합치는 `merge` 연산:

$$\begin{cases} x \equiv r_1 \pmod{m_1} \\ x \equiv r_2 \pmod{m_2} \end{cases} \Longrightarrow x \equiv x_\text{new} \pmod{\text{lcm}(m_1, m_2)}$$

**유도**: $x = r_1 + m_1 t$로 놓으면 두 번째 조건에서:

$$r_1 + m_1 t \equiv r_2 \pmod{m_2}$$
$$m_1 t \equiv r_2 - r_1 \pmod{m_2}$$

$g = \gcd(m_1, m_2)$로 놓으면, 해가 존재하는 조건은 $g \mid (r_2 - r_1)$이다. 만족되면 양변을 $g$로 나눠:

$$\frac{m_1}{g} t \equiv \frac{r_2 - r_1}{g} \pmod{\frac{m_2}{g}}$$

$\gcd(m_1/g,\, m_2/g) = 1$이므로, 확장 유클리드로 $(m_1/g)$의 역원 $u$를 구해:

$$t_0 = \frac{r_2 - r_1}{g} \cdot u \bmod \frac{m_2}{g}$$

최종 해:

$$x_\text{new} = r_1 + m_1 t_0 \bmod \text{lcm}(m_1, m_2)$$

### 정당성 — 왜 이것이 옳은가

**해의 존재 조건**: $g \mid (r_2 - r_1)$이 필요충분조건이다. 위반 시 `null` 반환.

**유일성**: $x \equiv r_1 \pmod{m_1}$이고 $x \equiv r_2 \pmod{m_2}$인 해는 $\text{lcm}(m_1, m_2)$ 주기로 유일하다.

**귀납 정당성**: 기저 사례 $k = 1$에서 $\{ x = r_1 \bmod m_1, M = m_1 \}$이 자명하다. 귀납 가정으로 $i - 1$개 조건의 해 $(x_\text{cur}, M_\text{cur})$가 올바르다고 하면, `merge`를 통해 $i$번째 조건까지 포함한 해 $(x_\text{new}, M_\text{new})$가 올바르다.

**까다로운 케이스**: $r_i$가 $m_i$보다 클 수 있으므로, 합병 전에 $r_i = r_i \bmod m_i$로 정규화하는 것이 안전하다. $\text{lcm}$ 계산 시 $m_1 / g \times m_2$ 순서로 계산해 중간 오버플로를 방지한다.

### 구현 디테일과 최적화

- **$r_i$ 정규화**: 합병 전 $r_i \leftarrow ((r_i \bmod m_i) + m_i) \bmod m_i$로 음수 잔차를 처리한다.
- **lcm 오버플로 방지**: `lcm = (m1 / g) * m2` 순서로 계산한다. bigint이므로 수치 오버플로는 없으나 중간 값이 커지면 연산 비용이 증가한다.
- **extendedEuclidean 활용**: $\gcd$와 베주 계수를 동시에 구하는 확장 유클리드를 사용해 코드를 간결하게 유지한다.
- **함정**: $k = 0$인 빈 입력에 대해 `{ x: 0n, M: 1n }`을 반환하는 관례를 지켜야 한다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function merge(r1, m1, r2, m2):
    g = gcd(m1, m2)
    if (r2 - r1) % g ≠ 0: return null      // 모순: 해 없음
    m2g = m2 / g
    diff = ((r2 - r1) / g) % m2g
    // 불변식: (m1/g) * t ≡ diff (mod m2g), gcd(m1/g, m2g) = 1
    { x: inv, ... } = extendedEuclidean(m1 / g, m2g)
    t = (diff * inv % m2g + m2g) % m2g     // t >= 0 보장
    lcm = m1 / g * m2
    x_new = (r1 + m1 * t) % lcm            // 불변식: 0 <= x_new < lcm
    return { x: x_new, M: lcm }

function crt(remainders, moduli):
    k = remainders.length
    if k == 0: return { x: 0n, M: 1n }

    // 불변식: cur_x ≡ r_i (mod m_i) for all processed i, 0 <= cur_x < cur_M
    cur_x = ((remainders[0] % moduli[0]) + moduli[0]) % moduli[0]
    cur_M = moduli[0]

    for i = 1 to k-1:
        result = merge(cur_x, cur_M, remainders[i], moduli[i])
        if result == null: return null      // 모순 발견
        { x: cur_x, M: cur_M } = result    // 불변식 갱신

    return { x: cur_x, M: cur_M }
```
