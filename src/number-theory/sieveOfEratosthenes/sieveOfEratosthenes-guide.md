# sieveOfEratosthenes — 에라토스테네스의 체 해설

## 성능 목표 예측

| 항목 | 값 |
|------|-----|
| 입력 범위 | $n \leq 10^7$ (권장, 메모리 제약에 따라 확장 가능) |
| 시간 복잡도 | $O(n \log \log n)$ |
| 공간 복잡도 | $O(n)$ |

**naive 접근의 한계.** $n$ 이하 모든 수에 대해 시행 나눗셈(`isPrimeTrial`)을 적용하면 $O(n\sqrt{n})$이다. $n = 10^6$이면 $10^9$번의 연산이 필요해 수 초가 걸린다. 소수 목록을 한꺼번에 구해야 한다면 훨씬 나은 방법이 필요하다.

**목표 복잡도와 근거.** 에라토스테네스의 체는 각 소수 $p$에 대해 $p^2$부터 $n$까지 $p$ 간격으로 배수를 제거한다. 소수의 역수 합 $\sum_{p \leq n} 1/p \approx \log \log n$ (메르텐스 정리)이므로, 총 제거 연산 횟수는 $n \cdot \sum_{p \leq n} 1/p \approx n \log \log n$이다. $n = 10^7$이면 약 $3 \times 10^7$ 연산으로 완료된다.

**공간 트레이드오프.** 길이 $n + 1$의 불리언 배열이 $O(n)$ 공간을 차지한다. 비트 배열을 사용하면 $O(n/8)$로 줄일 수 있으나, TypeScript에서는 일반 배열로도 $n = 10^7$에서 약 10MB 수준이다.

## 목표 함수

```ts
function sieveOfEratosthenes(n: number): number[]
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `n` | 소수를 구할 상한 | 음이 아닌 정수 |

**반환값**: $n$ 이하의 모든 소수를 오름차순으로 담은 배열.

**엣지케이스**:
1. `n = 0` → `[]` ($0$ 이하의 소수는 없음)
2. `n = 1` → `[]` ($1$은 소수가 아님)
3. `n = 2` → `[2]` (최소 소수)
4. `n = 10` → `[2, 3, 5, 7]`

## 핵심 아이디어

**핵심 아이디어**: "소수를 발견하는 순간 그 배수를 한꺼번에 제거하면 각 수를 개별 판정하지 않아도 된다"

모든 수에 시행 나눗셈을 적용하면 $O(n\sqrt{n})$으로 $n = 10^6$이면 $10^9$ 연산이 필요하다. 에라토스테네스의 체는 반대로 접근한다. 소수 $p$가 확정되면 $p^2, p^2 + p, p^2 + 2p, \ldots$를 한 번의 스캔으로 모두 합성수로 표시한다. $p^2$ 미만의 배수들은 더 작은 소수가 이미 제거했으므로 중복 작업이 없다. 소수의 역수 합이 $\log \log n$으로 수렴해 전체 연산이 $O(n \log \log n)$에 완료된다.

**풀이 구조**
1. 길이 $n + 1$의 불리언 배열을 true로 초기화, 인덱스 0과 1은 false로 설정
2. $i = 2$부터 $i^2 \leq n$인 동안: `isPrime[i]`가 true이면 $j = i^2$에서 시작해 $j += i$로 배수 제거
3. 외부 루프 종료 후 true인 인덱스를 모아 반환

**조건**: $n \leq 10^7$ 범위에서 메모리와 속도가 모두 실용적이다. $n > 10^8$이 되면 메모리 부족이 생길 수 있어 세그먼트 체를 고려해야 한다.

**대표 예시**: $n = 10$ 이하 소수 구하기
초기 배열: `[F, F, T, T, T, T, T, T, T, T, T]`. $i = 2$: $j = 4, 6, 8, 10$ 제거. $i = 3$: $j = 9$ 제거. $i = 4$에서 $4^2 = 16 > 10$이므로 외부 루프 종료. true인 인덱스 수집: `[2, 3, 5, 7]`.

**언제 쓰나**
특정 범위($n \leq 10^7$) 안의 모든 소수 목록이 한꺼번에 필요하거나, 이후 소수 판정 쿼리가 반복적으로 발생할 때 전처리 용도로 사용한다. 단 한 번의 판정만 필요하다면 시행 나눗셈이나 밀러-라빈이 더 적합하다.

### 원형 아이디어와 naive 접근

$2$부터 $n$까지 각 수에 대해 시행 나눗셈으로 소수 여부를 판정하면 $O(n\sqrt{n})$이다. 문제는 이미 합성수임이 확실한 수에 대해서도 반복 작업을 한다는 것이다. "합성수를 발견하면 그 배수를 한꺼번에 제거할 수 없을까?"라는 발상이 에라토스테네스의 체이다.

### 어떤 관찰이 돌파구가 되는가

- **핵심 관찰 1**: 소수 $p$의 배수($2p, 3p, 4p, \ldots$)는 모두 합성수이다. $p$가 확정되는 순간, 그 배수들을 모두 "제거"할 수 있다.
- **핵심 관찰 2**: $p$의 배수 중 $p^2$ 미만인 것들($2p, 3p, \ldots, (p-1)p$)은 더 작은 소수의 배수이기도 하다. 따라서 $p^2$부터 시작해도 된다. 이 관찰 덕분에 외부 루프를 $\sqrt{n}$까지만 돌린다.
- **핵심 관찰 3**: 소수의 역수 합 $\sum_{p \leq n} 1/p$이 $\log \log n$으로 느리게 증가하므로, 전체 배수 제거 연산의 합이 $n \log \log n$으로 수렴한다.

### 관찰을 형식화: 상태/구조 정의

상태를 불리언 배열 $\text{isPrime}[0 \ldots n]$으로 정의한다. 초기에 모두 `true`로 설정하고, 인덱스 $0$과 $1$은 `false`로 둔다. 외부 루프 변수 $i$에 대한 불변식:

"$i$ 미만의 모든 소수 $p$에 대해, $p$의 모든 배수($\leq n$)는 $\text{isPrime}$에서 `false`로 표시되어 있다."

이 불변식이 유지되면서 $i > \sqrt{n}$이 되면, $\text{isPrime}[k] = \text{true}$인 $k$들이 정확히 $n$ 이하의 소수이다.

### 점화식 또는 핵심 연산

알고리즘 구조는 다음과 같다:

$$\text{isPrime}[j] \leftarrow \text{false} \quad \forall j \in \{i^2,\, i^2 + i,\, i^2 + 2i,\, \ldots\} \text{ when } \text{isPrime}[i] = \text{true}$$

**유도 과정**: 소수 $p$의 배수 집합은 $\{kp : k \geq 2\}$이다. $k < p$인 배수 $kp$는 $k$의 배수이기도 하고 $k < p$이므로, $k$가 처리될 때 이미 제거된다. 따라서 $p$가 처리될 때는 $k = p$에서 시작하는 $p^2$부터 제거하면 충분하다. 결과적으로 $i = 2$부터 $\lfloor \sqrt{n} \rfloor$까지만 외부 루프를 돌린다.

**왜 $i^2$ 이하는 이미 제거됐는가?** $j = k \cdot i$ ($k < i$)이면 $j$는 $k$의 배수이고, $k < i$이므로 $k$가 외부 루프에서 처리될 때 이미 $\text{isPrime}[j] = \text{false}$로 설정됐다.

### 정당성 — 왜 이것이 옳은가

**완전성 (소수가 누락되지 않음)**: 소수 $p \leq n$은 어떤 작은 소수의 배수도 아니므로 $\text{isPrime}[p]$는 끝까지 `true`로 유지된다.

**건전성 (합성수가 포함되지 않음)**: 합성수 $n = a \cdot b$ ($a \leq b$)에서 $a \leq \sqrt{n}$이고, $a$의 최소 소인수 $p \leq a$가 외부 루프에서 처리될 때 $n$이 $p$의 배수로 제거된다.

**까다로운 케이스**: $i = 2$일 때 $i^2 = 4$부터 시작한다. $n = 4$이면 $\text{isPrime}[4] = \text{false}$가 올바르게 설정된다. $n = 0, 1$이면 외부 루프가 실행되지 않고 빈 배열이 반환된다.

### 구현 디테일과 최적화

- **배열 초기화**: `new Array(n + 1).fill(true)`로 초기화하고 인덱스 0, 1을 `false`로 설정한다.
- **외부 루프 상한**: `i * i <= n` 조건으로 $\sqrt{n}$까지만 반복한다. 부동소수점 오차를 피하기 위해 `Math.sqrt`보다 `i * i <= n`이 안전하다.
- **내부 루프 시작**: `j = i * i`에서 시작해 `j += i`로 진행한다.
- **성능 개선**: 비트 배열(`Uint8Array` 또는 `Int32Array`)을 사용하면 캐시 지역성이 향상된다.
- **함정**: $n < 2$이면 소수가 없으므로 즉시 빈 배열을 반환해야 한다. `isPrime[0]`, `isPrime[1]`을 `false`로 설정하지 않으면 $0$과 $1$이 소수로 잘못 포함된다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function sieveOfEratosthenes(n):
    if n < 2: return []

    // 불변식 초기화: isPrime[k] = true for 2 <= k <= n
    isPrime = array[0..n] filled with true
    isPrime[0] = false; isPrime[1] = false

    i = 2
    // 불변식: i 미만 소수들의 모든 배수가 false로 표시됨
    while i * i <= n:
        if isPrime[i]:
            j = i * i
            // 불변식: j = i * (i + k) 형태, k 증가
            while j <= n:
                isPrime[j] = false
                j += i
        i += 1

    result = []
    for k = 2 to n:
        if isPrime[k]: result.push(k)
    return result
```

## 관련 알고리즘과 확장

### 선형 체 (Linear Sieve)

에라토스테네스의 체는 $O(n \log \log n)$이지만, 각 합성수가 여러 소수에 의해 중복 제거될 수 있다. 선형 체(Euler's Sieve)는 각 합성수를 정확히 한 번씩 제거해 $O(n)$을 달성한다. 각 수 $i$에 대해 최소 소인수 이하의 소수들로만 배수를 생성한다.

### 세그먼트 체 (Segmented Sieve)

$n$이 크면 ($n > 10^8$) 메모리가 부족할 수 있다. 세그먼트 체는 $\sqrt{n}$ 이하 소수를 에라토스테네스 체로 미리 구한 뒤, $[L, R]$ 구간씩 분할해 체질한다. 각 구간의 크기를 $\sqrt{n}$ 정도로 설정하면 메모리 $O(\sqrt{n})$으로 $n$ 이하 모든 소수를 구할 수 있다.

### 소수 계수 $\pi(n)$

소수 정리에 의해 $n$ 이하 소수의 개수는 $\pi(n) \approx n / \ln n$이다. 에라토스테네스의 체로 구한 소수 배열의 길이가 이 값에 수렴한다. $n = 10^6$이면 약 $78498$개, $n = 10^7$이면 약 $664579$개이다.

### 최솟값 소인수 전처리

`sieve` 배열을 불리언 대신 최솟값 소인수 배열(`lpf[]`)로 구성하면, 이후 임의의 $m \leq n$의 소인수 분해를 $O(\log m)$에 처리할 수 있다. `lpf[i] = i`로 초기화하고 배수를 제거할 때 `lpf[j] = i`로 설정한다.
