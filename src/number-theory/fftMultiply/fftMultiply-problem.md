# 다항식 곱셈 (FFT Multiply)

## 한 줄 요약

> 함수는 두 정수 계수 배열 `a`, `b`를 받아 두 다항식의 곱 계수 배열을 반환한다.

## 스토리

신호 처리 연구팀이 두 파형을 합성하는 시스템을 개발하고 있다. 각 파형은 계수 배열로 표현된 다항식이다. 두 파형을 합성하면 각 주파수 성분이 상호 작용하여 새로운 파형이 된다.

첫 번째 파형이 $n$개, 두 번째가 $m$개의 계수를 가지면 합성 결과는 $n + m - 1$개의 계수를 갖는다. 문제는 $n$, $m$이 수만 이상으로 커질 때 단순 이중 반복으로는 초당 수십억 번의 곱셈이 필요해 실시간 처리가 불가능하다는 점이다.

연구팀은 빠르게 작동하는 합성 함수가 필요하다. 입력 배열 중 하나라도 비어 있으면 합성 결과도 빈 배열이다.

## 함수 인터페이스

```ts
export function fftMultiply(a: number[], b: number[]): number[];
```

- `a` — 첫 번째 다항식의 계수 배열 ($a[i]$는 $x^i$의 계수)
- `b` — 두 번째 다항식의 계수 배열 ($b[j]$는 $x^j$의 계수)
- 반환 — 곱 다항식의 계수 배열, 길이 $n + m - 1$ (단, 둘 중 하나가 빈 배열이면 `[]`)

## 제약 조건

- `a`, `b`의 원소는 정수 (number)
- 결과 계수도 정수로 반올림 처리
- 빈 배열 입력 시 결과도 빈 배열
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

두 배열을 다음 두 다항식의 계수 표현으로 본다.

$$A(x) = \sum_{i=0}^{n-1} a_i x^i,\quad B(x) = \sum_{j=0}^{m-1} b_j x^j$$

곱 다항식 $C(x) = A(x) \cdot B(x)$의 $k$번째 계수는 다음과 같다.

$$c_k = \sum_{i+j=k} a_i \cdot b_j,\quad k = 0, 1, \ldots, n+m-2$$

- 결과 배열의 길이는 정확히 $n + m - 1$이다.
- 부동소수점 오차가 발생하면 가장 가까운 정수로 반올림한다.
- 한쪽 배열이 비어 있으면 빈 배열 `[]`을 반환한다.

## 예시

```ts
fftMultiply([1, 2], [1, 3]);
// [1, 5, 6]  — (1 + 2x)(1 + 3x) = 1 + 3x + 2x + 6x² = 1 + 5x + 6x²

fftMultiply([1, 0, 1], [1, 1]);
// [1, 1, 1, 1]  — (1 + x²)(1 + x) = 1 + x + x² + x³

fftMultiply([2, 3, 4], [5, 6]);
// [10, 27, 38, 24]  — c₀=2·5, c₁=3·5+2·6, c₂=4·5+3·6, c₃=4·6

fftMultiply([], [1, 2, 3]);
// []  — 빈 배열 입력

fftMultiply([1, -1], [1, 1]);
// [1, 0, -1]  — (1 - x)(1 + x) = 1 - x²
```
