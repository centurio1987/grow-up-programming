# 인접 접미사 최장 공통 접두사 배열

## 한 줄 요약

> `kasaiLcp(s, sa)`는 문자열과 그 접미사 배열을 받아, 사전순으로 인접한 두 접미사 사이의 최장 공통 접두사 길이를 담은 LCP 배열을 반환한다.

## 스토리

문서 검색 엔진 개발자 서아는 문서 안에서 반복되는 문자열 패턴의 길이를 빠르게 구해야 한다. 이미 사전순으로 정렬된 접미사 목록이 주어졌는데, 인접한 두 접미사가 얼마나 겹치는지 알아야 추가 분석이 가능하다.

접미사 쌍마다 처음부터 직접 비교하면 최악의 경우 전체가 $O(n^2)$으로 폭발한다. 서아에게는 선형 시간 안에 모든 인접 쌍의 공통 접두사 길이를 한꺼번에 구하는 방법이 필요하다.

## 함수 인터페이스

```ts
export function kasaiLcp(s: string, sa: number[]): number[];
```

- `s` — 원본 문자열
- `sa` — `s`의 접미사 배열 (사전순 정렬된 접미사의 시작 인덱스 배열, 길이 $n = |s|$)
- 반환 — LCP 배열 (길이 $n$). `lcp[i]`는 `sa[i]`번째 접미사와 `sa[i+1]`번째 접미사의 최장 공통 접두사 길이. 마지막 원소 `lcp[n-1]`는 항상 0.

## 제약 조건

- $1 \leq |s| \leq 10^5$
- `sa`는 `s`의 올바른 접미사 배열 (길이 $n$, 각 원소는 0-based 인덱스)
- 시간 복잡도: $O(n)$
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

문자열 $s$(길이 $n$)와 접미사 배열 $\text{SA}$가 주어질 때, 사전순으로 인접한 접미사 쌍의 최장 공통 접두사(LCP) 길이를 담은 배열을 반환한다.

$$\text{LCP}[i] = \mathrm{lcp}\bigl(s[\text{SA}[i] \ldots],\; s[\text{SA}[i+1] \ldots]\bigr) \quad (0 \leq i < n - 1)$$

$$\text{LCP}[n-1] = 0$$

여기서 $\mathrm{lcp}(x, y)$는 두 문자열의 가장 긴 공통 접두사 길이이다.

반환 배열의 길이는 `sa`와 동일하게 $n$이며, 마지막 원소는 Kasai 표준 표기에 따라 반드시 0이다.

## 예시

```ts
kasaiLcp("banana", [5, 3, 1, 0, 4, 2]);
// [1, 3, 0, 0, 2, 0]
// lcp("a",     "ana")   = 1  → lcp[0]
// lcp("ana",   "anana") = 3  → lcp[1]
// lcp("anana", "banana")= 0  → lcp[2]
// lcp("banana","na")    = 0  → lcp[3]
// lcp("na",    "nana")  = 2  → lcp[4]
// 마지막 원소              = 0  → lcp[5]

kasaiLcp("abc", [0, 1, 2]);
// [0, 0, 0]  — 인접 접미사 간 공통 접두사 없음

kasaiLcp("aaaa", [3, 2, 1, 0]);
// [1, 2, 3, 0]
// lcp("a","aa")   = 1
// lcp("aa","aaa") = 2
// lcp("aaa","aaaa")= 3
// 마지막 원소     = 0

kasaiLcp("a", [0]);
// [0]  — 단일 접미사, 인접 쌍 없음
```
