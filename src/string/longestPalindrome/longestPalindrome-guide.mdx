# longestPalindrome 해설

## 성능 목표 예측

| 제약 | 값 |
|------|----|
| 문자열 길이 $n = |s|$ | $0 \leq n \leq 10^5$ |

**naive 접근의 한계:**
모든 부분 문자열을 검사한다.

```
best = ""
for i = 0 to n-1:
  for j = i to n-1:
    if isPalindrome(s[i..j]) and j-i+1 > len(best):
      best = s[i..j]
```

- 부분 문자열 수 $O(n^2)$, 각 검사 $O(n)$ → 전체 $O(n^3)$.

조금 개선: 각 중심에서 좌우로 확장하는 방식.

```
for center = 0 to 2n-1:   // 홀수/짝수 중심 분리
  expand from center while characters match
```

- 최악 `"aaaa...a"`: 중심 $n$개, 각 $O(n)$ 확장 → $O(n^2)$.
- $n = 10^5$이면 $10^{10}$ → 시간 초과.

낭비의 원인: 중심 $i$에서 확장할 때 얻은 "회문 범위" 정보를 다음 중심 $i+1$에서 활용하지 않는다.

**목표 복잡도:**

| 항목 | 목표 | 근거 |
|------|------|------|
| 시간 | $O(n)$ | 오른쪽 경계 $r$이 단조 증가, 총 확장 횟수 ≤ $2n$ |
| 공간 | $O(n)$ | 변환 문자열 $T$ + 반지름 배열 $p$ |

## 목표 함수

```ts
function longestPalindrome(s: string): string
```

**파라미터 표:**

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `s` | 입력 문자열 | $0 \leq |s| \leq 10^5$ |

**반환값:** `s`의 부분 문자열 중 가장 긴 회문. 동률이면 어느 것이나 반환 가능.

**엣지케이스:**

| 케이스 | 반환값 |
|--------|--------|
| `s = ""` | `""` |
| `s = "a"` | `"a"` |
| `s = "ab"` (모든 문자 다름) | `"a"` 또는 `"b"` (길이 1) |
| `s = "aaaa"` | `"aaaa"` |
| `s = "racecar"` | `"racecar"` |

## 핵심 아이디어

**핵심 아이디어**: "이미 확인한 회문 범위 안의 새 중심은, 대칭 위치의 반지름을 공짜로 재사용할 수 있다."

중심마다 양쪽으로 확장하는 방식은 직관적이지만 최악 O(n²)이다. Manacher 알고리즘은 지금까지 발견한 회문 중 가장 오른쪽으로 뻗은 것의 범위를 추적하다가, 새 중심이 그 범위 안에 들어오면 대칭점의 반지름을 초기값으로 재활용한다. 오른쪽 경계는 단조 증가하므로 실제 확장은 전체 통틀어 O(n)번만 일어난다.

**풀이 구조**
1. 문자 사이에 `#`을 삽입해 모든 회문을 홀수 길이로 통일한다.
2. 각 중심 i에 대해, 현재 오른쪽 경계 r과 중심 c를 유지한다.
3. i < r이면 대칭점 mirror = 2c - i의 반지름과 r - i 중 작은 값을 초기 반지름으로 쓴다.
4. 초기 반지름을 넘어서 실제로 확장이 가능한지 직접 비교한다.
5. r이 갱신되면 c와 r을 업데이트한다.

**조건**: 단일 문자열에서 가장 긴 회문 부분 문자열을 찾아야 할 때. 홀수/짝수 길이 회문을 모두 한 번에 처리해야 할 때.

**대표 예시**: `s = "abacaba"`에서 가장 긴 회문 찾기
`#`을 삽입하면 `#a#b#a#c#a#b#a#`가 된다. 중심 `c`를 처리하면서 이미 발견한 `"abacaba"` 전체 범위를 활용해 내부 중심들의 반지름 초기값을 설정한다. 실제 확장은 경계 부근에서만 발생하므로 전체 O(n)이다.

**언제 쓰나**
문자열에서 가장 긴 회문 부분 문자열을 O(n)에 구해야 할 때 사용한다. 입력 길이가 10만 이상이거나 시간 제한이 엄격한 경우 중심 확장 O(n²) 대신 Manacher를 선택한다.

### 원형 아이디어와 naive 접근

중심 확장 방식은 직관적이다. 회문은 중심을 기준으로 좌우 대칭이므로, 각 중심에서 양쪽으로 확장하며 일치하는 동안 반지름을 늘린다.

```
for i = 0 to n-1:
  // 홀수 길이: 중심 i
  r = 0
  while i-r-1 >= 0 and i+r+1 < n and s[i-r-1] == s[i+r+1]:
    r++
  // 짝수 길이: 중심 i~i+1 사이
  ...
```

중심 $2n-1$개(홀수 $n$개 + 짝수 $n-1$개)를 모두 처리하되, 각 중심에서 최대 $n$ 확장이 필요할 수 있다. 중심 간 정보 공유가 없어 $O(n^2)$이다.

### 어떤 관찰이 돌파구가 되는가

- **이미 확인된 회문 범위 안의 새 중심은 대칭점의 반지름을 초기값으로 사용할 수 있다.** 현재까지 발견한 회문 중 오른쪽 끝이 가장 먼 것의 중심을 $c$, 오른쪽 경계를 $r$이라 하자. 새 중심 $i < r$이면, $i$의 대칭점 $\text{mirror} = 2c - i$의 반지름 $p[\text{mirror}]$가 이미 계산되어 있다. $[l, r]$은 회문이므로, $i$에서의 반지름은 $\min(r - i, p[\text{mirror}])$부터 시작할 수 있다.
- **홀수/짝수 길이 회문을 더미 문자로 통합하면 단일 처리가 가능하다.** `"#"` 삽입으로 변환하면 모든 회문이 홀수 길이가 되어 중심 기반 처리만 하면 된다.
- **오른쪽 경계 $r$은 단조 증가한다.** 새로 확장이 일어날 때만 $r$이 증가하고, $r$은 최대 $2n+1$까지 증가할 수 있으므로 총 확장 횟수는 $O(n)$이다.

### 관찰을 형식화: 상태/구조 정의

**문자열 변환:**
$s = s_0 s_1 \ldots s_{n-1}$을 $T = \# s_0 \# s_1 \# \ldots \# s_{n-1} \#$로 변환한다. $|T| = 2n+1$.

이 형태가 필요한 이유: 원래 문자열에서 짝수 길이 회문(예: `"abba"`)은 중심이 두 문자 사이에 있어 홀수 길이 회문과 처리 방식이 다르다. `"#"` 삽입 후에는 짝수 길이 회문의 중심도 `"#"` 위치로 표현되어, 모든 회문을 홀수 길이로 통일할 수 있다.

원본 인덱스와 변환 인덱스 대응: 원본 $s[j]$는 $T[2j+1]$에 해당한다.

**상태 변수:**
- $p[i]$: $T$에서 중심 $i$의 최대 회문 반지름 ($T[i-p[i] \ldots i+p[i]]$가 회문)
- $c$: 현재까지 발견한 회문 중 오른쪽 끝이 가장 먼 것의 중심
- $r$: 그 오른쪽 경계 ($= c + p[c]$)

### 점화식 또는 핵심 연산

중심 $i$에서의 반지름 초기값:

$$p[i]_{\text{init}} = \begin{cases}
\min(r - i,\ p[2c - i]) & \text{if } i < r \\
0 & \text{if } i \geq r
\end{cases}$$

- $\min(r - i, p[2c-i])$의 의미:
  - $p[2c-i]$: 대칭점에서의 반지름 (재사용 가능한 하한)
  - $r - i$: 현재 회문 범위의 오른쪽 여유 공간 (이를 초과하면 범위 밖이므로 확인 필요)
  - 둘 중 작은 값이 안전한 초기값

이후 실제 확장:
$$\text{while } T[i - p[i] - 1] = T[i + p[i] + 1]: \quad p[i] \mathrel{+}= 1$$

$r$ 갱신:
$$\text{if } i + p[i] > r: \quad c = i,\ r = i + p[i]$$

**역변환:**
$T$에서 최대 $p[i]$를 찾은 뒤 원본 문자열의 시작 위치와 길이:

$$\text{start} = \frac{i - p[i]}{2}, \quad \text{length} = p[i]$$

### 정당성 — 왜 이것이 옳은가

불변식: 루프 내 항상 $T[c - r \ldots c + r]$은 회문이고, $r$은 단조 증가한다.

**재사용의 정당성:** $[l, r] = [c - p[c], c + p[c]]$가 회문이므로, $i$와 $\text{mirror} = 2c - i$는 $c$에 대해 대칭이다. 만약 $p[\text{mirror}] < r - i$이면, $i$ 주변 반지름도 정확히 $p[\text{mirror}]$이다(범위를 벗어나지 않으므로). $p[\text{mirror}] \geq r - i$이면 $r - i$까지는 보장되지만, $r$ 밖은 직접 확인해야 한다.

**$O(n)$의 정당성:** 확장 루프(`while`)는 $r$이 증가할 때만 실행된다. $r$이 증가한 횟수의 합 ≤ $|T| = 2n+1$이므로 총 확장 횟수는 $O(n)$이다. $i < r$인 경우에는 확장 없이 초기값을 그대로 사용하거나 $r$ 밖에서만 확장이 발생한다.

까다로운 케이스: $s = ""$이면 $T = "\#"$이고 $p[0] = 0$, length $= 0$ → 빈 문자열 반환. $s = "a"$이면 $T = "\#a\#"$이고 $p[1] = 1$, start $= 0$, length $= 1$ → `"a"` 반환.

### 구현 디테일과 최적화

- **경계 처리:** 확장 루프에서 `i - p[i] - 1 >= 0 and i + p[i] + 1 < |T|` 조건을 반드시 확인해야 한다. 없으면 배열 범위 초과 오류.
- **`"^"`와 `"$"` 센티넬 추가 방법:** $T$의 양 끝에 서로 다른 특수 문자(예: `"^#a#b#$"`)를 추가하면 경계 조건을 자동으로 처리할 수 있다. 센티넬은 절대 일치하지 않으므로 while 루프에서 범위 초과 검사가 불필요해진다.
- **역변환 주의:** start $= (i - p[i]) / 2$는 정수 나눗셈이다. 변환 문자열 $T$의 인덱스 $i$는 항상 짝수이거나(원본 `"#"`) 홀수(원본 문자)이며, $i - p[i]$는 항상 짝수이므로 정확히 나누어진다.
- **흔한 함정:** `i < r`이 아닌 `i <= r`로 조건을 쓰면 경계에서 이미 확인된 문자를 재확인하지 않아도 되는 최적화를 놓치거나 잘못된 초기값을 사용하게 된다.

## 수도 코드와 Activity Diagram

### 의사코드

```
transform(s):
  T = "#"
  for ch in s:
    T += ch + "#"
  return T                          // 불변식: |T| = 2n+1, s[i] <-> T[2i+1]

longestPalindrome(s):
  if s is empty: return ""
  T = transform(s)
  n2 = len(T)                       // = 2n+1
  p = array of length n2, all 0    // p[i] = 중심 i의 최대 반지름
  c = 0, r = 0                      // 불변식: T[c-r..c+r]은 회문, r 단조 증가
  best_c = 0                        // 최대 p[i]인 중심

  for i = 0 to n2-1:
    // 재사용: i가 현재 회문 범위 안이면 대칭점 활용
    if i < r:
      mirror = 2*c - i
      p[i] = min(r - i, p[mirror])  // 안전한 하한

    // 실제 확장
    left  = i - p[i] - 1
    right = i + p[i] + 1
    while left >= 0 and right < n2 and T[left] == T[right]:
      p[i]++; left--; right++       // 불변식: T[i-p[i]..i+p[i]] 회문

    // c, r 갱신
    if i + p[i] > r:
      c = i
      r = i + p[i]

    if p[i] > p[best_c]:
      best_c = i

  // 역변환
  start = (best_c - p[best_c]) / 2
  return s[start .. start + p[best_c] - 1]
```

```mermaid
    K -- No --> M{p[i] > p[best_c]?}
    L --> M
    M -- Yes --> N[best_c = i]
    M -- No --> O[다음 i]
    N --> O
    O --> D
    D -- 완료 --> P["start = (best_c - p[best_c]) / 2"]
    P --> Z[s[start .. start+p[best_c]-1] 반환]
```

**핵심 불변식:** 루프 내 항상 $T[c - r \ldots c + r]$은 회문이고, $r$은 단조 증가한다. 총 확장 횟수 ≤ $2n+1$이므로 전체 $O(n)$이 보장된다.
