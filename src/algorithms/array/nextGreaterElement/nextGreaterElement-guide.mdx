# nextGreaterElement — 가장 가까운 큰 수 (NGE)

## 성능 목표 예측

| 항목 | 값 |
|------|-----|
| 입력 크기 | $1 \leq N \leq 100{,}000$ |
| 원소 범위 | $-10^9 \leq nums[i] \leq 10^9$ |

**naive 접근의 문제점**: 각 위치 $i$에 대해 오른쪽을 선형 탐색해 첫 번째 큰 원소를 찾으면 $O(N^2)$이다. $N = 10^5$에서 $10^{10}$ 연산으로 시간 초과가 발생한다.

**목표 복잡도**: 시간 $O(N)$, 공간 $O(N)$. 각 원소가 스택에 최대 한 번 push되고 한 번 pop된다.

**공간 복잡도**: 스택과 결과 배열 각각 $O(N)$이다.

---

## 목표 함수

```ts
function nextGreaterElement(nums: number[]): number[]
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `nums` | 정수 배열 | $1 \leq N \leq 100{,}000$, $-10^9 \leq nums[i] \leq 10^9$ |

**반환값**: 각 위치 $i$의 NGE 값. 오른쪽에 더 큰 원소가 없으면 $-1$.

**엣지케이스**:

| 입력 | 기대 출력 | 이유 |
|------|-----------|------|
| `[5]` | `[-1]` | 단일 원소 — 오른쪽 없음 |
| `[3, 2, 1]` | `[-1, -1, -1]` | 단조 감소 — 모두 NGE 없음 |
| `[1, 2, 3]` | `[2, 3, -1]` | 단조 증가 |
| `[2, 1, 2, 4, 3]` | `[4, 2, 4, -1, -1]` | 혼합 |

## 핵심 아이디어

**핵심 아이디어**: "아직 NGE를 찾지 못한 인덱스들을 스택에 쌓아두고, 더 큰 값이 오는 순간 스택에서 꺼내 답을 기록한다."

각 위치의 NGE(Next Greater Element)를 구하기 위해 오른쪽을 매번 재탐색하면 O(N²)가 된다. 단조 감소 스택을 이용하면 새 원소가 스택 top보다 클 때 top을 팝하며 NGE를 기록하는 방식으로, 각 원소가 정확히 한 번씩 push/pop되어 전체 O(N)에 처리된다.

**풀이 구조**
1. `result = [-1] * N`, 빈 스택으로 초기화한다.
2. i=0..N-1을 순회하며, 스택 top의 값이 nums[i]보다 작은 동안 j=pop(), `result[j] = nums[i]`로 NGE를 기록한다.
3. 현재 인덱스 i를 스택에 push한다.
4. 순회 종료 후 스택에 남은 인덱스는 NGE 없음(-1로 이미 초기화).
5. `result`를 반환한다.

**조건**: 오른쪽 방향의 첫 번째 엄격히 큰 값을 찾는 문제. 같은 값은 NGE가 아님(조건이 `<`이지 `<=`가 아님).

**대표 예시**: `nums=[2,1,2,4,3]`
순회하면서 4가 등장할 때 2(인덱스 2), 1(인덱스 1), 2(인덱스 0)가 순서대로 팝되며 result = [4, 2, 4, -1, -1]이 완성된다.

**언제 쓰나**
"각 원소에서 오른쪽(또는 왼쪽)으로 가장 가까운 더 큰(또는 더 작은) 값을 찾는" 패턴에서 사용한다. 주가 온도 분석, 히스토그램 경계 계산, 순환 배열 NGE 등 다양한 변형 문제에 적용된다.

### 원형 아이디어와 naive 접근

각 위치 $i$에 대해 오른쪽을 선형 탐색한다.

```
for i from 0 to N-1:
    result[i] = -1
    for j from i+1 to N-1:
        if nums[j] > nums[i]:
            result[i] = nums[j]
            break
```

최악 단조 감소 배열에서 $O(N^2)$이다. 중복 탐색의 원인: 위치 $i$의 NGE를 찾은 이후 같은 범위를 $i+1$에서 다시 탐색한다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1**: 왼쪽에서 오른쪽으로 순회하면서, 어떤 위치 $j$의 값이 스택에 쌓인 이전 위치 $i$의 값보다 크면 — $j$가 $i$의 NGE임을 즉시 알 수 있다.
- **관찰 2**: "아직 NGE를 찾지 못한 인덱스"를 스택에 모아두고, 현재 값이 스택 top의 값보다 크면 pop해 NGE를 기록한다. 이 과정이 전체 $O(N)$이다.
- **관찰 3**: 스택 내 인덱스들의 `nums` 값을 단조 감소로 유지하면, 스택 top이 현재 값보다 작은지만 확인하면 된다.

### 관찰을 형식화: 상태/구조 정의

단조 감소 스택(인덱스 저장)을 유지한다.

$$\text{스택 내 인덱스} \; s_0 < s_1 < \ldots < s_k \; \text{에 대해} \; nums[s_0] \geq nums[s_1] \geq \ldots \geq nums[s_k]$$

(스택 아래에서 위로 단조 감소)

**불변식**: 스택에 있는 인덱스 $j$는 "아직 오른쪽에서 NGE를 찾지 못한 상태"이다.

이 정의가 왜 이 형태여야 하는가: 현재 인덱스 $i$에서 $nums[i] > nums[j]$인 스택 top $j$를 팝할 때, $i$가 $j$의 NGE임을 보장한다. 스택이 단조 감소이므로 팝 조건($nums[\text{top}] < nums[i]$)을 만족하는 모든 원소가 연속적으로 팝된다. 단조 증가로 유지하면 이 연속성이 깨진다.

### 점화식 또는 핵심 연산

$i = 0, 1, \ldots, N-1$에 대해:

```
while stack not empty and nums[stack.top()] < nums[i]:
    j = stack.pop()
    result[j] = nums[i]    // i가 j의 NGE
stack.push(i)
```

결과 배열의 나머지 (스택에 남은 인덱스): 이미 $-1$로 초기화되어 있으므로 추가 처리 불필요.

- `nums[stack.top()] < nums[i]`: 스택 top의 NGE를 현재 인덱스가 제공 — 엄격히 큰 경우만 처리
- `stack.push(i)`: 현재 인덱스를 "아직 NGE를 못 찾은 후보"로 등록

### 정당성 — 왜 이것이 옳은가

임의의 인덱스 $j$가 팝될 때의 상황: 팝 조건 $nums[j] < nums[i]$에 의해 $nums[i] > nums[j]$이다. 또한 $j$와 $i$ 사이에 있는 모든 인덱스 $k$ $(j < k < i)$에 대해 $nums[k] \leq nums[j]$이다 — 그렇지 않으면 $k$가 순회할 때 $j$가 먼저 팝되었을 것이다. 따라서 $i$는 $j$의 오른쪽에 있는 엄격히 큰 원소 중 첫 번째, 즉 NGE임이 보장된다.

순회 종료 후 스택에 남은 인덱스들: 그 이후에 더 큰 원소가 없었으므로 NGE가 없다. $result$를 $-1$로 초기화했으므로 별도 처리가 필요 없다.

같은 값의 원소: $nums[j] = nums[i]$이면 팝하지 않는다 (조건이 `<`이지 `<=`가 아님). 이는 올바르다 — 문제에서 "엄격히 큰" NGE를 요구하므로 같은 값은 NGE가 아니다.

### 구현 디테일과 최적화

- **결과 배열 초기화**: `result = [-1] * N`으로 초기화하면 순회 후 스택에 남은 원소를 별도로 처리할 필요가 없다.
- **스택에 인덱스를 저장**: 값 대신 인덱스를 저장해야 `result[j]`에 접근할 수 있다.
- **함정**: 팝 조건을 `<=`로 바꾸면 같은 값을 NGE로 잘못 처리한다.
- **함정**: 순회 후 스택 처리를 잊으면 스택에 남은 인덱스의 NGE가 $-1$이 아닌 쓰레기 값이 될 수 있다. 초기화로 해결한다.
- **오른쪽에서 왼쪽 순회 방법**: 결과는 동일하지만 로직이 달라진다. 스택에 값(또는 인덱스)을 저장하고, 오른쪽에서 왼쪽으로 순회하며 현재 원소보다 큰 스택 top이 NGE가 된다. 왼쪽에서 오른쪽 방향이 더 직관적이다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function nextGreaterElement(nums):
    N      ← len(nums)
    result ← [-1] * N         // 불변식: result[j]는 j의 NGE (아직 못 찾으면 -1)
    stack  ← 빈 스택 (인덱스)  // 불변식: 스택 내 nums 값은 단조 감소 (아래→위)

    for i from 0 to N-1:
        while stack 비어있지 않고 nums[stack.top()] < nums[i]:
            j ← stack.pop()
            result[j] ← nums[i]   // i가 j의 첫 번째 더 큰 원소
        stack.push(i)              // 단조 감소 유지

    return result
```

### Activity Diagram

```mermaid
flowchart TD
    A([시작]) --> B["result = [-1 x N] / stack = []"]
    B --> C{i < N?}
    C -- No --> H([return result])
    C -- Yes --> D{stack 비어있지 않고 nums[stack.top()] < nums[i]?}
    D -- Yes --> E["j = stack.pop() / result[j] = nums[i]"]
    E --> D
    D -- No --> F["stack.push(i)"]
    F --> G[i++]
    G --> C
```

**핵심 불변식**: 임의 시점에 스택 내 인덱스들의 `nums` 값은 아래에서 위로 단조 비증가이며, 스택에 있는 인덱스 $j$는 "현재까지 순회된 원소 중 $j$의 NGE가 아직 발견되지 않은 상태"이다. $i$가 팝한 $j$에 대해 $nums[i]$가 $j$의 정확한 NGE임이 보장된다.

## 복잡도 분석 심화

| 접근 방식 | 시간 | 공간 | 비고 |
|-----------|------|------|------|
| 이중 루프 (naive) | $O(N^2)$ | $O(1)$ | $N=10^5$에서 불가 |
| 단조 감소 스택 | $O(N)$ | $O(N)$ | 최적 |

**변형 1 — 순환 배열에서 NGE**: 배열이 순환한다고 가정하면 (마지막 원소 다음이 첫 원소) 배열을 두 번 이어 붙여 $2N$ 길이로 처리하거나, 인덱스를 $\% N$으로 처리한다. 스택에 최대 $N$개 원소가 쌓이므로 두 번째 순회에서 스택이 비워지면 종료한다.

**변형 2 — 다음 큰 수의 거리**: 값이 아닌 "거리"를 반환하는 변형. 팝할 때 `result[j] = i - j` (인덱스 차이)를 저장한다. 스택에 인덱스를 저장하는 것이 중요한 이유가 이 변형에서 더욱 명확해진다.

**변형 3 — 이전 큰 수 (Previous Greater Element, PGE)**: NGE를 오른쪽이 아닌 왼쪽에서 찾는 문제. 순회 방향을 바꾸거나, 왼쪽에서 오른쪽으로 순회하면서 "이미 스택에 있는 값 중 현재보다 큰 것"을 조회하는 방식으로 처리한다.
