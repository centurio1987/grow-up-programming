# Articulation Points (단절점)

## 한 줄 요약

> 함수는 정점 수 `n`과 무향 간선 목록을 받아, 제거 시 그래프가 분리되는 정점 번호 목록을 오름차순으로 반환한다.

## 스토리

도시 간 도로망을 관리하는 팀이 있다. 어느 날 팀장은 "교통 인프라에서 하나의 교차로가 폐쇄됐을 때 일부 지역이 완전히 고립될 수 있다"는 보고를 받았다. 그런 교차로를 미리 파악해 두면 재난 대비 계획을 세울 수 있다.

팀은 도시의 도로망 지도를 들고 앉아 교차로마다 "이 교차로가 없어지면 어떻게 될까?"를 따져보기 시작했다. 어떤 교차로는 없어져도 우회로가 있어 모두 연결되지만, 어떤 교차로는 없어지는 순간 동네 전체가 뚝 끊겼다.

팀이 찾고 싶은 것은 바로 그런 '없어지면 연결이 끊기는' 교차로 목록이다.

## 함수 인터페이스

```ts
export function articulationPoints(
  n: number,
  edges: [number, number][],
): number[];
```

- `n` — 정점 수. 정점 번호는 $0 \ldots n-1$
- `edges` — 무향 간선 목록. 각 원소 `[u, v]`는 정점 `u`와 `v` 사이의 무향 간선
- 반환 — 단절점 정점 번호를 오름차순으로 정렬한 배열. 단절점이 없으면 빈 배열

## 제약 조건

- $1 \leq V \leq 10^{5}$ (여기서 $V = n$)
- $0 \leq E \leq 10^{5}$ (여기서 $E$는 `edges`의 길이)
- 무방향 그래프
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

무향 그래프에서 **단절점(articulation point, cut vertex)** 을 모두 찾는다.

정점 $v$가 단절점이라는 것은, $v$를 그래프에서 제거했을 때 연결 성분의 개수가 증가한다는 의미이다.

반환 형식은 단절점 정점 번호를 오름차순으로 정렬한 배열이다.

경계 동작:
- 정점이 1개이면 단절점은 없다.
- 간선이 없는 그래프는 단절점이 없다.
- 자기 루프 `[v, v]`는 단절점 판별에 영향을 주지 않는다.
- 그래프가 이미 분리되어 있어도 각 성분 내에서 단절점을 찾는다.

## 예시

```ts
// 체인 그래프: 0 - 1 - 2
// 정점 1을 제거하면 {0}과 {2}로 분리 → 단절점
articulationPoints(3, [[0, 1], [1, 2]]); // [1]

// 두 사이클이 한 정점으로 연결
// {0,1,2} 사이클 + {2,3,4} 사이클, 2가 공유
// 정점 2를 제거하면 두 사이클이 분리 → 단절점
articulationPoints(5, [[0,1],[1,2],[2,0],[2,3],[3,4],[4,2]]); // [2]

// 삼각형 그래프: 0-1-2-0
// 어떤 정점을 제거해도 나머지 두 정점은 연결 유지
articulationPoints(3, [[0, 1], [1, 2], [2, 0]]); // []

// 간선 없는 그래프 — 어떤 정점을 제거해도 분리되지 않음
articulationPoints(5, []); // []

// V=1 — 단절점 없음
articulationPoints(1, []); // []
```
