# Strongly Connected Components (강한 연결 요소, SCC)

## 한 줄 요약

> 함수는 정점 수 `n`과 유향 간선 목록을 받아, 서로 양방향으로 도달 가능한 정점 그룹(SCC)을 정렬된 배열의 배열로 반환한다.

## 스토리

온라인 게임 개발팀의 분석가 서준은 플레이어 간 아이템 교환 네트워크를 들여다보고 있다. 아이템 교환은 항상 일방통행이 아니라, A가 B에게 주고 B가 다시 A에게 돌려줄 수도 있다. 서준은 "자원이 계속 순환하는 플레이어 그룹"을 찾고 싶다.

두 플레이어가 같은 순환 그룹에 속하려면 A에서 B로 가는 거래 경로도 존재하고 B에서 A로 돌아오는 경로도 존재해야 한다. 이런 그룹은 자원이 외부로 빠져나가지 않고 내부에서 계속 돌기 때문에, 불법 작업장 탐지에 활용할 수 있다.

거래가 단방향으로만 흐르는 플레이어들은 그 자신이 크기 1짜리 독립 그룹이 된다.

## 함수 인터페이스

```ts
export function stronglyConnectedComponents(
  n: number,
  edges: [number, number][],
): number[][];
```

- `n` — 정점 수. 정점 번호는 $0 \ldots n-1$
- `edges` — 유향 간선 목록. 각 원소 `[u, v]`는 $u \to v$ 방향의 간선
- 반환 — SCC 목록. 각 SCC 내 정점들은 오름차순 정렬. SCC들은 각 SCC의 최솟값 기준 오름차순 정렬

## 제약 조건

- $1 \leq V \leq 10^{5}$ (여기서 $V = n$)
- $0 \leq E \leq 10^{5}$ (여기서 $E$는 `edges`의 길이)
- 방향 그래프
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

유향 그래프에서 **강한 연결 요소(Strongly Connected Component, SCC)** 를 모두 찾는다.

두 정점 $u, v$가 같은 SCC에 속한다는 것은 $u$에서 $v$로 가는 경로와 $v$에서 $u$로 가는 경로가 모두 존재한다는 것과 동치이다.

반환 형식:
- 각 SCC 내부의 정점들은 오름차순 정렬한다.
- SCC 자체는 각 SCC의 최소 정점 번호 기준으로 오름차순 정렬한다.

경계 동작:
- 간선이 없으면 각 정점이 크기 1의 독립 SCC이다.
- 자기 루프 `[v, v]`가 있어도 해당 정점이 다른 정점과 양방향으로 연결되어 있지 않으면 여전히 혼자 SCC이다.
- 분리된 두 성분 각각에 내부 사이클이 있으면 각각 독립 SCC이다.
- 모든 정점은 정확히 하나의 SCC에 속한다.

## 예시

```ts
// 사이클: 0→1→2→0 — 세 정점 모두 하나의 SCC
stronglyConnectedComponents(3, [[0,1],[1,2],[2,0]]); // [[0,1,2]]

// 선형 DAG: 0→1→2→3 — 각 정점이 독립 SCC
stronglyConnectedComponents(4, [[0,1],[1,2],[2,3]]); // [[0],[1],[2],[3]]

// {0,1,2} SCC와 {3,4} SCC, 단방향 연결 2→3
stronglyConnectedComponents(5, [[0,1],[1,2],[2,0],[2,3],[3,4],[4,3]]); // [[0,1,2],[3,4]]

// 간선 없음 — 모두 독립
stronglyConnectedComponents(4, []); // [[0],[1],[2],[3]]

// V=1
stronglyConnectedComponents(1, []); // [[0]]
```
