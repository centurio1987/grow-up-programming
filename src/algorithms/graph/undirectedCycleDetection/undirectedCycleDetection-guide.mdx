# undirectedCycleDetection 해설

## 성능 목표 예측

| 제약 | 값 |
|------|----|
| 정점 수 $V$ | $1 \leq V \leq 10^5$ |
| 간선 수 $E$ | $0 \leq E \leq 10^5$ |
| 정점 번호 | $0 \ldots n-1$ |
| 그래프 종류 | 무향 |

**naive 접근의 비용**: 모든 단순 경로(simple path)를 열거해 사이클이 있는지 확인한다.
경로 수 $O(V!)$ → 완전히 불가.

DFS로 각 정점에서 출발해 부모가 아닌 이미 방문한 정점을 만나면 사이클로 판정한다.
$O(V + E)$ — 이미 최선이다.

**목표**: Union-Find로 $O(V + E)$ 안에 사이클 유무를 판별한다.
$V + E \leq 2 \times 10^5$으로 단일 패스(간선 목록 한 번 처리)로 충분.

**공간 트레이드오프**: Union-Find는 parent/rank 배열 $O(V)$만 사용. 인접 리스트도 필요 없다.

---

## 목표 함수

```ts
function undirectedCycleDetection(n: number, edges: [number, number][]): boolean
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `n` | 정점 수 | $1 \leq n \leq 10^5$ |
| `edges` | 무향 간선 목록 `[u, v]` | $0 \leq E \leq 10^5$ |
| 반환 | 사이클이 존재하면 `true`, 없으면 `false` | — |

**엣지케이스**

1. **자기 루프** $(v, v)$: `find(v) == find(v)` → 즉시 `true`.
2. **다중 간선** $(u, v)$가 두 번 이상: 첫 번째에서 union, 두 번째에서 같은 집합 발견 → `true`.
3. **간선 없음**: 아무 union도 없으므로 모든 find가 서로 다름 → `false`.
4. **단일 정점** ($n = 1$, 간선 없음): `false`.

## 핵심 아이디어

**핵심 아이디어**: "간선을 하나씩 추가할 때 양 끝이 이미 같은 집합이면, 그것이 곧 사이클이다."

DFS로 부모 방향이 아닌 방문된 정점을 탐지해도 $O(V+E)$이지만, Union-Find를 쓰면 더 단순하게 해결할 수 있다. 무향 그래프에서 사이클이 없는 상태는 spanning forest(트리)이고, 새 간선을 추가할 때 양 끝점이 이미 같은 연결 성분이면 그 간선이 사이클을 만든다. Union-Find는 이 "같은 집합 여부"를 경로 압축과 rank 결합으로 거의 $O(1)$ 수준에서 판별한다.

**풀이 구조**
1. `parent[v] = v`, `rank[v] = 0`으로 Union-Find를 초기화한다.
2. 간선 목록을 순서대로 순회하며 각 간선 `(u, v)`에 대해 `find(u)`와 `find(v)`를 구한다.
3. 두 루트가 같으면 사이클 → `true`를 즉시 반환한다.
4. 다르면 `union(find(u), find(v))`으로 두 집합을 병합한다.
5. 모든 간선을 처리해도 사이클이 없으면 `false`를 반환한다.

**조건**: 무향 그래프 전용. 유향 그래프에서는 Union-Find가 방향을 무시해 오탐이 발생하므로 DFS 3색 방법을 써야 한다.

**대표 예시**: 도로 네트워크에서 불필요한 순환 도로 탐지
도시들을 정점, 도로를 간선으로 모델링하고 도로를 하나씩 추가할 때, 양 도시가 이미 다른 경로로 연결돼 있다면 새 도로는 순환을 만든다. Union-Find가 이를 즉시 감지한다.

**언제 쓰나**
무향 그래프에서 사이클 존재 여부를 확인하거나, Kruskal MST 알고리즘에서 간선 추가 시 사이클을 방지할 때 사용한다. 인접 리스트 없이 간선 목록만으로 동작하므로 공간도 $O(V)$로 절약된다.

### 원형 아이디어와 naive 접근

사이클이란 서로 다른 정점들로 이루어진 닫힌 경로이다. 무향 그래프에서 사이클을 탐지하는 가장 직관적인 방법:

```
for each vertex v:
  DFS from v (with parent tracking)
  if we encounter a visited vertex that is not the parent:
    return true
return false
```

이 방법은 $O(V + E)$이고 정확하다. 하지만 인접 리스트를 구성해야 하고($O(V + E)$ 공간), DFS 콜 스택이 $O(V)$ 깊이까지 쌓일 수 있다.

더 단순한 관찰: 무향 그래프에서 사이클이 존재한다는 것은, 트리(spanning forest)에 "여분의 간선"이 있다는 것과 동치이다. 트리는 $V$개 정점에 $V-1$개 간선을 가지므로, $V$번째 간선이 추가될 때 반드시 사이클이 생긴다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1**: 무향 그래프에서 사이클이 없다면 spanning forest이다. Spanning forest는 $V$개 정점, 최대 $V-1$개 간선을 가진다. 따라서 간선 $E$개를 처리하면서 $V$번째 간선이 "같은 연결 성분 내"에서 추가될 때 사이클이 형성된다.
- **관찰 2**: "같은 연결 성분에 속하는지"를 효율적으로 관리하는 자료구조가 Union-Find(Disjoint Set Union)이다. `find(u) == find(v)`이면 $u$와 $v$는 이미 같은 성분에 있고, 이때 $(u, v)$ 간선은 사이클을 형성한다.
- **관찰 3**: 인접 리스트 구성이 필요 없다. 간선 목록을 한 번 순서대로 처리하면서 Union-Find만 갱신하면 된다. 공간이 $O(V)$로 줄어든다.

### 관찰을 형식화: 상태/구조 정의

Union-Find 자료구조:

$$\text{parent}[v] = \begin{cases} v & v \text{가 자신의 집합의 루트} \\ \text{다른 정점} & \text{경로 압축 후 부모 또는 루트 방향} \end{cases}$$

$$\text{rank}[v] = v \text{가 루트일 때 해당 집합의 트리 깊이 상한}$$

find 연산 (경로 압축):

$$\text{find}(v) = \begin{cases} v & \text{if } \text{parent}[v] = v \\ \text{find}(\text{parent}[v]) & \text{otherwise, 동시에 } \text{parent}[v] \leftarrow \text{root} \end{cases}$$

왜 경로 압축인가? 매 find가 루트까지 경로를 평탄화(flatten)하여, 이후 find가 $O(1)$에 수렴하게 한다. 이로써 $E$번의 find 전체 비용이 $O(E \cdot \alpha(V)) \approx O(E)$가 된다.

사이클 판정:

$$\text{find}(u) = \text{find}(v) \iff (u, v) \text{는 사이클을 형성}$$

### 점화식 또는 핵심 연산

각 간선 $(u, v)$를 처리할 때:

1. $r_u \leftarrow \text{find}(u)$, $r_v \leftarrow \text{find}(v)$
2. $r_u = r_v$이면 → **사이클**: `true` 반환
3. $r_u \neq r_v$이면 → `union(r_u, r_v)`: 두 집합을 병합

union by rank:

$$\text{union}(r_u, r_v): \begin{cases}
\text{parent}[r_v] \leftarrow r_u & \text{if } \text{rank}[r_u] > \text{rank}[r_v] \\
\text{parent}[r_u] \leftarrow r_v & \text{if } \text{rank}[r_u] < \text{rank}[r_v] \\
\text{parent}[r_v] \leftarrow r_u,\; \text{rank}[r_u]++ & \text{if } \text{rank}[r_u] = \text{rank}[r_v]
\end{cases}$$

각 항의 의미:
- `find(u) == find(v)`: $u$와 $v$가 이미 같은 연결 성분. 새 간선은 여분 = 사이클.
- union by rank: 낮은 랭크 트리를 높은 랭크 트리에 붙여 트리 높이가 $O(\log V)$ 이하를 유지. 경로 압축과 결합해 $O(\alpha(V))$ 상각.

### 정당성 — 왜 이것이 옳은가

Union-Find는 정점들의 동치 관계(연결됨)를 추적한다. 간선 $(u, v)$를 추가할 때:
- $\text{find}(u) \neq \text{find}(v)$: $u$와 $v$가 서로 다른 연결 성분에 있다. 이 간선을 추가하면 두 성분이 병합되고, 새로운 트리가 생성된다. 사이클이 아님.
- $\text{find}(u) = \text{find}(v)$: $u$와 $v$가 같은 연결 성분에 있다. 이미 $u$에서 $v$로 가는 경로가 존재한다. 이 간선을 추가하면 $u \to \cdots \to v \to u$ 형태의 사이클이 생성된다.

이 판별은 양방향(무향)에서 올바르다. 유향 그래프에서는 방향이 있어 $u$에서 $v$로 경로가 있더라도 $v$에서 $u$로의 경로가 없을 수 있으므로 Union-Find를 쓸 수 없다.

까다로운 케이스: 자기 루프 $(v, v)$. `find(v) == find(v)`는 항상 참 → 즉시 `true` 반환.

까다로운 케이스: 다중 간선. 첫 번째 $(u, v)$에서 union. 두 번째 $(u, v)$에서 `find(u) == find(v)` → `true`.

### 구현 디테일과 최적화

**경로 압축 함정**: 재귀 경로 압축에서 `parent[v] = find(parent[v])`로 루트를 직접 가리키게 한다. 단순히 `return parent[parent[v]]`("two-pass halving")도 동일한 상각 복잡도를 가진다.

**rank vs size**: union by rank는 트리 높이를 최소화하고, union by size는 더 구현하기 쉽다. 둘 다 $O(\log V)$ 높이 보장이며, 경로 압축과 결합하면 $O(\alpha(V))$.

**인접 리스트 불필요**: 이 문제에서는 DFS가 아니라 Union-Find를 쓰므로, 간선 목록을 직접 순회한다. 인접 리스트를 만들 필요가 없어 공간이 $O(V)$로 줄어든다.

**DFS 대안의 장단점**: DFS(부모 추적)는 인접 리스트 $O(V + E)$ 공간이 필요하고, 재귀 깊이가 $V = 10^5$에 달할 수 있다. Union-Find는 $O(V)$ 공간, 스택 사용 없음으로 더 안전하다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function find(parent, v):
  if parent[v] != v:
    parent[v] = find(parent, parent[v])  -- 경로 압축: 루트를 직접 가리킴
    -- 불변식: find(v) 이후 parent[v]는 루트를 직접 가리킴
  return parent[v]

function union(parent, rank, u, v):
  -- u, v는 이미 find로 구한 루트
  if rank[u] < rank[v]:
    swap(u, v)                  -- 높은 rank를 루트로 선택
  parent[v] = u                 -- 낮은 rank 트리를 높은 rank 트리에 붙임
  if rank[u] == rank[v]:
    rank[u]++                   -- 높이가 같을 때만 rank 증가
  -- 불변식: 합쳐진 집합의 트리 높이 ≤ log₂(집합 크기)

function undirectedCycleDetection(n, edges):
  parent[0..n-1] = [0, 1, ..., n-1]  -- 각 정점이 자신의 루트
  rank[0..n-1]   = 0

  for [u, v] in edges:
    ru = find(parent, u)
    rv = find(parent, v)
    if ru == rv:                -- 같은 집합 = 사이클 형성
      return true
    union(parent, rank, ru, rv) -- 불변식: union 후 두 정점은 같은 집합

  return false                  -- 모든 간선이 서로 다른 성분을 연결 = 사이클 없음
```

**핵심 불변식:**
`find(v)`가 같은 루트를 반환하는 두 정점은 같은 연결 성분이다. union 이전에 두 루트가 같으면 해당 간선은 사이클을 형성한다. 이 불변식은 union 전후로 항상 유지된다.
