# Bridges in Graph (다리 검출)

## 한 줄 요약

> 함수는 정점 수 `n`과 무향 간선 목록을 받아, 제거 시 그래프가 분리되는 간선 목록을 정규화·정렬해 반환한다.

## 스토리

전력망 설계 회사의 엔지니어 민준은 변전소들을 연결하는 송전선 네트워크를 분석하고 있다. 어떤 송전선은 끊어져도 우회 경로가 있어 전력 공급이 유지되지만, 어떤 송전선은 끊어지는 순간 특정 구역이 완전히 정전된다.

민준은 그런 '핵심 송전선'을 미리 파악해 이중화(백업 라인 설치) 우선순위를 정하고 싶다. 네트워크 지도에서 하나씩 선을 지워보며 고립되는 구역이 생기는지 확인하는 작업이 필요하다.

이 분석을 자동화하면 수천 개의 노선 중에서도 한번에 취약한 구간을 뽑아낼 수 있다.

## 함수 인터페이스

```ts
export function bridgesInGraph(
  n: number,
  edges: [number, number][],
): [number, number][];
```

- `n` — 정점 수. 정점 번호는 $0 \ldots n-1$
- `edges` — 무향 간선 목록. 각 원소 `[u, v]`는 정점 `u`와 `v` 사이의 무향 간선
- 반환 — 다리 간선 목록. 각 원소 `[u, v]`는 $u < v$로 정규화. 전체는 사전순(첫 원소 → 둘째 원소) 정렬. 다리가 없으면 빈 배열

## 제약 조건

- $1 \leq V \leq 10^{5}$ (여기서 $V = n$)
- $0 \leq E \leq 10^{5}$ (여기서 $E$는 `edges`의 길이)
- 무방향 그래프
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

무향 그래프에서 **다리(bridge, cut edge)** 를 모두 찾는다.

간선 $(u, v)$가 다리라는 것은, 해당 간선을 제거했을 때 연결 성분의 개수가 증가한다는 의미이다.

반환 형식:
- 각 다리 `[u, v]`는 $u < v$가 되도록 정규화한다.
- 전체 배열은 사전순(첫 원소 기준, 같으면 둘째 원소 기준)으로 정렬한다.

경계 동작:
- 정점이 1개이면 다리가 없다.
- 간선이 없으면 다리가 없다.
- 같은 정점 쌍에 대한 중복 간선(평행 간선)이 있으면 그 간선은 다리가 아니다 — 한 쪽을 제거해도 나머지로 연결이 유지된다.
- 그래프가 이미 분리되어 있어도 각 성분 내에서 다리를 찾는다.

## 예시

```ts
// 체인: 0-1-2-3 — 모든 간선이 다리
bridgesInGraph(4, [[0,1],[1,2],[2,3]]); // [[0,1],[1,2],[2,3]]

// 사이클: 0-1-2-3-0 — 어떤 간선을 제거해도 연결 유지
bridgesInGraph(4, [[0,1],[1,2],[2,3],[3,0]]); // []

// 두 사이클이 단일 간선으로 연결
// {0,1,2} 사이클 + {3,4,5} 사이클, 2-3이 유일한 연결
// 2-3을 제거하면 두 덩어리로 분리
bridgesInGraph(6, [[0,1],[1,2],[2,0],[2,3],[3,4],[4,5],[5,3]]); // [[2,3]]

// 중복 간선: [0,1]이 두 번 — 하나를 제거해도 나머지로 연결 유지
bridgesInGraph(2, [[0,1],[0,1]]); // []

// 단일 간선 — 그 간선이 다리
bridgesInGraph(2, [[1,0]]); // [[0,1]]
```
