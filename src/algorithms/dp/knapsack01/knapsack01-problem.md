# 0/1 배낭 문제

## 한 줄 요약

> 함수는 물건들의 무게 배열, 가치 배열, 배낭 최대 용량을 받아 담을 수 있는 최대 가치 합을 반환한다.

## 스토리

탐험가 민준은 오지 탐사를 앞두고 배낭을 꾸리고 있다. 각 장비는 고유한 무게와 탐사 효용 점수를 가지며, 배낭은 정해진 무게 한도를 넘길 수 없다. 같은 장비는 한 개뿐이라 가져가거나 두고 가거나 둘 중 하나다.

민준은 무게 제한 안에서 효용 점수의 합을 최대로 만들고 싶다. 장비가 많고 서로 조합이 복잡해지면서 머릿속으로는 도저히 최적을 찾을 수 없었다.

## 함수 인터페이스

```ts
export function knapsack01(
  weights: number[],
  values: number[],
  W: number,
): number;
```

- `weights` — 각 물건의 무게 목록
- `values` — 각 물건의 가치 목록 (`weights[i]`와 `values[i]`는 같은 물건을 나타냄)
- `W` — 배낭의 최대 적재 용량
- 반환 — 용량 `W` 이하로 담을 때 얻을 수 있는 최대 가치 합

## 제약 조건

- $1 \leq n \leq 100$ (물건 개수, `weights`와 `values`의 길이)
- `weights`와 `values`의 길이는 동일하다.
- $1 \leq weights[i], values[i] \leq 10^4$
- $1 \leq W \leq 10^4$
- 각 물건은 0번 또는 1번만 선택할 수 있다.
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

각 물건은 담거나 담지 않거나 둘 중 하나다. 동일한 물건을 두 번 이상 담을 수 없다.

선택한 물건들의 무게 합이 $W$를 초과하지 않는 범위에서 가치 합을 최대화한다.

`weights`와 `values`가 비어 있거나 $W = 0$이면 담을 수 있는 물건이 없으므로 $0$을 반환한다.

모든 물건의 무게가 $W$를 초과하면 $0$을 반환한다.

## 예시

```ts
knapsack01([1, 3, 4, 5], [1, 4, 5, 7], 7);
// 9 — 무게 3짜리(가치 4)와 무게 4짜리(가치 5)를 담으면 무게 합 7, 가치 합 9

knapsack01([3, 4], [4, 5], 4);
// 5 — 무게 3짜리는 W=4에 들어가고 무게 4짜리도 들어가지만, 단일 선택으로 가치 5가 최대

knapsack01([1, 2, 3], [10, 20, 30], 10);
// 60 — 무게 합 6 ≤ 10이므로 모두 담을 수 있고 가치 합 60

knapsack01([5], [10], 4);
// 0 — 유일한 물건이 배낭에 들어가지 않음

knapsack01([], [], 10);
// 0 — 물건이 없음
```
