# 주사위 합 확률

## 한 줄 요약

> 함수는 주사위 던지는 횟수 N과 임계값 K를 받아, 나온 눈의 합이 K 이상일 확률을 반환한다.

## 스토리

보드게임 제작사 기획자 은지는 새 게임의 난이도 균형을 맞추고 있다. 플레이어는 공정한 6면 주사위를 N번 굴려 나온 눈의 합을 점수로 받는데, "합이 K 이상이어야 특별 보너스를 획득한다"는 규칙을 검토 중이다.

은지는 각 N과 K 조합에 대해 보너스 획득 확률을 미리 표로 만들어야 한다. 시뮬레이션 없이 정확한 확률 값이 필요하며, N이 최대 1000까지 커질 수 있어 반복 계산을 효율적으로 처리해야 한다.

## 함수 인터페이스

```ts
export function expectedValueDp(N: number, K: number): number;
```

- `N` — 주사위를 던지는 횟수 ($1 \leq N \leq 1000$)
- `K` — 합이 이 값 이상이어야 하는 임계값 ($0 \leq K \leq 6N$)
- 반환 — 눈의 합이 $K$ 이상일 확률 ($[0, 1]$ 범위의 실수)

## 제약 조건

- $1 \leq N \leq 1000$
- $0 \leq K \leq 6N$
- 주사위는 $1$~$6$의 눈을 각각 $\tfrac{1}{6}$ 확률로 나타낸다.
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

주사위 $N$번의 눈의 합이 가질 수 있는 범위는 $N$ 이상 $6N$ 이하이다.

$K \leq N$이면 합이 항상 $K$ 이상이므로 확률은 $1.0$이다.

$K > 6N$이면 합이 절대 $K$ 이상이 될 수 없으므로 확률은 $0.0$이다.

$K = 0$이면 합은 항상 $0$ 이상이므로 확률은 $1.0$이다.

반환값은 부동소수점 수이며, 오차 $10^{-9}$ 이내이면 정답으로 인정한다.

## 예시

```ts
expectedValueDp(1, 1);
// 1.0 — 주사위를 한 번 굴리면 반드시 1 이상

expectedValueDp(1, 7);
// 0.0 — 한 번에 7 이상은 불가능

expectedValueDp(1, 4);
// 0.5 — 4, 5, 6이 나오는 경우: 3/6 = 0.5

expectedValueDp(2, 12);
// 약 0.02778 — 두 번 모두 6이 나와야 하므로 1/36

expectedValueDp(2, 7);
// 약 0.58333 — 두 번 던져 합이 7 이상인 경우의 수 21/36
```
