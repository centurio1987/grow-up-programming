# 외판원 순회 최소 비용

## 한 줄 요약

> 함수는 도시 간 거리 행렬을 받아, 모든 도시를 정확히 한 번씩 방문하고 출발 도시로 돌아오는 최소 비용을 반환한다.

## 스토리

배송 회사의 경로 최적화 담당 현수는 하루에 여러 거점을 모두 들르고 다시 출발지로 돌아오는 최단 경로를 찾아야 한다. 거점마다 방문 순서를 달리하면 총 이동 거리가 크게 달라진다.

현수가 다루는 거점은 최대 $20$개이며, 각 거점 쌍 사이의 이동 비용이 주어진다. 가능한 모든 방문 순서를 시도하면 경우의 수가 너무 많아 시간이 부족하므로 효율적인 방법이 필요하다.

## 함수 인터페이스

```ts
export function tspBitmask(dist: number[][]): number;
```

- `dist` — $n \times n$ 거리 행렬 (`dist[i][j]`는 도시 $i$에서 도시 $j$로 이동하는 비용)
- 반환 — 도시 $0$에서 출발하여 모든 도시를 정확히 한 번씩 방문하고 도시 $0$으로 돌아오는 최소 비용

## 제약 조건

- $1 \leq n \leq 20$ (도시 수, `dist`는 $n \times n$ 정방 행렬)
- $0 \leq dist[i][j] \leq 10^6$
- $dist[i][i] = 0$
- 방향이 있는 그래프이므로 $dist[i][j] \neq dist[j][i]$일 수 있다.
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

반드시 도시 $0$에서 출발하고 모든 $n$개의 도시를 정확히 한 번씩 방문한 뒤 도시 $0$으로 복귀해야 한다.

$n = 1$인 경우 다른 도시가 없으므로 비용은 $0$이다.

$n = 2$인 경우 0 → 1 → 0 경로밖에 없으므로 $dist[0][1] + dist[1][0]$을 반환한다.

## 예시

```ts
tspBitmask([[0]]);
// 0 — 도시가 하나뿐이므로 이동 없음

tspBitmask([[0, 5], [5, 0]]);
// 10 — 0 → 1 → 0: 5 + 5 = 10

tspBitmask([
  [0, 10, 15, 20],
  [10,  0, 35, 25],
  [15, 35,  0, 30],
  [20, 25, 30,  0],
]);
// 80 — 0 → 1 → 3 → 2 → 0: 10 + 25 + 30 + 15 = 80

tspBitmask([
  [0, 1, 10],
  [10, 0, 1],
  [1, 10, 0],
]);
// 3 — 0 → 1 → 2 → 0: 1 + 1 + 1 = 3 (비대칭 행렬에서 방향 중요)
```
