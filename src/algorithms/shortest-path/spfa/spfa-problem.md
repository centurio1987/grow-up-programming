# SPFA (Shortest Path Faster Algorithm)

## 한 줄 요약

> 함수는 음수 간선을 포함할 수 있는 그래프 (음수 사이클 없음)와 시작 정점을 받아 각 정점까지의 최단 거리 배열을 반환한다.

## 스토리

게임 개발자 준서는 플레이어가 맵을 이동할 때 최소 이동 비용을 실시간으로 계산하는 시스템을 만들고 있다. 맵에는 이동 비용을 줄여주는 부스터 구간이 있어 일부 경로의 비용이 음수다. 단, 부스터만 무한히 순환하며 비용을 무한정 줄이는 경로는 없도록 맵이 설계되어 있다.

준서는 음수 간선이 포함된 그래프에서도 효율적으로 최단 경로를 계산하고 싶다. 단순히 모든 간선을 반복해서 완화하는 방식은 너무 느리다.

핵심 아이디어는 거리가 실제로 갱신된 정점의 이웃만 다시 검사하는 것이다. 갱신이 없으면 그 이웃들은 더 이상 검사할 필요가 없으므로 불필요한 연산을 크게 줄일 수 있다.

## 함수 인터페이스

```ts
export function spfa(
  n: number,
  edges: [number, number, number][],
  src: number,
): number[];
```

- `n` — 정점 수. 정점 번호는 $0$부터 $n - 1$.
- `edges` — 방향 간선 목록. 각 원소는 $[u, v, w]$ 형태이며 가중치 $w$ (음수 허용)인 방향 간선 $u \to v$.
- `src` — 시작 정점 번호.
- 반환 — 길이 $n$인 배열. 인덱스 $v$에는 $\text{src} \to v$의 최단 거리. 도달할 수 없는 정점은 `Infinity`.

## 제약 조건

- $1 \leq V \leq 10^5$ (정점 수)
- $0 \leq E \leq 2 \cdot 10^5$ (간선 수)
- $-10^9 \leq w(u, v) \leq 10^9$ (가중치; 음수 허용)
- $0 \leq \text{src} < V$
- $\text{src}$에서 도달 가능한 음수 사이클은 존재하지 않는다
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

음수 가중치를 포함할 수 있는 방향 그래프 $G = (V, E)$가 주어진다. 시작 정점 $\text{src}$에서 모든 정점 $v$까지의 최단 거리 $d(\text{src}, v)$를 구하라.

$\text{src}$에서 도달 가능한 음수 사이클은 없다고 보장된다. 음수 사이클 검출이 필요한 경우에는 이 함수를 사용하지 않는다.

- $\text{src}$ 자신까지의 거리는 $0$이다.
- 도달할 수 없는 정점의 거리는 `Infinity`다.
- 동일한 두 정점 사이에 다중 간선이 있으면 모두 고려해야 한다.
- 거리가 갱신된 정점만 재처리한다. 한 번 대기열에 등록된 정점도 거리가 다시 갱신되면 재등록될 수 있다.

## 예시

```ts
// 음수 간선이 있지만 음수 사이클 없음
const edges: [number, number, number][] = [
  [0, 1, 4],
  [0, 2, 5],
  [1, 2, -3],
  [2, 3, 4],
];
spfa(4, edges, 0);
// [0, 4, 1, 5]
// — 0→2 최단: 0→1→2 = 1 (직접 간선 5보다 짧음)

// 도달 불가능한 정점은 Infinity
spfa(3, [[0, 1, 2]], 0);
// [0, 2, Infinity] — 정점 2로 가는 간선 없음

// 늦게 발견된 더 짧은 경로가 전파됨
spfa(3, [[0, 1, 10], [0, 2, 1], [2, 1, 1]], 0);
// [0, 2, 1]
// — 0→1: 먼저 10이 등록되지만 이후 0→2→1 = 2로 갱신됨

// 시작점이 중간 정점인 경우
spfa(4, [[0, 1, 1], [1, 2, 1], [2, 3, 1]], 1);
// [Infinity, 0, 1, 2] — src=1 기준; 정점 0은 도달 불가
```
