# Dijkstra 최단 경로

## 한 줄 요약

> 함수는 음수가 아닌 가중치의 방향 그래프와 시작 정점을 받아 각 정점까지의 최단 거리 배열을 반환한다.

## 스토리

물류 회사의 배송 경로 시스템을 개발하는 수빈은 창고에서 각 배송지까지의 최소 이동 비용을 계산해야 한다. 도로마다 통행료나 연료비 같은 이동 비용이 붙어 있고, 비용은 항상 0 이상이다.

수빈은 배송 기사들이 출발지에서 각 목적지까지 가장 저렴하게 이동할 수 있는 경로 비용을 미리 계산해 두고 싶다. 도로는 단방향인 경우도 있어서 A에서 B로 가는 비용과 B에서 A로 가는 비용이 다를 수 있다.

출발 창고 하나를 기준으로 나머지 모든 배송지까지의 최소 비용을 한 번에 구하면 배송 계획을 효율적으로 세울 수 있다.

## 함수 인터페이스

```ts
export function dijkstra(
  n: number,
  edges: [number, number, number][],
  src: number,
): number[];
```

- `n` — 정점 수. 정점 번호는 $0$부터 $n - 1$.
- `edges` — 방향 간선 목록. 각 원소는 $[u, v, w]$ 형태이며 가중치 $w \geq 0$인 방향 간선 $u \to v$.
- `src` — 시작 정점 번호.
- 반환 — 길이 $n$인 배열. 인덱스 $v$에는 $\text{src} \to v$의 최단 거리. 도달할 수 없는 정점은 `Infinity`.

## 제약 조건

- $1 \leq V \leq 10^5$ (정점 수)
- $0 \leq E \leq 2 \cdot 10^5$ (간선 수)
- $0 \leq w(u, v) \leq 10^9$ (가중치; 음수 없음)
- $0 \leq \text{src} < V$
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

가중치가 음수가 아닌 방향 그래프 $G = (V, E)$가 주어진다. 시작 정점 $\text{src}$에서 모든 정점 $v$까지의 최단 거리 $d(\text{src}, v)$를 구하라.

최단 거리는 경로 위 간선 가중치의 합이 최소인 경로 비용으로 정의된다:

$$d(\text{src}, v) = \min_{P \in \mathcal{P}(\text{src}, v)} \sum_{(u, w) \in P} w(u, w)$$

여기서 $\mathcal{P}(\text{src}, v)$는 $\text{src} \to v$ 경로의 집합이다.

- $\text{src}$ 자신까지의 거리는 $0$이다.
- 도달할 수 없는 정점의 거리는 `Infinity`다.
- 동일한 두 정점 사이에 다중 간선이 있으면 모두 고려해야 한다.
- 셀프 루프(자기 자신을 향하는 간선)는 거리 갱신에 영향을 주지 않는다.

## 예시

```ts
// 여러 경로 중 최단 경로 선택
const edges: [number, number, number][] = [
  [0, 1, 4],
  [0, 2, 1],
  [2, 1, 2],
  [1, 3, 1],
  [2, 3, 5],
];
dijkstra(4, edges, 0);
// [0, 3, 1, 4]
// — 0→1 최단: 0→2→1 = 3 (직접 간선 4보다 짧음)

// 도달 불가능한 정점은 Infinity
dijkstra(3, [[0, 1, 2]], 0);
// [0, 2, Infinity] — 정점 2로 가는 간선 없음

// 간선이 없을 때 시작점만 0
dijkstra(3, [], 1);
// [Infinity, 0, Infinity] — src만 0, 나머지 Infinity

// 방향 그래프: 역방향 간선은 사용 불가
dijkstra(2, [[0, 1, 5]], 1);
// [Infinity, 0] — 1→0 방향 간선이 없으므로 정점 0은 도달 불가
```
