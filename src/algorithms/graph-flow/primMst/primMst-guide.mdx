# primMst 해설

## 성능 목표 예측

| 항목 | 값 |
|------|----|
| 정점 V | $1 \leq V \leq 10^4$ |
| 간선 E | $0 \leq E \leq V(V-1)/2$ (밀집 그래프) |
| 가중치 w | $0 \leq w \leq 10^9$ |

**naive 접근의 문제점:**
모든 신장 트리를 열거해 최소 가중치를 찾는 방법: 신장 트리 수는 $V^{V-2}$ (Cayley 공식)으로 $V = 10^4$에서 불가능하다.

단순 배열 기반 Prim(각 단계에서 모든 정점을 선형 탐색): $O(V^2) = O(10^8)$.
$V = 10^4$, $E = V^2/2 \approx 5 \times 10^7$에서 $O(V^2) = 10^8$ → 경계에 걸리거나 초과 가능.

**목표 복잡도:** $O((V + E) \log V)$ 시간 (이진 힙 기반 Prim), $O(V + E)$ 공간.

**근거:** 우선순위 큐(최소 힙)를 사용하면 최소 간선 선택이 $O(\log V)$이다. 정점 방문 $V$회, 간선 처리 $E$회이므로 $O((V + E) \log V)$. 밀집 그래프($E \sim V^2$)에서는 $O(E \log V) = O(V^2 \log V)$가 되어 단순 배열보다 느릴 수 있다. 이 경우 $O(V^2)$ 배열 기반 Prim이 더 나을 수 있지만, 문제 제약에서 간선이 최대 $V(V-1)/2$로 열려 있고 $V \leq 10^4$이면 힙 기반이 실용적이다.

**공간 복잡도:** 인접 리스트 $O(V + E)$, 힙 $O(E)$ (지연 삭제 시 힙에 중복 항목 존재 가능), 방문 배열 $O(V)$.

---

## 목표 함수

```ts
primMst(n: number, edges: [number, number, number][]): number
```

| 매개변수 | 의미 | 제약 |
|----------|------|------|
| `n` | 정점 개수 $V$ (인덱스: $0 \ldots n-1$) | $1 \leq n \leq 10^4$ |
| `edges` | 무방향 간선 `[u, v, w]` 목록 | $E \leq V(V-1)/2$ |
| 반환값 | MST 가중치 합; 연결 그래프가 아니면 $-1$ | — |

**엣지케이스:**

1. **$n = 1$ (단일 정점):** 신장 트리는 간선 0개. 가중치 합 `0`을 반환한다.
2. **비연결 그래프:** 힙이 소진됐는데 `visitedCount < n`이면 `-1` 반환.
3. **가중치 0 포함:** 합이 `0`이어도 유효. `-1`(비연결)과 혼동하지 말 것.
4. **최대 입력 ($V = 10^4$, $w = 10^9$):** MST 합이 최대 $(V-1) \times 10^9 \approx 10^{13}$. `number` 타입으로 처리 가능하나 정밀도 주의.
5. **중복 간선 (같은 `[u, v]` 쌍):** 힙에 같은 이웃 정점이 여러 번 삽입되지만, 방문 배열로 중복 방문이 차단되므로 정확도에는 영향 없다.

## 핵심 아이디어

**핵심 아이디어**: "현재 트리와 인접한 미방문 정점 중 가장 가까운 것을 최소 힙으로 빠르게 찾는다 — 새 정점을 추가할 때마다 이웃을 힙에 넣으면 항상 최선 후보가 준비되어 있다."

시작 정점 하나에서 출발해 현재 트리 경계에서 가장 가중치가 작은 간선을 반복적으로 선택한다. 이 탐욕 선택이 컷 성질에 의해 항상 MST에 포함됨이 보장된다. 최소 힙이 최선 후보를 $O(\log E)$에 제공하고, 이미 방문한 정점은 지연 삭제로 무시하기 때문에 복잡한 decrease-key 없이 구현이 단순하다.

**풀이 구조**
1. 무방향 간선을 양방향 인접 리스트로 구성한다.
2. 시작 정점 0을 `(가중치=0, 정점=0)`으로 최소 힙에 삽입한다.
3. 힙에서 `(w, u)`를 꺼낸다. `visited[u]`가 `true`이면 지연 삭제로 무시한다.
4. `visited[u] = true`, `totalWeight += w`, `visitedCount++`.
5. $u$의 미방문 이웃 $(v, w_{uv})$를 힙에 삽입한다.
6. 힙이 빌 때까지 반복하고, `visitedCount == n`이면 `totalWeight`를, 아니면 `-1`을 반환한다.

**조건**: 무방향 가중치 그래프여야 한다. 연결 그래프이면 MST 가중치 합을, 비연결이면 `-1`을 반환한다. 힙 항목은 반드시 `(weight, vertex)` 순서여야 가중치 기준 정렬이 된다.

**대표 예시**: 케이블 네트워크 최소 구축 비용
$n$개의 빌딩을 인터넷 케이블로 연결할 때 총 케이블 길이를 최소화하는 문제다. 현재 연결된 빌딩 집합에서 가장 짧은 케이블로 새 빌딩을 추가해 나가면 자연스럽게 MST가 완성된다. 최소 힙 덕분에 $O((V+E)\log V)$에 처리된다.

**언제 쓰나**
"최소 신장 트리(MST)를 구하라"는 문제에서 정점 수가 크고 간선이 밀집된 그래프에 사용한다. 간선 수보다 정점 수가 중요한 상황(밀집 그래프)에서 Kruskal 대신 Prim이 더 효율적일 수 있다.

### 원형 아이디어와 naive 접근

최소 신장 트리를 가장 단순하게 생각하면: 모든 신장 트리를 열거하고 가중치 합이 최소인 것을 선택한다.

```
naive:
  minWeight = INF
  for each spanning tree T of G:
    minWeight = min(minWeight, sum of weights in T)
  return minWeight
```

신장 트리 수가 지수적이어서 불가능하다.

다음으로 단순한 방법: 임의의 시작 정점에서 출발해, 현재 트리와 인접한 간선 중 최소 가중치 간선을 매번 선형 탐색으로 찾는다.

```
naive_prim:
  visited = {start}
  totalWeight = 0
  repeat n-1 times:
    minEdge = min weight edge from visited to not-visited
    totalWeight += minEdge.weight
    visited.add(minEdge.dest)
  return totalWeight
```

매 반복에서 "방문 집합에서 미방문 집합으로 가는 최소 간선"을 선형 탐색하면 $O(V)$이고, 이를 $n-1$번 반복하면 $O(V^2)$이다. $V = 10^4$이면 $10^8$ 연산으로 경계에 걸린다.

**낭비의 정체:** 매 반복마다 이미 검사한 간선들을 다시 검사한다. 방문 집합이 점점 커지면서 인접 간선의 최솟값이 변화하는데, 이를 효율적으로 추적하지 않는 것이 낭비이다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1 (컷 성질, Cut Property):** 임의의 컷 $(S, V \setminus S)$에서 그 컷을 건너는 최소 가중치 간선은 반드시 어떤 MST에 포함된다. 이것은 Prim의 탐욕적 선택이 매 단계에서 안전함을 보장한다.

- **관찰 2 (현재 트리에서의 컷):** Prim에서 $T$를 현재 방문한 정점 집합이라 하면, 매 단계에서 선택하는 컷은 $(T, V \setminus T)$이다. 이 컷의 최소 가중치 간선을 선택하면 컷 성질에 의해 MST 조건이 유지된다.

- **관찰 3 (힙으로 최솟값 효율화):** "현재 트리와 인접한 미방문 정점 중 가장 가까운 것"을 매번 찾는 것이 병목이다. 새 정점을 트리에 추가할 때마다 그 이웃을 힙에 삽입하면, 힙이 항상 현재 트리와 인접한 미방문 정점 중 최솟값 후보를 정렬 유지한다.

### 관찰을 형식화: 상태/구조 정의

**방문 배열:** `visited[v]` ∈ {`true`, `false`}. `visited[v] = true`이면 $v$는 이미 MST에 추가된 정점이다.

**최소 힙:** `(weight, vertex)` 쌍을 저장한다. `weight`는 현재 트리와 `vertex`를 잇는 간선의 가중치이다.

**지연 삭제(lazy deletion):** 힙에 같은 정점이 여러 가중치로 중복 삽입될 수 있다. 힙에서 꺼낸 정점이 이미 방문했으면(`visited[v] = true`) 무시하고 다음을 꺼낸다. 이렇게 하면 힙에서 "무효 항목"을 즉시 삭제하지 않아도 된다.

지연 삭제를 사용하는 이유: 힙의 특정 항목을 직접 수정(decrease-key)하려면 힙 인덱스를 별도로 관리해야 한다. 지연 삭제는 이 복잡도 없이 동일한 효과를 낸다. 힙에 중복 항목이 생겨 공간이 늘어나지만($O(E)$), 실제 방문은 정점당 한 번이므로 총 연산은 $O((V + E) \log E)$이다.

### 점화식 또는 핵심 연산

새 정점 $u$를 MST에 추가할 때, 이웃 $v$를 힙에 삽입하는 조건:

$$
\text{if } \neg \text{visited}[v]: \quad \text{minHeap.push}(w_{uv}, v)
$$

힙에서 꺼낸 `(w, u)`를 MST에 추가하는 조건:

$$
\text{if } \neg \text{visited}[u]: \quad \text{totalWeight} \mathrel{+}= w, \quad \text{visitedCount} \mathrel{+}= 1
$$

- $w_{uv}$: 간선 $(u, v)$의 가중치. 미방문 $v$에 대해 현재 알려진 최소 연결 비용 후보를 힙에 추가한다.
- $\neg \text{visited}[u]$: 힙에서 꺼낼 때 이미 방문한 정점이면 무시(지연 삭제).
- `totalWeight += w`: 컷 성질에 의해 이 $w$는 현재 컷 $(T, V \setminus T)$의 최소 가중치 간선이므로 MST에 포함된다.

**누적 가중치:**

$$
W_{\text{MST}} = \sum_{k=1}^{n-1} w_k
$$

여기서 $w_k$는 $k$번째로 MST에 추가된 간선의 가중치이다.

### 정당성 — 왜 이것이 옳은가

**컷 성질에 의한 귀납:** 기저 사례로 시작 정점 하나를 $T$에 넣는다.
단계 $k$에서 $T$에 $k$개의 정점이 있고 현재 선택된 간선들이 MST 부분집합을 이룬다고 가정한다.
단계 $k+1$에서 컷 $(T, V \setminus T)$의 최소 가중치 간선 $e = (u, v, w)$를 선택한다.
컷 성질에 의해 $e$는 어떤 MST에 포함되므로, $T \cup \{v\}$와 선택된 간선들이 MST의 부분집합을 유지한다.
이를 $n-1$번 반복하면 MST 전체가 구성된다.

**지연 삭제의 정확성:** 힙에 같은 정점 $v$가 가중치 $w_1 < w_2$로 두 번 삽입됐다고 하자.
$w_1$이 먼저 꺼내져 $v$가 방문 처리된다. 이후 $w_2$ 항목이 꺼여도 `visited[v] = true`이므로 무시된다. 항상 최소 가중치로 처리되므로 정확하다.

**까다로운 케이스:**
- 가중치 동일한 간선 여러 개: 어느 것을 선택해도 MST 조건을 만족하므로 정확성에 영향 없다.
- 고립 정점(연결되지 않은 정점): 힙이 소진돼도 방문하지 못하면 `visitedCount < n` → `-1` 반환.

### 구현 디테일과 최적화

- **시작 정점 선택:** 정점 0에서 `(0, 0)` (가중치 0, 정점 0)으로 시작한다. 시작 정점 자체는 가중치 0으로 트리에 추가되어 `totalWeight`에 0을 더한다. 어느 정점에서 시작해도 MST 가중치는 동일하다.

- **힙 항목 형식:** `(weight, vertex)` 순서로 힙에 넣어야 힙이 가중치 기준으로 정렬된다. `(vertex, weight)` 순서로 넣으면 정점 인덱스 기준으로 정렬되어 잘못된 결과가 나온다.

- **지연 삭제 vs decrease-key:** 지연 삭제는 힙에 중복 항목이 쌓이지만 구현이 단순하다. decrease-key를 지원하는 피보나치 힙을 사용하면 $O(E + V \log V)$로 개선되지만 구현이 매우 복잡하다. 실용적으로 이진 힙 + 지연 삭제로 충분하다.

- **함정 — `visitedCount` 대신 `totalWeight`로 연결 여부 판단:** `totalWeight == 0`이어도 유효한 MST일 수 있다(가중치 0 간선). 연결 여부는 반드시 `visitedCount`로 판단해야 한다.

- **함정 — 무방향 간선의 단방향 등록:** `edges`의 `[u, v, w]`를 `adj[u]`에만 등록하면 `v → u` 방향의 간선이 누락된다. 반드시 `adj[u]`와 `adj[v]` 양쪽에 등록해야 한다.

- **함정 — 힙에서 꺼낸 후 visited 체크 전에 totalWeight 증가:** 지연 삭제에서 꺼낸 항목이 이미 방문한 정점인지 먼저 확인해야 한다. `visited` 체크 전에 `totalWeight`를 늘리면 무효 항목의 가중치까지 합산된다.

### 의사코드

```
function primMst(n, edges):
    // 1. 인접 리스트 구성 (무방향)
    adj[0..n-1] = []
    for (u, v, w) in edges:
        adj[u].push((v, w))
        adj[v].push((u, w))            // 불변식: adj는 무방향 그래프를 대칭으로 표현

    // 2. 초기화
    visited[0..n-1] = false            // 불변식: visited[v] = v가 MST에 추가됐는가
    minHeap = MinHeap()
    minHeap.push((0, 0))               // (가중치=0, 시작 정점=0)
    totalWeight  = 0                   // 불변식: 현재까지 MST에 추가된 간선 가중치 합
    visitedCount = 0                   // 불변식: 현재까지 MST에 추가된 정점 수

    // 3. 힙 기반 탐욕 확장
    while minHeap is not empty:
        (w, u) = minHeap.pop()

        if visited[u]: continue        // 지연 삭제: 이미 방문한 정점은 무시

        visited[u] = true
        totalWeight  += w              // 불변식: 컷 성질에 의해 w는 현재 컷 최솟값
        visitedCount += 1

        for (v, weight) in adj[u]:
            if not visited[v]:
                minHeap.push((weight, v))  // 불변식: 미방문 이웃의 연결 비용 후보 삽입

    // 4. 결과
    if visitedCount == n:
        return totalWeight             // 모든 정점 방문 = 연결 그래프
    else:
        return -1                      // 힙 소진 후 미방문 정점 존재 = 비연결

// 핵심 불변식:
//   힙에서 꺼낸 (w, u)에서 visited[u]=false이면,
//   w는 현재 MST 트리와 u를 잇는 간선들 중 최솟값이다.
//   visited[u] = true 설정 후 totalWeight += w는 컷 성질에 의해 항상 MST에 포함됨이 보장.
```

**핵심 불변식:** 힙에서 `(w, u)`를 꺼낼 때 `visited[u] = false`이면, $w$는 현재 방문 집합 $T$와 $u$를 연결하는 간선들 중 힙이 관리하는 최솟값이다. 지연 삭제에 의해 $u$가 이후 더 작은 가중치로 다시 처리될 수 없으므로, 이 $w$가 컷 $(T, V \setminus T)$의 최소 간선임이 보장되어 컷 성질이 성립한다.
