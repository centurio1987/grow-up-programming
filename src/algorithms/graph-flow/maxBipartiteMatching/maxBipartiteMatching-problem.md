# 최대 이분 매칭 (Maximum Bipartite Matching)

## 한 줄 요약

> 왼쪽 정점 수 `left`, 오른쪽 정점 수 `right`, 이분 간선 목록 `edges` 를 받아, 어떤 두 간선도 끝점을 공유하지 않는 간선 부분집합의 **최대 크기**를 반환한다.

## 스토리

취업박람회 하루 전날, 행사 운영팀이 좌석 배정을 마무리하고 있다. 지원자 여러 명이 각자 원하는 기업 부스 목록을 제출했다. 각 지원자는 본인이 원하는 부스 중 최대 **한 곳**에만 배정될 수 있고, 한 부스에는 같은 시간대에 **한 명**만 들어갈 수 있다.

운영팀 목표는 **가능한 한 많은 지원자에게 자리를 배정**하는 것이다. 지원자가 원하는 부스만 선택해야 하는 조건을 지키면서, 배정된 쌍의 수를 최대로 만들어야 한다.

지원자와 부스 목록을 정리해서 넘겨주면, 최대 몇 쌍까지 연결할 수 있는지 알려주는 함수가 필요하다.

## 함수 인터페이스

```ts
export function maxBipartiteMatching(
  left: number,
  right: number,
  edges: [number, number][],
): number;
```

- `left` — 왼쪽 정점의 개수 $L$. 왼쪽 정점 번호는 $0 \ldots L-1$.
- `right` — 오른쪽 정점의 개수 $R$. 오른쪽 정점 번호는 $0 \ldots R-1$.
- `edges` — 이분 간선 목록. 각 원소 `[u, v]` 는 왼쪽 정점 `u` 와 오른쪽 정점 `v` 사이의 간선이다. `u` 는 $0 \leq u < L$, `v` 는 $0 \leq v < R$.
- 반환 — 최대 매칭의 크기 $|M|$.

## 제약 조건

- $1 \leq L, R \leq 10^3$
- $0 \leq E \leq L \cdot R$ (여기서 $E$ 는 간선의 수)
- 간선의 첫 번째 원소는 왼쪽 정점 번호, 두 번째 원소는 오른쪽 정점 번호다.
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

이분 그래프 $G = (L \cup R, E)$ 에서 **매칭** $M \subseteq E$ 은 어떤 두 간선도 공통 끝점을 공유하지 않는 간선 부분집합이다.

$$M \subseteq E \text{ s.t. } \forall e_1, e_2 \in M,\; e_1 \neq e_2 \Rightarrow e_1 \cap e_2 = \emptyset$$

최대 매칭은 $|M|$ 이 가장 큰 매칭이다.

$$\text{maxBipartiteMatching}(G) = \max_{M \text{ matching}} |M|$$

- 왼쪽 정점 하나는 매칭에서 최대 하나의 오른쪽 정점과 연결된다. 오른쪽도 마찬가지다.
- 간선이 없으면 매칭 크기는 `0` 이다.
- 동일한 간선이 중복 입력될 수 있으나, 매칭 판정에는 하나로 취급한다.
- 매칭 크기의 상한은 $\min(L, R)$ 이다.

## 예시

```ts
maxBipartiteMatching(3, 3, [
  [0, 0], [0, 1],
  [1, 0],
  [2, 2],
]); // 3 — (0,1), (1,0), (2,2) 세 쌍 모두 끝점 겹침 없이 연결 가능

maxBipartiteMatching(2, 2, [
  [0, 0],
  [1, 0],
]); // 1 — 오른쪽 정점 0은 한 번만 매칭될 수 있어 한 쌍만 연결 가능

maxBipartiteMatching(3, 3, []); // 0 — 간선 없음, 연결할 쌍 없음

maxBipartiteMatching(4, 1, [
  [0, 0], [1, 0], [2, 0], [3, 0],
]); // 1 — 오른쪽 정점이 하나뿐이므로 최대 1쌍

maxBipartiteMatching(3, 3, [
  [0, 0], [0, 1],
  [1, 0],
  [2, 1], [2, 2],
]); // 3 — 단순 순서대로 매칭하면 2쌍이지만, 재배정으로 3쌍까지 가능
```
