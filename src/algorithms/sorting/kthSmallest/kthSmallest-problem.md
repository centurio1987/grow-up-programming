# Kth Smallest Element

## 한 줄 요약

> 정수 배열 `A` 와 정수 `k` 가 주어질 때, **`A` 를 오름차순으로 정렬했을 때 앞에서 `k` 번째에 오는 값**을 반환한다. (`k` 는 1부터 센다.)

## 스토리

기상청 전산실. 오늘 전국 **`N` 개 관측 지점**에서 측정한 오늘의 최저기온(℃)이 배열 하나로 들어왔다.
한겨울이라 영하(음수)도 섞여 있고, 우연히 **같은 기온**이 찍힌 지점들도 있다.

기자에게서 전화가 온다. "오늘 **`k` 번째로 추운** 지점이 몇 도였죠?"

가장 추운 곳이 1번째, 그다음이 2번째… 이렇게 추운 순서대로 줄을 세웠을 때 **`k` 번째 자리에 있는 기온**이
바로 답이다. 같은 기온인 지점이 여럿이면 그 지점들은 줄에서 **각각 한 칸씩** 차지한다.

여러분이 할 일은, 이 측정값 배열과 순번 `k` 를 받아 **`k` 번째로 작은(=추운) 값**을 돌려주는 함수를 만드는 것이다.

## 중요도 · 난이도

| 항목 | 값 |
|------|-----|
| 중요도 | ★★ 중 — 빈출 |
| 난이도 | 중급 |

## 함수 인터페이스

```ts
export function kthSmallest(A: number[], k: number): number;
```

- `A` — 측정값(정수) 배열.
- `k` — 1부터 시작하는 순번. `k = 1` 이면 가장 작은 값, `k = N` 이면 가장 큰 값을 가리킨다.
- 반환 — `A` 를 오름차순 정렬했을 때 앞에서 `k` 번째에 오는 값 하나.

## 제약 조건

- $1 \leq N \leq 100{,}000$ — 여기서 $N$ 은 배열 `A` 의 길이다.
- $-10^9 \leq A[i] \leq 10^9$ — 각 원소는 이 범위의 **정수**이며 **음수일 수 있다.** (실수·`NaN`·`Infinity` 는 입력으로 주어지지 않는다.)
- $1 \leq k \leq N$ — `k` 는 **1-based** 순번이라 항상 유효한 범위 안에 있다(빈 배열이나 범위 밖 `k` 는 입력으로 주어지지 않는다).
- 입력 `A` 의 순서는 **보장되지 않는다**(정렬되어 있을 수도, 아닐 수도 있다).
- **시간 제한 1초, 메모리 제한 256 MB.** (성능 기준은 단순 연산으로 CPU 가 1초에 약 $10^8$ 번 계산한다는 가정을 따른다.)
- 함수는 입력 `A` 를 **제자리에서 변경해도 된다** — 호출이 끝난 뒤 `A` 의 원소 순서는 보장하지 않는다.

## 문제 상세

배열 `A` 를 **오름차순(작은 값 → 큰 값)** 으로 정렬한 결과를 $S$ 라 하면, 답은 다음과 같이 정의된다.

$$\text{kthSmallest}(A, k) = S[k-1] \quad (\text{0-based 인덱스})$$

즉 "**`k` 번째로 작은 값**"은 정렬된 줄에서 앞에서부터 `k` 칸째에 있는 값이다. 다음 규칙을 따른다.

- **순번은 1부터.** `k = 1` 은 최솟값, `k = N` 은 최댓값을 가리킨다. (0번째는 없다.)
- **중복은 각각 한 자리.** 같은 값이 여러 개면 정렬된 줄에서 그 값들이 **개수만큼 연속된 자리**를 차지한다.
  예를 들어 `[1, 1, 2]` 에서 `k = 2` 의 답은 (또 다른) `1` 이다.
- **음수·0 포함.** 값의 부호는 정렬 순서에만 영향을 준다. 가장 작은 음수가 맨 앞에 온다.
- **반환은 값 하나.** 인덱스가 아니라 그 자리에 있는 **원소의 값**을 돌려준다.
- 제약상 `k` 는 항상 `1..N` 범위이므로, 이 범위 밖 입력에 대한 동작은 정의하지 않는다.

## 예시

```ts
kthSmallest([3, 1, 2], 1);              // 1  — 작은 순 [1,2,3]에서 1번째
kthSmallest([3, 1, 2], 2);              // 2  — 작은 순 [1,2,3]에서 2번째
kthSmallest([3, 1, 2], 3);              // 3  — 작은 순 [1,2,3]에서 3번째(=최댓값)

kthSmallest([7, 10, 4, 3, 20, 15], 3);  // 7  — 작은 순 [3,4,7,10,15,20]에서 3번째
kthSmallest([7, 10, 4, 3, 20, 15], 4);  // 10 — 작은 순 [3,4,7,10,15,20]에서 4번째

kthSmallest([-5, 0, 5, -10, 10], 1);    // -10 — 작은 순 [-10,-5,0,5,10]에서 1번째(가장 추운 값)

kthSmallest([2, 1, 2, 1, 3, 1], 4);     // 2  — 작은 순 [1,1,1,2,2,3]에서 4번째(중복은 각각 한 자리)

kthSmallest([5, 5, 5, 5], 3);           // 5  — 모든 값이 같으면 어떤 k든 그 값(경계: 전부 동일)

kthSmallest([42], 1);                   // 42 — 원소가 하나뿐이라 1번째가 곧 그 값(경계: N=1)

// 값 범위 경계: 원소가 ±1,000,000,000 까지 올 수 있다
kthSmallest([1_000_000_000, -1_000_000_000, 0], 2); // 0 — 작은 순 [-1000000000, 0, 1000000000]에서 2번째
```

각 예시의 설명은 **그 출력이 정의에 어떻게 들어맞는지**만 보여줄 뿐, 어떻게 구하는지(풀이 방법)는 말하지 않는다.

### 순번으로 답을 읽는 법 (그림)

`[7, 10, 4, 3, 20, 15]`, `k = 3` 의 답이 왜 `7` 인지 — 값을 **작은 순으로 줄 세웠을 때** `k` 번째 자리를 보면 된다:

```
원래 배열:        7   10   4    3    20   15
작은 순 줄 세우기:  3    4   7    10   15   20
순번(k):          1    2   3    4    5    6
                            ↑
                           k=3  → 7
```
