# 최적 파일 병합

## 한 줄 요약

> 파일 크기 배열을 받아, 인접한 파일을 반복 병합하여 하나로 합칠 때 드는 최소 총 비용을 반환한다.

## 스토리

백업 시스템을 관리하는 서버 엔지니어 태양은 여러 개의 데이터 파일을 하나로 합치는 작업을 자동화하려 한다. 파일은 일렬로 나열되어 있으며, 한 번에 **인접한 두 파일 묶음**만 합칠 수 있다. 두 묶음을 합칠 때 드는 시간은 두 묶음의 크기 합에 비례한다.

예를 들어 크기 10, 20, 30인 파일이 있을 때, 10과 20을 먼저 합치면 비용 30이 들고 이후 30과 30을 합치면 비용 60이 들어 총 90이다. 반면 20과 30을 먼저 합치면 비용 50, 이후 10과 50을 합치면 50이 들어 총 100이다. 어떤 순서로 합치느냐에 따라 총 비용이 달라진다.

태양은 $n$개의 파일을 하나로 합칠 때 가능한 모든 순서 중 총 비용이 가장 작아지는 순서를 찾아야 한다. 파일 수가 수천 개에 달해도 빠르게 결과를 내야 한다.

## 함수 인터페이스

```ts
export function knuthOptimization(freq: number[]): number;
```

- `freq` — 각 파일의 크기를 나타내는 비음수 정수 배열 (인접한 순서로 나열)
- 반환 — 모든 파일을 하나로 합칠 때 드는 최소 총 비용

## 제약 조건

- $1 \leq n \leq 5000$ (여기서 $n$은 `freq`의 길이)
- $\text{freq}[i] \geq 0$
- 반환값은 $\geq 0$
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

$n$개의 인접한 파일 크기 `freq[0..n-1]`가 주어진다. 한 번의 병합 연산은 인접한 두 그룹을 하나로 합치며, 이때 비용은 두 그룹의 크기 합이다. 모든 파일이 하나의 그룹이 될 때까지 병합을 반복한다.

가능한 모든 병합 순서 중 **총 비용의 최솟값**을 반환하라.

- 길이 1인 배열은 병합할 대상이 없으므로 0을 반환한다.
- 모든 크기가 0이면 어떤 순서로 병합해도 비용은 0이다.

## 예시

```ts
knuthOptimization([10]);          // 0 — 파일 하나, 병합 불필요
knuthOptimization([10, 20]);      // 30 — 한 번 병합: 10+20=30
knuthOptimization([1, 2, 3]);     // 9 — (1+2)=3 비용, 이후 (3+3)=6 비용, 총 9
knuthOptimization([10, 20, 30]);  // 90 — (10+20)=30, 이후 (30+30)=60, 총 90

// 경계 케이스
knuthOptimization([0, 5, 0]);         // 10 — (0+5)=5, 이후 (5+0)=5, 총 10
knuthOptimization([0, 0, 0, 0]);      //  0 — 모두 0, 비용 없음
knuthOptimization([10, 10, 10, 10]);  // 80 — 균등 병합 최적
```
