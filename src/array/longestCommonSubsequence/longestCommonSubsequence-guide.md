# longestCommonSubsequence — 최장 공통 부분 수열 (LCS)

## 성능 목표 예측

| 항목 | 값 |
|------|-----|
| 문자열 길이 | $0 \leq n = |s| \leq 1{,}000$, $0 \leq m = |t| \leq 1{,}000$ |
| 문자 종류 | ASCII |

**naive 접근의 문제점**: $s$와 $t$의 모든 공통 부분 수열을 열거하면 최악 $O(2^n \cdot 2^m)$이다. 재귀로 구현해도 메모이제이션 없이는 같은 $(i, j)$ 쌍이 지수적으로 재계산된다.

**목표 복잡도**: 시간 $O(nm)$, 공간 $O(nm)$ 또는 롤링으로 $O(\min(n,m))$. $n \cdot m = 10^6$으로 충분하다.

**공간 복잡도**: 2D DP 테이블은 $O(nm)$. 점화식이 이전 행만 참조하므로 행 롤링으로 $O(m)$까지 줄일 수 있다.

---

## 목표 함수

```ts
function longestCommonSubsequence(s: string, t: string): number
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `s` | 첫 번째 문자열 | $0 \leq |s| \leq 1{,}000$ |
| `t` | 두 번째 문자열 | $0 \leq |t| \leq 1{,}000$ |

**반환값**: 두 문자열의 공통 부분 수열 중 가장 긴 것의 길이 (순서 유지, 연속 아님).

**엣지케이스**:

| 입력 | 기대 출력 | 이유 |
|------|-----------|------|
| `s=""`, `t="abc"` | `0` | 빈 문자열 |
| `s="abc"`, `t="abc"` | `3` | 동일 문자열 |
| `s="abc"`, `t="xyz"` | `0` | 공통 문자 없음 |
| `s="abcde"`, `t="ace"` | `3` | LCS = "ace" |

## 핵심 아이디어

**핵심 아이디어**: "두 문자열의 마지막 문자가 같으면 공짜로 하나를 추가하고, 다르면 둘 중 하나를 포기한 최적해를 선택한다."

LCS는 두 문자열의 접두어 쌍을 상태로 잡는 2D DP로 풀린다. s[i-1]==t[j-1]이면 dp[i][j]=dp[i-1][j-1]+1로 LCS에 하나를 추가하고, 다르면 dp[i][j]=max(dp[i-1][j], dp[i][j-1])로 어느 한쪽을 제외한 최적을 이어받는다. 테이블을 왼쪽 위에서 오른쪽 아래 방향으로 채우면 된다.

**풀이 구조**
1. (n+1)×(m+1) 크기의 dp 테이블을 0으로 초기화한다.
2. i=1..n, j=1..m을 순회하며 s[i-1]==t[j-1]이면 dp[i][j]=dp[i-1][j-1]+1을 적용한다.
3. 문자가 다르면 dp[i][j]=max(dp[i-1][j], dp[i][j-1])을 적용한다.
4. dp[n][m]을 반환한다.

**조건**: 부분 수열은 순서를 유지하되 연속일 필요가 없다. 빈 문자열과의 LCS는 항상 0.

**대표 예시**: `s="abcde", t="ace"`
dp 테이블을 채우면 dp[5][3]=3이 나온다. "a", "c", "e"가 공통 부분 수열 중 가장 긴 것이다.

**언제 쓰나**
두 시퀀스에서 공통으로 등장하는 패턴의 최대 길이를 구할 때 사용한다. diff 도구, 유전자 서열 비교, 표절 검사 등 두 시퀀스의 공통 구조를 파악하는 문제에서 표준 알고리즘이다.

### 원형 아이디어와 naive 접근

$s$와 $t$의 공통 부분 수열을 재귀로 탐색한다. $f(i, j)$를 $s[0..i-1]$과 $t[0..j-1]$의 LCS 길이라 하면:

```
f(i, j):
    if i == 0 or j == 0: return 0
    if s[i-1] == t[j-1]: return f(i-1, j-1) + 1
    return max(f(i-1, j), f(i, j-1))
```

메모이제이션 없이는 같은 $(i, j)$가 지수적으로 반복 계산된다. 트리 높이가 $\max(n, m)$이고 분기가 2이면 $O(2^{n+m})$이 된다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1**: $f(i, j)$는 $f(i-1, j)$, $f(i, j-1)$, $f(i-1, j-1)$ 중 최대 세 값에만 의존한다 — 중복 부분 문제가 명확하다.
- **관찰 2**: 고유한 부분 문제의 수가 $(n+1)(m+1) = O(nm)$이다. 테이블로 채우면 각 셀을 한 번만 계산한다.
- **관찰 3**: $s[i-1] = t[j-1]$인 경우 반드시 이 문자를 LCS에 포함하는 것이 최적이다 (포함하지 않으면 $\max(f(i-1,j), f(i,j-1)) \leq f(i-1,j-1) \leq f(i-1,j-1)+1$이므로 손해가 없다).

### 관찰을 형식화: 상태/구조 정의

상태 $dp[i][j]$를 다음과 같이 정의한다.

$$dp[i][j] = s[0..i-1] \text{과 } t[0..j-1] \text{의 최장 공통 부분 수열의 길이}$$

초기 조건:

$$dp[0][j] = 0 \quad (0 \leq j \leq m)$$
$$dp[i][0] = 0 \quad (0 \leq i \leq n)$$

빈 문자열과 어떤 문자열의 공통 부분 수열은 항상 빈 수열이므로 길이는 $0$이다.

이 정의가 왜 이 형태여야 하는가: "접두어 쌍"으로 상태를 잡으면 $dp[i][j]$가 $dp[i-1][*]$과 $dp[i][j-1]$에만 의존해 순환 의존성이 없다. "임의 부분 문자열 쌍"으로 확장하면 상태 수가 $O(n^2 m^2)$으로 폭발한다.

### 점화식 또는 핵심 연산

$1 \leq i \leq n$, $1 \leq j \leq m$에 대해:

$$dp[i][j] = \begin{cases} dp[i-1][j-1] + 1 & \text{if } s[i-1] = t[j-1] \\ \max\bigl(dp[i-1][j],\; dp[i][j-1]\bigr) & \text{otherwise} \end{cases}$$

결과: $dp[n][m]$

- $dp[i-1][j-1] + 1$: $s[i-1]$과 $t[j-1]$이 같으므로 이 문자를 LCS의 마지막 원소로 추가
- $dp[i-1][j]$: $s[i-1]$을 LCS에 포함하지 않고 $t[0..j-1]$과의 최적해 사용
- $dp[i][j-1]$: $t[j-1]$을 LCS에 포함하지 않고 $s[0..i-1]$과의 최적해 사용

### 정당성 — 왜 이것이 옳은가

귀납적으로 증명한다. 기저: $dp[0][*] = dp[*][0] = 0$은 빈 문자열과의 LCS가 $0$임을 정확히 나타낸다.

귀납 가정: 모든 $(i', j')$ $(i' < i$ 또는 $j' < j)$에 대해 $dp[i'][j']$이 정확하다고 가정한다. $dp[i][j]$에서 $s[i-1] = t[j-1]$이면 이 문자를 LCS에 포함하는 것이 항상 최적이다 (포함하지 않는 경우는 $\max(dp[i-1][j], dp[i][j-1]) \leq dp[i-1][j-1]+1$이므로 더 나쁘거나 같다). $s[i-1] \neq t[j-1]$이면 최소한 하나는 LCS에서 제외해야 하므로 두 경우 중 최댓값이 최적이다.

관찰: $dp[i][j] \leq \min(i, j)$가 항상 성립한다 — LCS는 두 문자열 중 짧은 것보다 길 수 없다.

### 구현 디테일과 최적화

- **공간 최적화**: 점화식이 $dp[i-1][*]$ 행과 $dp[i][j-1]$ 셀만 참조한다. 이전 행 하나만 유지하면 $O(m)$ 공간으로 해결 가능하다. 단, $dp[i-1][j-1]$ 값을 갱신 전에 미리 저장해야 한다.
- **루프 순서**: $i$ (행)가 바깥, $j$ (열)가 안쪽. $dp[i][j]$가 $dp[i][j-1]$을 참조하므로 $j$는 왼쪽에서 오른쪽으로 증가해야 한다.
- **함정**: 인덱스 오프셋 — $s[i-1]$이 문자 인덱스이고 $dp$ 인덱스는 1-based임을 혼동하지 않아야 한다.
- **함정**: 초기화를 $0$이 아닌 다른 값으로 하면 경계 조건이 깨진다. `dp[0][*]`과 `dp[*][0]`은 반드시 $0$이어야 한다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function longestCommonSubsequence(s, t):
    n ← len(s),  m ← len(t)
    dp ← (n+1) × (m+1) 크기의 0 행렬     // 불변식: dp[i][j] = s[0..i-1]과 t[0..j-1]의 LCS 길이

    for i from 1 to n:
        for j from 1 to m:
            if s[i-1] == t[j-1]:
                dp[i][j] ← dp[i-1][j-1] + 1   // 공통 문자 추가
            else:
                dp[i][j] ← max(dp[i-1][j],     // s[i-1] 제외
                               dp[i][j-1])     // t[j-1] 제외

    return dp[n][m]
```

### Activity Diagram

```mermaid
flowchart TD
    A([시작]) --> B["dp[0..n][0..m] = 0"]
    B --> C{i <= n?}
    C -- No --> J([return dp[n][m]])
    C -- Yes --> D{j <= m?}
    D -- No --> I["i++ / j=1"]
    I --> C
    D -- Yes --> E{s[i-1] == t[j-1]?}
    E -- Yes --> F["dp[i][j] = dp[i-1][j-1] + 1"]
    E -- No --> G["dp[i][j] = max(dp[i-1][j], dp[i][j-1])"]

    G --> H
    H --> D
```

**핵심 불변식**: $dp[i][j]$ 셀이 채워진 시점에 $dp[i][j]$는 $s[0..i-1]$과 $t[0..j-1]$의 실제 LCS 길이이며, $0 \leq dp[i][j] \leq \min(i, j)$가 항상 성립한다.

## 복잡도 분석 심화

| 접근 방식 | 시간 | 공간 | 비고 |
|-----------|------|------|------|
| 순수 재귀 (메모이제이션 없음) | $O(2^{n+m})$ | $O(n+m)$ | 불가 |
| 2D DP | $O(nm)$ | $O(nm)$ | 표준 |
| 행 롤링 1D DP | $O(nm)$ | $O(\min(n,m))$ | 공간 최적 |
| Hirschberg 알고리즘 | $O(nm)$ | $O(\min(n,m))$ | LCS 복원도 $O(\min(n,m))$ 공간 |

**LCS와 편집 거리의 관계**: $\text{editDistance}(s, t) = n + m - 2 \cdot \text{LCS}(s, t)$ (삽입/삭제만 허용, 교체 불허 시). 교체도 허용하면 이 관계가 성립하지 않는다.

**LCS 복원**: `dp[n][m]`에서 역추적한다. $s[i-1] = t[j-1]$이면 이 문자를 LCS에 포함하고 $(i-1, j-1)$로 이동. 그렇지 않으면 `dp[i-1][j] >= dp[i][j-1]`이면 $(i-1, j)$로, 그렇지 않으면 $(i, j-1)$로 이동. $O(n+m)$ 시간에 복원된다.

**변형 — Longest Common Substring (연속 부분 문자열)**: LCS와 달리 연속이어야 하는 경우. $dp[i][j] = s[i-1] = t[j-1]$이면 $dp[i-1][j-1] + 1$, 아니면 $0$. 전체 최댓값이 답. 이 경우 $dp[i][j]$가 "접미사 일치 길이"를 나타낸다.
