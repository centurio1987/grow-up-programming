# largestRectangleInHistogram — 히스토그램 최대 직사각형

## 성능 목표 예측

| 항목 | 값 |
|------|-----|
| 입력 크기 | $1 \leq N \leq 100{,}000$ |
| 높이 범위 | $0 \leq heights[i] \leq 10{,}000$ |

**naive 접근의 문제점**: 각 막대 쌍 $(i, j)$에 대해 해당 구간의 최소 높이를 계산하고 넓이를 구하면 $O(N^2)$ 또는 $O(N^3)$이다. $N = 10^5$에서 $O(N^2) = 10^{10}$으로 시간 초과가 발생한다.

**목표 복잡도**: 시간 $O(N)$, 공간 $O(N)$. 각 원소가 스택에 최대 한 번 push되고 한 번 pop되므로 전체 $O(N)$이 보장된다.

**공간 복잡도**: 스택 $O(N)$. 최악의 경우(단조 증가 배열) 모든 원소가 스택에 쌓인다.

---

## 목표 함수

```ts
function largestRectangleInHistogram(heights: number[]): number
```

| 파라미터 | 의미 | 제약 |
|----------|------|------|
| `heights` | 히스토그램 각 막대의 높이 | $1 \leq N \leq 100{,}000$, $0 \leq heights[i] \leq 10{,}000$ |

**반환값**: 히스토그램 안에서 만들 수 있는 가장 큰 직사각형의 넓이. (막대 너비는 모두 $1$)

**엣지케이스**:

| 입력 | 기대 출력 | 이유 |
|------|-----------|------|
| `[0, 0, 0]` | `0` | 모든 높이 0 |
| `[5]` | `5` | 단일 막대 |
| `[1, 2, 3, 4, 5]` | `9` | 단조 증가 — 최적은 낮은 높이로 넓게 |
| `[5, 4, 3, 2, 1]` | `9` | 단조 감소 |

## 핵심 아이디어

**핵심 아이디어**: "막대가 스택에서 팝되는 순간이 바로 그 막대의 오른쪽 경계가 결정되는 순간이다."

각 막대를 높이로 삼는 최대 직사각형의 너비는, 그 막대보다 낮은 첫 번째 왼쪽 막대와 오른쪽 막대 사이의 거리다. 단조 증가 스택을 유지하면서 새 막대가 스택 top보다 낮을 때 팝하면, 팝되는 막대의 왼쪽 경계(새 스택 top)와 오른쪽 경계(현재 인덱스)를 동시에 알 수 있다.

**풀이 구조**
1. 빈 스택과 `maxArea = 0`으로 초기화한다.
2. i=0..N(N 포함, 높이 0인 가상 막대)을 순회한다.
3. 스택 top의 높이가 현재 높이보다 큰 동안: j=pop(), R=i, L=스택 top(없으면 -1), `area = heights[j] * (R-L-1)`로 갱신한다.
4. 현재 인덱스를 스택에 push한다.
5. `maxArea`를 반환한다.

**조건**: 각 막대의 너비는 1. 높이가 0인 막대도 입력으로 허용.

**대표 예시**: `heights=[2,1,5,6,2,3]`
5와 6이 스택에 쌓였다가 2가 오면 순서대로 팝된다. 6이 팝될 때 너비 1로 면적 6, 5가 팝될 때 너비 2로 면적 10이 계산된다. 최종 답은 10이다.

**언제 쓰나**
"각 원소가 어느 범위까지 확장될 수 있는지"를 효율적으로 구하는 문제에서 사용한다. 히스토그램 직사각형 외에도 빗물 채우기, 주가 범위 계산 등 "좌우 경계를 스택으로 관리"하는 패턴에 적용된다.

### 원형 아이디어와 naive 접근

각 막대 $i$를 "직사각형의 높이"로 삼아, 해당 높이로 만들 수 있는 최대 너비를 찾는다.

```
best = 0
for i from 0 to N-1:
    h = heights[i]
    // i를 중심으로 양방향으로 h 이상인 구간을 확장
    left = i, right = i
    while left > 0 and heights[left-1] >= h: left--
    while right < N-1 and heights[right+1] >= h: right++
    best = max(best, h * (right - left + 1))
```

각 막대에 대해 좌우 확장이 최악 $O(N)$이므로 전체 $O(N^2)$이다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1**: 막대 $i$가 팝(pop)되는 시점, 즉 막대 $i$보다 낮은 막대 $j$가 처음 등장하는 순간이 정확히 막대 $i$의 "오른쪽 경계"이다. 이 정보는 순회 중 자연스럽게 얻을 수 있다.
- **관찰 2**: 스택에 단조 증가 순서로 인덱스를 유지하면, 어떤 막대 $j$가 팝될 때 스택에 남아 있는 top이 $j$의 "왼쪽 경계"가 된다 — $j$보다 낮은 최초의 왼쪽 막대다.
- **관찰 3**: 각 막대는 스택에 최대 한 번 push되고 한 번 pop된다. 따라서 전체 연산이 $O(N)$이다.

### 관찰을 형식화: 상태/구조 정의

단조 증가 스택(인덱스 저장)을 유지한다.

$$\text{스택 내 인덱스} \; s_0 < s_1 < \ldots < s_k \; \text{에 대해} \; heights[s_0] \leq heights[s_1] \leq \ldots \leq heights[s_k]$$

인덱스 $j$가 팝될 때:
- 오른쪽 경계: $R = i$ (현재 순회 중인 인덱스, $heights[i] < heights[j]$)
- 왼쪽 경계: $L = \text{스택의 새 top}$ (없으면 $L = -1$)
- 넓이: $heights[j] \times (R - L - 1)$

이 정의가 왜 올바른가: 스택이 단조 증가이므로 $j$가 팝될 때 스택에 남은 top은 $heights[\text{top}] \leq heights[j]$이고 top이 $j$의 바로 왼쪽에 있는 "막대 $j$보다 낮은 첫 번째 좌측 막대"이다. 따라서 $L$과 $R$ 사이의 모든 막대가 $heights[j]$ 이상임이 보장된다.

### 점화식 또는 핵심 연산

$i = 0, 1, \ldots, N$ ($i = N$은 가상의 높이 $0$ 막대):

```
while stack not empty and heights[stack.top()] > heights[i]:
    j = stack.pop()
    R = i
    L = stack.empty() ? -1 : stack.top()
    area = heights[j] * (R - L - 1)
    maxArea = max(maxArea, area)
stack.push(i)
```

- $i = N$의 가상 막대 ($heights = 0$): 순회 후 스택에 남은 모든 원소를 처리하기 위한 트릭이다.
- $R - L - 1$: $L$과 $R$은 경계 (포함되지 않음)이므로, $j$가 최소 높이인 구간의 실제 너비는 $R - L - 1$이다.

### 정당성 — 왜 이것이 옳은가

임의의 최대 직사각형은 어떤 막대 $j$를 높이로 삼고, $j$가 최솟값인 구간 $[L+1, R-1]$을 너비로 삼는다. 스택이 단조 증가를 유지하므로, $j$가 팝될 때의 $L$과 $R$이 정확히 이 경계를 제공한다 — $L$은 $j$ 왼쪽에서 첫 번째로 $heights[j]$보다 낮은 막대이고, $R$은 $j$ 오른쪽에서 첫 번째로 낮은 막대이다.

스택에 push되지 않고 지나치는 인덱스는 없다 (모든 $i$가 while 조건 충족 여부에 상관없이 push됨). 모든 막대가 정확히 한 번 팝되므로 모든 직사각형이 고려된다.

높이가 $0$인 막대: 넓이가 $0$이므로 최대 넓이에 영향을 주지 않는다.

### 구현 디테일과 최적화

- **가상 막대 트릭**: $i = N$에서 $heights[N] = 0$으로 처리해 스택에 남은 모든 원소를 자동으로 flush한다. 이 없이는 순회 후 스택을 별도로 처리해야 한다.
- **함정**: $heights[j] \geq h$ 대신 $heights[j] > h$로 팝 조건을 설정해야 한다. 등호를 포함하면 동일 높이 막대가 팝되어 넓이 계산이 중복될 수 있다. (단, 결과는 동일하나 로직이 복잡해진다.)
- **함정**: $L = \text{stack.empty()} ? -1 : \text{stack.top()}$에서 스택이 비어있을 때 $-1$을 사용해야 한다. $0$을 사용하면 구간 시작이 인덱스 $0$이 아니라 $1$이 되어 너비가 줄어든다.
- **함정**: 스택에 높이가 아닌 **인덱스**를 저장해야 한다. 높이를 저장하면 $L$ 계산 시 인덱스를 역추적할 수 없다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function largestRectangleInHistogram(heights):
    N       ← len(heights)
    stack   ← 빈 스택 (인덱스 저장)    // 불변식: 스택 내 heights 값이 단조 증가
    maxArea ← 0

    for i from 0 to N:                 // N 포함: 가상의 높이 0 막대
        h ← (i == N) ? 0 : heights[i]

        while stack 비어있지 않고 heights[stack.top()] > h:
            j ← stack.pop()            // j가 최소 높이인 직사각형 계산
            R ← i                      // 불변식: R = j 오른쪽의 첫 낮은 막대
            L ← (stack 비어있으면) -1 else stack.top()   // 불변식: L = j 왼쪽의 첫 낮은 막대
            area ← heights[j] * (R - L - 1)
            maxArea ← max(maxArea, area)

        stack.push(i)                  // 단조 증가 유지

    return maxArea
```
