# 모든 정점에서의 거리 합

## 한 줄 요약

> 가중치 없는 트리에서 각 정점을 출발점으로 삼았을 때 나머지 모든 정점까지 거리의 합을 담은 배열을 반환한다.

## 스토리

물류 회사가 전국의 창고 네트워크 중 어느 창고를 거점으로 삼으면 전체 배송 거리의 합이 가장 짧아지는지 분석하려 한다. 창고들은 도로로 연결되어 있으며 도로망은 사이클이 없는 구조다. 각 도로의 길이는 모두 동일하게 1로 취급한다.

어떤 창고를 거점으로 선택했을 때의 효율을 계산하려면, 그 창고에서 다른 모든 창고까지의 거리를 더해야 한다. 창고 수가 수만 개이므로, 각 창고마다 따로 탐색을 돌리는 방식은 너무 느리다.

모든 창고에 대한 거리 합을 한 번에 효율적으로 계산하는 방법이 필요하다.

## 함수 인터페이스

```ts
export function treeRerooting(n: number, edges: [number, number][]): number[];
```

- `n` — 정점 수
- `edges` — 무방향 간선 목록. 각 원소 `[u, v]`는 정점 `u`와 `v`를 잇는 간선 (0-based 인덱스). 모든 간선의 가중치는 1
- 반환 — 길이 `n`인 배열. 인덱스 `v`의 값은 정점 `v`에서 다른 모든 정점까지의 거리 합

## 제약 조건

- $1 \leq N \leq 10^5$ (정점 수)
- `edges`의 길이는 $N - 1$, 주어진 그래프는 연결된 트리를 형성한다
- 각 간선 $[u, v]$에 대해 $0 \leq u, v < N$, $u \neq v$ (0-based)
- 간선 방향은 없다. `[u, v]`와 `[v, u]`는 같은 간선
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

정점 $v$에서 정점 $u$까지의 거리는 두 정점을 잇는 경로 위의 간선 수다.

반환 배열의 인덱스 $v$에는 $\sum_{u \neq v} d(v, u)$의 값이 들어간다. 자기 자신까지의 거리는 0이므로 합산에서 제외해도 결과가 같다.

정점이 하나뿐이면 거리 합은 0이다.

## 예시

```ts
treeRerooting(1, []);
// [0] — 정점이 하나: 다른 정점 없음

treeRerooting(2, [[0, 1]]);
// [1, 1] — 각 정점에서 나머지 1개까지 거리 1

treeRerooting(3, [[0,1],[1,2]]);
// [3, 2, 3]
// S(0) = d(0,1)+d(0,2) = 1+2 = 3
// S(1) = d(1,0)+d(1,2) = 1+1 = 2
// S(2) = d(2,0)+d(2,1) = 2+1 = 3

treeRerooting(5, [[0,1],[0,2],[1,3],[1,4]]);
// [6, 5, 9, 8, 8]
//        0
//       / \
//      1   2
//     / \
//    3   4
// S(0) = 1+1+2+2   = 6
// S(1) = 1+2+1+1   = 5
// S(2) = 1+2+3+3   = 9
// S(3) = 2+1+3+2   = 8
// S(4) = 2+1+3+2   = 8

// 경계: 체인 트리 (선형, n=5)
treeRerooting(5, [[0,1],[1,2],[2,3],[3,4]]);
// [10, 7, 6, 7, 10]
// 체인 중앙에 가까울수록 거리 합이 작다

// 경계: 스타 트리 (n=5)
treeRerooting(5, [[0,1],[0,2],[0,3],[0,4]]);
// [4, 7, 7, 7, 7]
// 중심(0): 잎 4개까지 각 거리 1 → 합 4
// 잎: 중심까지 1, 다른 잎 3개까지 각 2 → 1+3×2 = 7
```
