# A* 탐색

## 한 줄 요약

> 함수는 그래프, 시작 정점, 목표 정점, 휴리스틱 함수를 받아 시작에서 목표까지의 최단 경로 비용을 반환한다.

## 스토리

지도 앱 개발자 지수는 수백만 개의 교차로가 이어진 도시 지도에서 경로를 탐색하는 기능을 만들고 있다. 문제는 교차로 수가 너무 많아서 모든 경로를 탐색하면 시간이 너무 오래 걸린다는 것이다.

지수는 목적지까지의 직선 거리를 미리 알 수 있다면 탐색 방향을 목적지 쪽으로 집중할 수 있다는 아이디어를 떠올렸다. "현재까지 이동한 실제 거리"와 "여기서 목적지까지 추정 거리"를 합산해 우선순위를 정하면 불필요한 방향은 탐색하지 않아도 된다.

단, 이 추정값은 실제 거리를 절대 초과해서는 안 된다. 과대평가를 하면 정작 최단 경로를 놓칠 수 있기 때문이다. 추정값이 정확할수록 탐색 속도가 빨라지고, 추정값이 항상 0이면 모든 방향을 균등하게 탐색하는 기존 방식과 동일해진다.

## 함수 인터페이스

```ts
export function aStarSearch(
  n: number,
  edges: [number, number, number][],
  src: number,
  goal: number,
  h: (v: number) => number,
): number;
```

- `n` — 정점 수. 정점 번호는 $0$부터 $n - 1$.
- `edges` — 방향 간선 목록. 각 원소는 $[u, v, w]$ 형태이며 가중치 $w \geq 0$인 방향 간선 $u \to v$.
- `src` — 시작 정점 번호.
- `goal` — 목표 정점 번호.
- `h` — 휴리스틱 함수. `h(v)`는 정점 $v$에서 목표 정점까지의 추정 비용. admissible 조건 $0 \leq h(v) \leq d(v, \text{goal})$을 만족한다고 보장된다.
- 반환 — $\text{src} \to \text{goal}$의 최단 경로 비용. 도달할 수 없으면 `Infinity`.

## 제약 조건

- $1 \leq V \leq 10^5$ (정점 수)
- $0 \leq E \leq 2 \cdot 10^5$ (간선 수)
- $0 \leq w(u, v) \leq 10^9$ (가중치; 음수 없음)
- $0 \leq h(v) \leq d(v, \text{goal})$ (admissible 휴리스틱)
- $0 \leq \text{src}, \text{goal} < V$
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

가중치가 음수가 아닌 방향 그래프 $G = (V, E)$가 주어진다. 시작 정점 $\text{src}$에서 목표 정점 $\text{goal}$까지의 최단 경로 비용을 구하라.

각 정점 $v$에는 admissible 휴리스틱 $h(v)$가 제공된다. $h(v)$는 $v$에서 $\text{goal}$까지의 실제 최단 거리를 절대 초과하지 않는다. $h \equiv 0$으로 설정하면 일반 단일 출발지 최단 경로 탐색과 동일하게 동작해야 한다.

- $\text{src} = \text{goal}$이면 경로 비용은 $0$이다.
- 목표 정점에 도달할 수 없으면 `Infinity`를 반환한다.
- 동일한 두 정점 사이에 다중 간선이 있으면 모두 고려해야 한다.

## 예시

```ts
// 정점 5개, 0 → 4로 가는 최단 경로
const edges: [number, number, number][] = [
  [0, 1, 1],
  [0, 2, 4],
  [1, 2, 2],
  [1, 3, 5],
  [2, 3, 1],
  [3, 4, 3],
];
aStarSearch(5, edges, 0, 4, () => 0); // 7 — 0→1→2→3→4 비용 합산

// src === goal이면 비용은 항상 0
aStarSearch(3, [[0, 1, 10]], 1, 1, () => 0); // 0 — 출발지와 목적지 동일

// 도달 불가능한 목표
aStarSearch(3, [[0, 1, 2]], 0, 2, () => 0); // Infinity — 정점 2로 가는 간선 없음

// admissible 휴리스틱 사용 (남은 정점 수로 추정)
const edgesLinear: [number, number, number][] = [
  [0, 1, 1],
  [1, 2, 1],
  [2, 3, 1],
  [0, 2, 10],
];
aStarSearch(4, edgesLinear, 0, 3, (v) => Math.max(0, 3 - v)); // 3 — 0→1→2→3 비용 합산
```
