# Bellman-Ford 최단 경로 해설

## 성능 목표 예측

| 항목 | 값 |
|------|-----|
| V (정점 수) | $1 \leq V \leq 500$ |
| E (간선 수) | $0 \leq E \leq V(V-1) \approx 2.5 \times 10^5$ |
| 가중치 범위 | $-10^9 \leq w(u, v) \leq 10^9$ |

### Naive 접근의 한계

음수 간선이 존재할 때, Dijkstra의 greedy 확정 원리가 깨진다. 예를 들어, $u$를 확정한 후 음수 간선을 통해 $u$에 더 짧게 도달할 수 있기 때문이다.

가장 단순한 대안은 모든 간선 쌍을 무한 반복 완화하며 수렴을 기다리는 것이다. 수렴 조건이 없으면 종료 보장이 없고, 그래서 "반복 횟수"를 어떻게 결정할지가 핵심이 된다.

### 목표 복잡도와 근거

Bellman-Ford는 $V-1$번의 완화 라운드를 수행한다.

$$O(V \cdot E) = 500 \times 2.5 \times 10^5 = 1.25 \times 10^8$$

연산 내용이 덧셈·비교로 단순하므로 통과 가능하다. $V \leq 500$이라는 제약이 핵심이며, 만약 $V = 10^5$이었다면 $O(V \cdot E) = 2 \times 10^{10}$으로 불가능하다.

### 공간 복잡도

- 거리 배열: $O(V)$
- 별도 행렬 불필요 — 음수 간선도 1D 배열로 처리

## 목표 함수

```ts
function bellmanFord(
  n: number,
  edges: [number, number, number][],
  src: number,
): { dist: number[]; hasNegativeCycle: boolean }
```

| 파라미터 | 의미 | 제약 |
|---------|------|------|
| `n` | 정점의 개수 $V$ | $1 \leq n \leq 500$ |
| `edges` | 방향 간선 목록 `[u, v, w]` | 가중치 음수 허용 |
| `src` | 시작 정점 $s$ | $0 \leq src < n$ |

**반환값**:
- `dist[v]`: $s \to v$ 최단 거리. 도달 불가능하면 `Infinity`
- `hasNegativeCycle`: $s$에서 도달 가능한 음수 사이클이 존재하면 `true`

**엣지케이스**:
1. `dist[src] = 0`: 자기 자신까지의 거리는 항상 0
2. `dist[u] === Infinity`일 때 완화 시도 금지: `Infinity + w`는 수치 오버플로우 또는 잘못된 갱신 유발
3. 음수 사이클이 $s$에서 도달 불가능한 곳에 있으면 `hasNegativeCycle = false`
4. 간선이 없는 그래프: `dist[src] = 0`, 나머지 `Infinity`, `hasNegativeCycle = false`

## 핵심 아이디어

**핵심 아이디어**: "최단 경로는 간선을 최대 V-1개 쓴다 — 이 사실로 반복 횟수를 확정한다."

음수 간선이 있으면 Dijkstra의 greedy 확정이 깨진다. Bellman-Ford는 "음수 사이클이 없다면 최단 단순 경로는 간선 V-1개면 충분하다"는 관찰을 이용해 모든 간선을 V-1번 반복 완화한다. V번째 라운드에서 여전히 완화가 가능하면 음수 사이클이 존재한다는 뜻이다.

**풀이 구조**
1. `dist` 배열 초기화: `dist[src] = 0`, 나머지 Infinity
2. V-1번 반복: 모든 간선 $(u, v, w)$에 대해 `dist[u] + w`가 `dist[v]`보다 작으면 `dist[v]` 갱신
3. V번째 라운드에서 같은 완화를 시도해 갱신이 발생하면 음수 사이클로 판정
4. `{ dist, hasNegativeCycle }` 반환

**조건**: 음수 간선 허용. 음수 사이클이 있는 경우 검출만 가능하며 최단 거리는 정의되지 않음.

**대표 예시**: 환율 차익 거래 경로 탐색 (음수 사이클 = 무한 수익 구간 탐지)
각 통화 사이의 환율을 로그 변환해 음수 가중치 간선으로 만들면, 음수 사이클이 차익 거래 기회를 나타낸다. Bellman-Ford의 V번째 라운드 완화 여부로 이를 판별할 수.

**언제 쓰나**
음수 가중치 간선이 있고 음수 사이클 존재 여부까지 확인해야 할 때 사용한다. 정점 수 V가 수백~수천 수준이고 Dijkstra를 쓸 수 없는 상황에서 선택한다.

### 원형 아이디어와 naive 접근

음수 간선이 있는 그래프에서 최단 경로를 구하는 가장 단순한 접근은 모든 경로를 열거하는 것이다. 그러나 경로 수가 지수적이므로 불가능하다.

조금 더 나은 접근으로 "모든 간선을 순서 없이 반복 완화"하는 방법을 생각할 수 있다.

```
dist = [Infinity] * n; dist[src] = 0

무한 반복:
    changed = false
    for (u, v, w) in edges:
        if dist[u] + w < dist[v]:
            dist[v] = dist[u] + w
            changed = true
    if not changed: break
```

이 방법은 음수 사이클이 없으면 수렴하지만, **몇 번을 반복해야 수렴하는지 알 수 없다**. 또한 음수 사이클이 있으면 무한 루프에 빠진다. 반복 횟수를 한정하는 이론적 근거가 필요하다.

### 어떤 관찰이 돌파구가 되는가

- **관찰 1**: 음수 사이클이 없는 그래프에서 단순 최단 경로(정점을 반복하지 않는 경로)는 최대 $V-1$개의 간선을 사용한다. $V$개 이상의 간선을 사용하면 비둘기집 원리에 의해 어떤 정점이 두 번 등장하며(사이클이 생기고), 음수 사이클이 없다면 그 사이클을 제거하는 것이 항상 이득이다.

- **관찰 2**: $i$번의 완화 라운드 후 `dist[v]`는 "최대 $i$개의 간선을 사용하는 $s \to v$ 최단 경로"를 반영한다. 이를 귀납적으로 증명할 수 있다.

- **관찰 3**: 따라서 $V-1$번 반복하면 모든 실제 최단 경로가 확정된다. 만약 $V$번째 라운드에서도 완화가 가능하다면, $V$개의 간선을 사용하는 경로가 더 짧다는 뜻 = 사이클을 포함하는 경로가 더 짧다 = 음수 사이클 존재.

### 관찰을 형식화: 상태/구조 정의

**상태 정의**: `dist[v]` = 현재까지 발견한 $s \to v$ 최단 경로 비용.

$dp[i][v]$를 "최대 $i$개의 간선을 사용하는 $s \to v$ 최단 경로 비용"으로 정의하면:

$$dp[i][v] = \min\bigl(dp[i-1][v],\; \min_{(u,v,w) \in E}\bigl(dp[i-1][u] + w\bigr)\bigr)$$

- 첫 번째 항: $v$를 간선 $i$번째에 "포함시키지 않음" (이전과 동일)
- 두 번째 항: $i$번째 간선으로 $(u, v)$를 선택

왜 이 정의여야 하는가: 간선 수를 기준으로 상태를 정의해야만 반복 횟수의 상한을 $V-1$로 결정할 수 있다. "비용 기준" 상태는 음수 간선에서 단조성이 보장되지 않는다.

**공간 최적화**: $dp[i]$를 계산할 때 $dp[i-1]$만 필요하다. 그런데 같은 라운드에서 이미 갱신된 `dist[u]`를 사용해도 정확성이 유지된다(더 짧은 경로를 조기에 발견할 뿐이다). 따라서 1D 배열을 in-place로 갱신해도 된다.

### 점화식 또는 핵심 연산

**완화 연산** (매 라운드, 모든 간선에 대해):

$$dist[v] \leftarrow \min\bigl(dist[v],\; dist[u] + w(u, v)\bigr) \quad \text{if } dist[u] \neq \infty$$

- $dist[u] + w(u, v)$: 현재까지 발견한 $s \to u$ 경로에 간선 $(u, v)$를 추가한 비용
- $dist[u] \neq \infty$ 조건: 아직 도달하지 못한 정점에서 완화를 시도하면 수치 오류 발생

**음수 사이클 검출** ($V$번째 라운드):

$$\exists (u, v, w) \in E : dist[u] \neq \infty \land dist[u] + w < dist[v] \implies \text{음수 사이클 존재}$$

### 정당성 — 왜 이것이 옳은가

**귀납적 정당화**: $i$번째 라운드 종료 후, `dist[v]`는 최대 $i$개의 간선을 사용하는 $s \to v$ 경로 중 최솟값이다.

**기저 ($i = 0$)**: `dist[src] = 0`(간선 0개로 자기 자신 도달), 나머지 `Infinity`.

**귀납 단계**: $i-1$번째 라운드 후 주장이 성립한다고 가정하자. $i$번째 라운드에서 간선 $(u, v, w)$를 완화할 때 $dist[u]$는 최대 $i-1$개의 간선을 쓰는 $s \to u$ 최솟값이다. $dist[u] + w$는 최대 $i$개의 간선을 쓰는 $s \to v$ 경로 하나이다. 모든 간선에 대해 최솟값을 취하면 $i$번째 주장이 성립한다.

**음수 사이클이 없을 때**: 단순 경로는 최대 $V-1$개의 간선을 쓰므로 $i = V-1$에서 `dist`가 완전히 수렴한다. $V$번째 라운드에서 완화가 불가능하다.

**까다로운 케이스**: 도달 불가능한 정점을 경유하는 음수 사이클 — `dist[u] = Infinity`인 $u$에서 완화를 시도하지 않으므로 자동으로 걸러진다. `hasNegativeCycle`은 $s$에서 실제로 도달 가능한 사이클만 감지한다.

### 구현 디테일과 최적화

**`Infinity` 처리**: TypeScript에서 `Infinity + w`는 `Infinity`이지만, 일부 언어에서는 오버플로우가 발생한다. `dist[u] !== Infinity` 조건을 반드시 추가한다.

**조기 종료**: 어느 라운드에서도 갱신이 발생하지 않았다면 이미 수렴했으므로 루프를 종료해도 된다. 최악의 경우에는 $V-1$번 모두 돌아야 하지만, 희소 그래프에서 조기 종료로 실질적 속도 향상이 가능하다.

**함정 — 중복 카운트**: 같은 라운드 내에서 이미 갱신된 `dist[u]`를 사용하면 실질적으로 $V-1$번보다 적은 라운드로 수렴할 수 있다. 이것은 정확성에 문제가 없다(더 빨리 수렴할 뿐). 단, 음수 사이클 검출 정확성에는 영향이 없다.

**함정 — 루프 순서**: 간선 목록 순서가 결과에 영향을 주지 않는다. 어떤 순서로 완화해도 $V-1$번 반복 후 결과는 동일하다.

## 수도 코드와 Activity Diagram

### 의사코드

```
function bellmanFord(n, edges, src):
  dist ← [Infinity] * n           // 불변식: dist[v]는 현재까지 발견한 s→v 최선값
  dist[src] ← 0

  // V-1번 완화 라운드
  for i in 1 .. n-1:
    for (u, v, w) in edges:
      if dist[u] ≠ Infinity and dist[u] + w < dist[v]:
        dist[v] ← dist[u] + w
  // 불변식: i번 라운드 종료 후, dist[v]는 간선 ≤ i개를 쓰는 s→v 최솟값

  // V번째 라운드: 음수 사이클 검출
  hasNegativeCycle ← false
  for (u, v, w) in edges:
    if dist[u] ≠ Infinity and dist[u] + w < dist[v]:
      hasNegativeCycle ← true     // V개의 간선을 쓰는 더 짧은 경로 = 음수 사이클
      break

  return { dist, hasNegativeCycle }
```

**핵심 불변식**: $i$번째 외부 루프 종료 시, `dist[v]`는 $s$에서 $v$까지 최대 $i$개의 간선을 사용하는 경로 중 최솟값이다.
