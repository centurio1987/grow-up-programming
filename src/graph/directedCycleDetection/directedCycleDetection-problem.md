# Directed Cycle Detection (유향 그래프 사이클 탐지)

## 한 줄 요약

> 함수는 정점 수 `n`과 유향 간선 목록을 받아, 그래프에 사이클이 존재하면 `true`, 없으면 `false`를 반환한다.

## 스토리

소프트웨어 패키지 관리 시스템을 개발하는 팀이 있다. 각 패키지는 다른 패키지에 의존할 수 있는데, A가 B에 의존하고 B가 다시 A에 의존하면 설치가 불가능한 순환 의존성이 생긴다.

팀은 새 패키지를 등록할 때마다 "이 의존 관계 그래프에 순환 고리가 생기지 않는가"를 확인해야 한다. 순환 고리가 하나라도 있으면 전체 설치 과정이 멈춘다.

설치 단계에 진입하기 전에 이 검사를 통과해야만 배포가 진행된다.

## 함수 인터페이스

```ts
export function directedCycleDetection(
  n: number,
  edges: [number, number][],
): boolean;
```

- `n` — 정점 수. 정점 번호는 $0 \ldots n-1$
- `edges` — 유향 간선 목록. 각 원소 `[u, v]`는 $u \to v$ 방향의 간선
- 반환 — 사이클이 존재하면 `true`, 존재하지 않으면 `false`

## 제약 조건

- $1 \leq V \leq 10^{5}$ (여기서 $V = n$)
- $0 \leq E \leq 10^{5}$ (여기서 $E$는 `edges`의 길이)
- 방향 그래프
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

유향 그래프에 **사이클이 존재하는지 여부** 를 판별한다.

사이클이란 같은 정점에서 출발해 방향을 따라 이동했을 때 다시 그 정점으로 돌아올 수 있는 경로이다.

경계 동작:
- 자기 루프 `[v, v]`는 사이클로 간주한다.
- 무향 그래프와 달리, 방향이 반대인 두 간선 `[u, v]`와 `[v, u]`는 사이클을 형성한다.
- 교차 간선(이미 완료된 서로 다른 경로를 연결하는 간선)은 사이클이 아니다.
- 그래프가 분리된 여러 성분으로 이루어진 경우, 어느 하나에라도 사이클이 있으면 `true`를 반환한다.
- 간선이 없으면 사이클이 없다.

## 예시

```ts
// 유향 사이클: 0→1→2→0
directedCycleDetection(3, [[0,1],[1,2],[2,0]]); // true

// 자기 루프: 0→0
directedCycleDetection(1, [[0,0]]); // true

// 양방향 간선: 0→1, 1→0 — 유향 사이클 형성
directedCycleDetection(2, [[0,1],[1,0]]); // true

// 선형 DAG: 0→1→2→3 — 사이클 없음
directedCycleDetection(4, [[0,1],[1,2],[2,3]]); // false

// 다이아몬드 DAG: 0→1, 0→2, 1→3, 2→3 — 교차 간선, 사이클 아님
directedCycleDetection(4, [[0,1],[0,2],[1,3],[2,3]]); // false

// 간선 없음
directedCycleDetection(5, []); // false
```
