# Bridges in Graph (다리 검출)

## 중요도 · 난이도

| 항목 | 값 |
|------|-----|
| 중요도 | ★ 하 — 특정 분야·고급 |
| 난이도 | 고급 |

## 함수 인터페이스

```ts
export function bridgesInGraph(
  n: number,
  edges: [number, number][],
): [number, number][];
```

## 제약 조건

- $1 \leq V \leq 10^{5}$ (여기서 $V = n$, 정점 번호는 $0 \ldots n-1$)
- $0 \leq E \leq 10^{5}$ (여기서 $E$ 는 `edges` 의 길이)
- `edges` 는 무향 간선 목록

## 문제 상세

무향 그래프 $G = (V, E)$ 에서 **다리(bridge, cut edge)** 를 모두 찾는다.

간선 $(u, v)$ 가 다리라는 것은, 해당 간선을 제거했을 때 연결 성분의 개수가 증가한다는 의미이다.

반환 형식:

- 각 다리 $[u, v]$ 는 $u < v$ 가 되도록 정규화한다.
- 전체 배열은 사전순(첫 원소 → 둘째 원소)으로 정렬한다.

## 예시

다음과 같은 그래프를 생각하자.

```
    0 --- 1 --- 2
          |
          3 --- 4
```

- `n = 5`
- `edges = [[0,1], [1,2], [1,3], [3,4]]`

각 간선을 하나씩 제거하면 모두 그래프가 둘로 쪼개진다. 따라서 모든 간선이 다리이다.

반환값은 `[[0,1], [1,2], [1,3], [3,4]]`.

다른 예: 삼각형 그래프.

- `n = 3`
- `edges = [[0,1], [1,2], [2,0]]`

어떤 간선을 제거해도 나머지 두 간선으로 연결이 유지되므로 다리가 없다. 반환값은 `[]`.
