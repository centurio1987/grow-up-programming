# 0-1 BFS (이진 가중치 최단 경로)

## 한 줄 요약

> 함수는 정점 수 `n`, 가중치가 0 또는 1인 유향 간선 목록, 출발 정점 `source`를 받아, 각 정점까지의 최소 비용을 담은 배열을 반환한다.

## 스토리

게임 맵을 설계하는 개발자 준혁은 캐릭터가 이동할 때 "일반 이동"과 "순간이동" 두 종류를 쓸 수 있게 했다. 일반 이동은 비용이 1이고, 같은 구역 내 순간이동 연결은 비용이 0이다.

준혁은 특정 지점에서 출발해 맵의 모든 지점까지 도달하는 데 드는 최솟값 비용을 계산하고 싶다. 비용이 0인 경로를 최대한 활용하면 기대보다 훨씬 싸게 도달할 수 있는 지점도 있다.

아예 이동 경로가 없는 지점은 도달 불가를 나타내는 -1로 표시하기로 했다.

## 함수 인터페이스

```ts
export function zeroOneBfs(
  n: number,
  edges: [number, number, number][],
  source: number,
): number[];
```

- `n` — 정점 수. 정점 번호는 $0 \ldots n-1$
- `edges` — 유향 간선 목록. 각 원소 `[u, v, w]`는 $u \to v$ 방향의 간선이며 가중치 $w \in \{0, 1\}$
- `source` — 출발 정점 번호 ($0 \leq \text{source} < n$)
- 반환 — 길이 `n`의 배열. 인덱스 `i`의 값은 `source`에서 정점 `i`까지의 최소 비용. 도달 불가능한 정점은 `-1`. `source` 자신은 `0`

## 제약 조건

- $1 \leq V \leq 10^{5}$ (여기서 $V = n$)
- $0 \leq E \leq 10^{5}$ (여기서 $E$는 `edges`의 길이)
- $w \in \{0, 1\}$ (가중치는 0 또는 1)
- $0 \leq \text{source} < n$
- 방향 그래프
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

모든 간선의 가중치가 $0$ 또는 $1$인 그래프에서 출발점 $s$로부터 모든 정점까지의 **최소 비용(가중치 합)** 을 구한다.

반환값은 길이 $n$의 배열이며 인덱스 $i$의 값은 $s$에서 $i$까지의 최소 가중치 합이다.

경계 동작:
- 도달 불가능한 정점의 값은 `-1`이다.
- `source` 자신의 거리는 `0`이다.
- 비용 0인 간선들만으로 연결된 정점들은 서로 거리가 `0`이다.
- 비용 0인 간선들로 이루어진 사이클이 있어도 무한 루프 없이 올바른 결과를 반환해야 한다.
- 간선이 없으면 `source` 이외의 모든 정점은 `-1`이다.

## 예시

```ts
// 모든 가중치 1 — 간선 수 = 비용
zeroOneBfs(4, [[0,1,1],[1,2,1],[2,3,1]], 0); // [0, 1, 2, 3]

// 모든 가중치 0 — 어디든 비용 0
zeroOneBfs(4, [[0,1,0],[1,2,0],[2,3,0]], 0); // [0, 0, 0, 0]

// 혼합: 0→3(비용0)→2(비용0)가 0→1(비용1)→2(비용1)보다 저렴
// 2까지 비용은 0, 1까지 비용은 1
zeroOneBfs(4, [[0,1,1],[1,2,1],[0,3,0],[3,2,0]], 0); // [0, 1, 0, 0]

// 도달 불가: 0→1(비용1) | 2→3(비용0), source=0
zeroOneBfs(4, [[0,1,1],[2,3,0]], 0); // [0, 1, -1, -1]

// source만 존재 — 자신만 0
zeroOneBfs(3, [], 1); // [-1, 0, -1]

// V=1
zeroOneBfs(1, [], 0); // [0]
```
