# Max Profit

## 한 줄 요약

> 함수는 일별 주식 가격 배열을 받아, 한 번의 매수·매도로 얻을 수 있는 최대 이익을 반환한다.

## 스토리

한 투자자가 특정 종목의 $N$일치 주식 가격 데이터를 보고 있다. 그는 단 한 번, 어느 날 사서 그보다 늦은 날 팔 계획이다. 물론 사는 날보다 파는 날이 같거나 뒤여야 한다.

시장이 계속 하락하는 경우처럼 어떤 조합으로도 이익을 낼 수 없다면 포기하고 0을 기록한다. 배열이 비어 있으면 거래 자체가 불가능하므로 마찬가지로 0이다.

주어진 가격 데이터에서 한 번의 거래로 얻을 수 있는 최대 이익은 얼마인가?

## 함수 인터페이스

```ts
export function maxProfit(A: number[]): number;
```

- `A` — 길이 $N$인 정수 배열. `A[i]`는 $i$번째 날의 주식 가격
- 반환 — 한 번의 거래로 얻을 수 있는 최대 이익. 이익이 없거나 배열이 비어있으면 0

## 제약 조건

- $0 \leq N \leq 400{,}000$ ($N$은 `A`의 길이)
- 각 원소는 $0 \leq A[i] \leq 200{,}000$인 정수
- 시간 제한: 1초, 메모리 제한: 256 MB

## 문제 상세

매수일 $P$, 매도일 $Q$를 $0 \leq P \leq Q < N$ 범위에서 선택할 때, 거래 이익은 $A[Q] - A[P]$다. 최대 이익은 다음과 같이 정의된다.

$$\text{maxProfit}(A) = \max\left(0,\; \max_{0 \leq P \leq Q < N}\, \big(A[Q] - A[P]\big)\right)$$

이익이 음수가 되는 경우는 거래하지 않은 것으로 간주하여 0을 반환한다. 배열의 길이가 0이거나 1인 경우에도 거래가 불가능하므로 0을 반환한다.

## 예시

```ts
maxProfit([23171, 21011, 21123, 21366, 21013, 21367]);
// 356 — day 1(21011)에 매수, day 5(21367)에 매도: 21367 - 21011 = 356

maxProfit([7, 1, 5, 3, 6, 4]);
// 5 — day 1(1)에 매수, day 4(6)에 매도: 6 - 1 = 5

maxProfit([7, 6, 4, 3, 1]);
// 0 — 단조 감소로 이익을 낼 수 없음

maxProfit([]);
// 0 — 배열이 비어있어 거래 불가

maxProfit([42]);
// 0 — 원소가 하나뿐이어서 매수·매도 쌍을 만들 수 없음

maxProfit([0, 200000]);
// 200000 — day 0에 0원에 매수, day 1에 200000원에 매도
```
